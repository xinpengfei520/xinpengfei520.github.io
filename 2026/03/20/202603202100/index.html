别在 ChatGPT 和 Claude 之间反复横跳了！揭秘大厂疯狂抢夺开发者背后的真相

发表于 2026-03-20 更新于 2026-04-08 分类于 观点 阅读次数： 本文字数： 3.2k 阅读时长 ≈ 3 分钟

你有没有这种感觉？

上个月刚把工作流切到 Claude Code，这个月 OpenAI 又放出 Codex 大招。群里一半人喊”Claude 真香”，另一半在转发”Codex 碾压测试”。你打开钱包一看，两边的 Pro 订阅都在自动扣费。

别急着站队。因为你纠结的这个问题背后，藏着一场比”谁家模型更强”大得多的战争——AI 大厂正在用真金白银，疯狂抢夺开发者。

今天我们不聊”哪个模型更好用”这种月经话题。我想和你聊聊，这场开发者争夺战背后的商业逻辑，以及它会怎样影响你我每一个写代码的人。

一场价值万亿的抢人大战

先看几个数字，感受一下这场战争的烈度。

2026 年 3 月，Anthropic 的 Claude Code 年化收入突破 25 亿美元，占公司总收入近五分之一。OpenAI 的 Codex 紧随其后，年化收入刚过 10 亿美元。差距是明显的，但两家都在狂奔。

更疯狂的是收购。上周 OpenAI 宣布收购 Python 工具链公司 Astral——就是做 uv、ruff 这些工具的那个团队。再往前，Anthropic 收购了 Bun，Google DeepMind 买下了 Antigravity 团队。

你没看错。每一家认真做开发者生态的 AI 公司，都在买开发工具公司。

这不是巧合。Sam Altman 在接受 WIRED 采访时说了一句大实话：”编程可能是通向 AGI 最有可能的路径。”翻译成商业语言就是：谁拿下了开发者，谁就拿下了 AI 时代的入口。

为什么开发者成了兵家必争之地？

你可能会想：AI 公司赚钱不是靠 C 端用户吗？ChatGPT 周活都快 10 亿了。

没错，但 C 端用户和企业客户是两回事。

Axios 最近披露了一组关键数据：在企业 AI 采购中，Anthropic 已经拿下了首次购买客户 73% 的支出份额。10 周前，Anthropic 和 OpenAI 在企业市场还是五五开，现在天平已经明显倾斜。

为什么？因为企业买 AI 不是买聊天机器人，是买生产力工具。而生产力工具的核心用户，就是开发者。

想想这个链条：

  1. 开发者用你的编程工具 → 写出的代码依赖你的 API
  2. 代码上线 → 公司持续消耗你的 token
  3. 公司尝到甜头 → 签年度企业合同
  4. 更多开发者被公司要求使用 → 飞轮转起来

这就是为什么 Anthropic 花 1 亿美元建”Claude 合作伙伴网络”，为什么 OpenAI 把 Sora、Atlas 浏览器这些”花活”往后放，集中火力做编程和企业服务。

开发者不是用户，是渠道。

OpenAI 的焦虑：起了个大早，赶了个晚集

WIRED 那篇深度报道里有个细节特别有意思。

2021 年，OpenAI 就做出了第一版 Codex。那时候 Greg Brockman 兴奋地展示它怎么把英文指令变成代码。微软拿它做了 GitHub Copilot，几个月就拿下了几十万用户。

然后呢？OpenAI 的 Codex 团队被拆了。一部分人去做 DALL-E，一部分人去训练 GPT-4。ChatGPT 爆火之后，所有资源都涌向了消费级产品。视频、语音、图片、浏览器、硬件……OpenAI 什么都想做。

与此同时，Anthropic 在闷头做一件事：把模型的编程能力做到极致。

Brockman 自己在播客里承认：Anthropic 很早就专注于用真实代码仓库训练模型，而不只是用竞赛题。”这是我们学晚了的一课。”

2024 年中，Anthropic 发布 Claude Sonnet 3.5。Cursor 接入这个模型后，使用量开始暴涨。几个月后，Anthropic 开始内测 Claude Code。OpenAI 想收购 Cursor，被拒绝了。

等 OpenAI 反应过来组建专门团队，已经是 2024 年底。整整晚了大半年。

在 AI 行业，半年就是一个时代。

Anthropic 做对了什么？

说实话，Anthropic 的崛起有运气成分——比如 QuitGPT 运动给它送来了 250 万 ChatGPT 出走用户。但运气之外，它确实做对了几件关键的事。

第一，极致聚焦。

当 OpenAI 在做视频生成器、浏览器、电商功能的时候，Anthropic 只做一件事：让 Claude 成为最好的编程 AI。Sherwood News 的标题概括得精准：”OpenAI 在发布一切，Anthropic 在打磨一件事。”

结果是：全球 GitHub 上 4% 的公开代码提交，是 Claude Code 自动生成的。一个月前这个数字还只有 2%。

第二，开发者体验优先。

Claude Code 直接跑在终端里，能访问你所有的文件和应用。这不是”聊天式编程”，而是真正的代理式编程——你把任务丢给它，它自己去翻代码库、改文件、跑测试。

日本乐天的工程师用 Claude Code，在 1250 万行代码的 vLLM 项目里，7 小时完成了一个特定任务，精度 99.9%。Anthropic 自己的工程师，70% 到 90% 的代码在 Claude 辅助下完成。

第三，生态卡位。

MCP（Model Context Protocol）是 Anthropic 在 2024 年推出的开放协议，你可以把它理解成”AI 的 USB-C 接口”。任何工具只要实现了 MCP，就能和 Claude 无缝对接。

最近 Anthropic 又推出了 Claude Code Channels，让开发者通过 Telegram、Discord 直接和 Claude Code 对话，随时随地让它帮你写代码。从同步等待变成异步协作——这才是真正的工作流革命。

OpenAI 的反击：超级应用 + 疯狂收购

OpenAI 当然不会坐以待毙。

就在今天，消息传出 OpenAI 正在开发桌面端”超级应用”，要把 ChatGPT、Codex 编程助手和 Atlas 浏览器合并成一个产品。应用负责人 Fidji Simo 说，产品碎片化已经成了质量的障碍。

翻译一下：之前摊子铺太大了，现在要收缩战线，集中打。

收购 Astral 更是一步狠棋。uv 是 Python 社区增长最快的包管理工具，ruff 是最流行的 linter 之一。拿下它们，OpenAI 就在 Python 开发者的工作流里钉了一颗钉子。

再加上 GPT-5.4 在 OSWorld-Verified 测试中拿到 75% 的成绩（Claude Opus 4.6 是 72.7%），模型能力上的差距正在缩小。Reddit 上已经有不少开发者表示，Codex 5.3 的编程体验开始超越 Claude。

这场仗远没到终局。

对普通开发者来说，这意味着什么？

说了这么多大厂博弈，你可能想问：跟我有什么关系？

关系大了。

短期来看，你是最大受益者。 两家疯狂竞争的结果，就是工具越来越好、价格越来越低。Claude Pro 20 刀包含 Claude Code，ChatGPT Plus 20 刀包含 Codex。半年前这些功能都是额外收费的。GPT-5.4 mini 的 API 价格已经低到 0.75 美元/百万 token。

中期来看，你需要警惕锁定效应。 当你的代码库深度依赖某家的 API，当团队的工作流完全绑定某个编程 Agent，切换成本会越来越高。OpenAI 买 Astral、Anthropic 推 MCP，本质上都是在构建护城河。

长期来看，编程这件事本身在变。 当 AI 能自主完成 70% 以上的代码，”写代码”不再是核心技能，”定义问题”和”审查代码”才是。Cursor 的创始团队说得很直白：软件开发的未来，不是写代码。

我的建议很简单：

  • 两边都试，不要押注一方
  • 关注工作流整合能力，而不只是模型跑分
  • 投资自己”定义问题”的能力，这是 AI 最难替代的

写在最后

ChatGPT 和 Claude 之间的竞争，表面上是两个产品的对比，底层是两种商业哲学的碰撞。

OpenAI 想做 AI 时代的 Google——什么都有，无处不在。Anthropic 想做 AI 时代的 AWS——深耕基础设施，让开发者离不开。

谁会赢？可能都不会。因为最终受益的，是我们这些在两者之间”反复横跳”的开发者。正是这种竞争，推动着整个行业以不可思议的速度进化。

所以下次有人问你”该选 ChatGPT 还是 Claude”，你可以告诉他：别选。两个都用，然后享受这个开发者被疯抢的黄金时代。
