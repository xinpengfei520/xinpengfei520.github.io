OpenAI 放出最强模型 Sol，却第一次「发不出去」：GPT-5.6 被政府按住了

发表于 2026-06-27 更新于 2026-07-11 分类于 AI 阅读次数： 本文字数： 2.5k 阅读时长 ≈ 2 分钟
OpenAI 6 月 26 日预览 GPT-5.6 三个模型：旗舰 Sol、均衡 Terra、最便宜 Luna，定价 5/30、2.5/15、1/6 美元每百万 token。但最强的 Sol 只开给约 20 家受信任机构，ChatGPT 用户暂时摸不到，因为美国政府要求先审再放。前沿模型第一次像受管制品一样发布，这比模型本身更值得盯。

哈喽，我是飞飞。

6 月 26 日，OpenAI 发了它「史上最强」的模型 GPT-5.6。按惯例，这种消息我刷到都会点进去看一眼能不能马上用上。结果这次点进去，愣了一下。

最强的那个模型，我用不上。你大概率也用不上。

不是排队、不是灰度、不是要充会员。是 OpenAI 主动把它按住了，只发给大约 20 家机构，普通人连 ChatGPT 里都见不到。原因写在公告里：美国政府要求先审一遍，再决定放多大。

这事比模型本身有意思多了。

先把三个模型说清楚

GPT-5.6 这次不是一个模型，是一家三口。

老大叫 Sol，旗舰，OpenAI 说是迄今最强的那个，编程、网络安全、生物这几块都往前蹦了一大截。老二叫 Terra，主打均衡，性能跟上一代 GPT-5.5 差不多，但便宜了一半，是给那种「天天要跑大量活、又不想为每次都掏旗舰的钱」的场景准备的。老三叫 Luna，最快、最便宜，干轻量杂活。

价格摆出来你就懂这个梯队了。按每百万 token 算，Sol 是 5 美元进、30 美元出；Terra 砍到 2.5 和 15；Luna 再降到 1 和 6。从老大到老三，一档比一档省。

这套梯队我看着特别亲切。因为我自己每天就是这么用模型的。

让 agent 干一件正经活之前，我都会下意识掂量一下，这活值不值得动用最贵的那档。润色稿子、改流水线脚本这种要动脑子的，舍得上旗舰；批量改个文件名、跑个格式转换这种纯体力杂活，就手动切到便宜模型省点额度。每个月给模型交的钱不算少，这个「过秤」的动作早成了肌肉记忆，省下来的都是真金白银。OpenAI 现在把这件事直接做进了产品线，Sol、Terra、Luna 摆成三档，等于替你把「这活配哪档模型」的选择题提前列好了。

命名变了，这步是抄 Claude 的

还有个容易被划过去的细节：OpenAI 这回顺手把命名规则也改了。

以前是 GPT-4、GPT-5 这么往上数，一个数字一个版本。现在数字只代表「第几代」，而 Sol、Terra、Luna 代表三个能力档位，各自能按自己的节奏往前升。说白了就是，旗舰更新一波，不一定非得拖着便宜款一起改名。

这套「代际号 + 能力名」的玩法，明眼人一看就知道是跟 Anthropic 的 Claude 学的。Claude 早就在用类似的分层名字。OpenAI 这次几乎是照着搬。

我不觉得这是坏事。模型一多，光靠数字根本记不住谁比谁强、谁比谁贵。给能力档位起个固定名字，用户认起来确实顺。只是当行业老大都开始抄老二的命名了，你能感觉到这俩咬得有多紧。

真正反常的是「发不出去」这件事

回到开头那个让我愣住的点。

一个公司憋了大招、说自己造出了最强的东西，正常剧情是敲锣打鼓全网开放、生怕你不知道。GPT-5.6 反着来：最强的 Sol 只给约 20 家受信任的合作机构，走 API 和 Codex，ChatGPT 里压根没有。OpenAI 还特意说明，这是应美国政府的要求，提前把模型能力和发布计划报备过了，政府点头才小范围放。

为什么政府要管？核心顾虑是网络安全和生物这两块。Sol 在这些方向太能打了。OpenAI 自己也承认，Sol 在安全能力上够到了它内部框架里的「高」档，虽然还没到「危急」。在一个叫 ExploitBench 的漏洞测试上，Sol 用大约三分之一的输出量就追平了对手的同级模型。这种能力，用来帮防守方查漏补缺是好事，可一旦被人拿去干坏事，政府就坐不住了。

所以你看到的画面是：模型造好了，技术上随时能发，但闸门捏在审批流程手里。

这不是 OpenAI 一家的待遇。在它之前，Anthropic 的几个强模型也碰过类似的限制。前沿模型正在被当成一类需要先过审、才能见人的东西。OpenAI 在公告里其实憋着火，话说得挺直接：这种政府审一遍才放的流程，不该变成以后的默认操作，它会把最好的工具挡在真正需要的人外面。

我能理解这股憋屈。但反过来站到管的那边想，也不是全无道理。一个能大幅提升网络攻击效率的东西，跟一个普通 App，确实不该用同一套规则发。这道理跟你不会让管制刀具在便利店随便卖是一样的。难的从来不是「该不该管」，而是那条线到底划在哪、谁来划、划完之后还能不能改。这几个问题，现在没一个人答得上来。

对我们这种天天用 AI 干活的人意味着什么

说点实在的。短期看，这事对你我影响没那么大。

Terra 和 Luna 这两个能用得上的档位，性能、价格都摆在那，等过几周全面开放，该提速提速、该省钱省钱。真正被锁住的是金字塔尖那个 Sol，而日常写代码、跑流水线，多数活儿压根用不到最顶配。

但往远了看，我心里是有点别扭的。

我自己这套写作流水线，挂着十几个 skill，拉资讯、查重、配图、发公众号，全靠 AI 跑。这套东西的底座，就是 Codex 和 Claude Code 这类模型。每一个 skill 都是照着这些模型读配置、认格式、调工具的脾气，一点点磨出来的。真要换个新底座，这些 skill 吐出来的格式十有八九会飘，光是把那套专门防 AI 味的 grep 规则重新调顺，就够我搭进去好几个晚上。所以我对「最新最强的能力，第一时间能接进现成工作流」这件事，是真有依赖的。

GPT-5.6 把这个默认前提打破了：从今往后，最强的那一档可能先得过一道审批，才轮得到普通开发者。我磨了大半年的这套流水线，未来想升级到顶配底座，中间可能要多卡一道我控制不了的门。

能力会不会因此卡在少数人手里？最前沿的生产力工具,会不会以后都带个「需审批」的门槛？这些我现在答不上来，但它确实是个值得一直盯着的转向。

写在最后

绕回开头。GPT-5.6 这天发生的事，表面是 OpenAI 又出了个更强的模型，三档梯队、价格屠刀、命名翻新，热闹得很。

可真正写进历史的那一笔，可能是另一件不起眼的事：一个公司造出了自己最强的东西，第一反应不是赶紧发，而是先送去给政府看一眼。前沿模型从「造出来就发」，开始走向「造出来还得先报备」。

技术的脚还在往前狂奔，可发布这道门，第一次被外面的手扶住了。这道门以后是越关越紧，还是只是这一阵子的紧张，我盯着，有结果回来跟你聊。

最后想问问你：如果以后最强的 AI 模型都要先过一道政府审批才能用，你是觉得这是该有的刹车，还是挡了普通人路的多余关卡？你手里的活，离不离得开那个最顶配的模型？
