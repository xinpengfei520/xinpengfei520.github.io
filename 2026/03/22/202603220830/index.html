微信 ClawBot 来袭：微信终于正式支持原生 OpenClaw 了

发表于 2026-03-22 更新于 2026-04-16 分类于 AI 阅读次数： 本文字数： 3.3k 阅读时长 ≈ 3 分钟

你有没有想过，有一天你可以在微信里直接跟一个 AI Agent 对话，让它帮你查资料、写文档、处理日常琐事？

不用想了。2026 年 3 月 22 日，腾讯正式上线了微信 ClawBot 插件——微信原生支持 OpenClaw。

腾讯公关总监张军在社交媒体上官宣了这个更新。这意味着，全球用户量最大的即时通讯应用，终于拥抱了当下最火的开源 AI Agent 框架。

这件事比你想的要大得多。

OpenClaw 是什么？为什么这么火？

先说背景。

OpenClaw 是由 Peter Steinberger 创建的开源 AI Agent 框架。它的前身叫 Clawdbot，后来改名 Moltbot，最终定名 OpenClaw。截至 2026 年 3 月，它在 GitHub 上拿下了 24.7 万颗星，4.77 万个 Fork——这个数字放在整个开源史上都排得上号。

NVIDIA CEO 黄仁勋称它为”下一个 ChatGPT”。

OpenClaw 之所以火，是因为它解决了一个很实际的问题：让 AI 不只是聊天，而是能真正帮你干活。 它可以操控你的电脑、浏览网页、执行任务，甚至串联多个工具完成复杂工作流。

简单说，ChatGPT 是你的顾问，OpenClaw 是你的员工。

但 OpenClaw 一直有一个痛点——它主要运行在桌面端。你得打开电脑、打开终端、敲命令。对于大多数人来说，这个门槛太高了。

微信 ClawBot 的出现，把这个门槛直接砸平了。

腾讯 ClawBot 怎么用？

说起来非常简单，升级到微信的最新版本 8.0.70，进入 我 - 设置 - 插件，就能看到 ClawBot 的身影了。

第一步：安装。 一行终端命令搞定。

如果你使用的是腾讯云的 lighthouse 服务器，那么更简单，直接打开腾讯云服务器页面后会自动显示升级，升级完成后，点击授权，然后微信扫码即可开启。

第二步：授权。 微信扫码，授权 微信ClawBot 作为你的个人插件。

第三步：对话。 打开微信，找到 微信ClawBot，跟它说话。就像跟朋友聊天一样。

目前的系统要求：iOS 微信 8.0.70 及以上版本。Android 版本还在灰度中。

有几个重要的限制需要知道：

  • 只能作为个人助手使用，不能拉进群聊。 这是隐私和安全考虑。
  • ClawBot 只是消息通道。 它不会自动操作你的微信——不会帮你发朋友圈、不会自动回复消息、不会读取你的聊天记录。所有操作都需要你主动发起。
  • 目前是灰度发布。 不是所有用户都能立即使用。

腾讯的”龙虾战略”

ClawBot 不是腾讯在 AI Agent 领域的唯一动作。

了解内情的人都知道，腾讯内部有一个”龙虾特遣队”（Lobster Task Force），专门负责 AI Agent 的布局。为什么叫龙虾？因为 OpenClaw 的 logo 是一只龙虾钳子。

这个特遣队产出了三个核心产品：

产品 定位 特点
云端龙虾 (Lighthouse) 云端 AI Agent 基于腾讯云，企业级部署
自研龙虾 (WorkBuddy) 办公协作 Agent 集成企业微信和腾讯文档
本地龙虾 (QClaw) 桌面操控 Agent 基于 OpenClaw，通过微信/QQ 远程控制 PC

ClawBot 是这个战略拼图的关键一块——它把微信变成了 AI Agent 的入口。

QClaw 更有意思。它让你通过微信远程控制家里或办公室的电脑。出门在外，给 QClaw 发一条微信：”帮我把桌面上那个 Excel 里的数据整理一下，发到我邮箱。”它就会操控你的 PC 完成这个任务。

这不是遥远的未来。这是腾讯已经在做的事。

安全：房间里的大象

每次聊到 AI Agent，安全问题就是绕不过去的话题。

OpenClaw 在安全方面的记录并不完美。2026 年初，安全研究人员发现了一个编号为 CVE-2026-25253 的零点击漏洞——攻击者可以在不需要用户交互的情况下，利用 OpenClaw 执行恶意操作。全球超过 4 万个暴露的 OpenClaw 实例 受到影响。

中国政府也对此有所警觉。据报道，国家安全部门已经限制国家机关使用 OpenClaw，理由是安全风险。

腾讯显然意识到了这一点。ClawBot 的设计做了几个关键的安全隔离：

  1. 纯消息通道。 ClawBot 只负责传递指令和结果，不直接操控微信的任何功能。
  2. 不接入群聊。 避免在多人场景下的信息泄露风险。
  3. 用户主动触发。 所有操作都需要用户明确发起，没有自动执行。
  4. 灰度发布。 先小范围验证安全性，再逐步放开。

这些措施够不够？说实话，还需要时间验证。但至少说明腾讯不是闭着眼睛往前冲。

对普通开发者意味着什么

如果你是一个开发者，ClawBot 的上线意味着几件事。

第一，AI Agent 的分发渠道变了。 以前你做一个 AI Agent，用户得装桌面客户端。现在，微信就是入口。13 亿微信用户都是潜在的触达对象。这对独立开发者来说是巨大的机会。

第二，你可以基于 ClawBot 构建自己的工作流。 想象一下：你写一个脚本，每天早上让 ClawBot 帮你汇总邮件、整理日程、生成待办清单，然后通过微信推送给你。以前这需要写一套完整的应用，现在只需要一个 OpenClaw 配置文件。

第三，移动端 AI Agent 的体验跨过了及格线。 以前在手机上用 AI Agent，要么体验糟糕，要么功能受限。微信作为载体，天然解决了通知、交互、跨平台的问题。你不需要教用户怎么打开终端、怎么装依赖——他们只需要在微信里发一条消息。

第四，生态的想象空间打开了。 OpenClaw 本身支持 MCP（Model Context Protocol），可以接入各种第三方工具和 API。ClawBot 作为微信端的入口，意味着你可以通过微信触发整个 MCP 工具链——查数据库、调接口、生成报告、发邮件。这是一个从”聊天”到”办事”的质变。

不过也有需要冷静的地方。ClawBot 目前的能力边界很明确。它不是一个全能助手，它是一个消息通道。真正的计算和执行还是在后端完成的。不要期望它能替代你电脑上的完整 OpenClaw 体验。

而且，灰度阶段的稳定性和响应速度还有待观察。早期用户反馈显示，高峰期偶尔会出现消息延迟。这不是 ClawBot 本身的问题，而是后端算力调度的问题——毕竟同时服务大量用户的 AI Agent 请求，跟普通聊天消息的资源消耗完全不在一个量级。

这场 AI Agent 入口之争

微信入场 AI Agent，不是孤立事件。

看看全球的格局：Apple 在 iOS 18 里深度集成了 Siri + AI Agent 能力。Google 在 Android 上推出了 Project Astra。Meta 在 WhatsApp 和 Instagram 里嵌入了 Meta AI。

每一家巨头都在抢同一个位置：成为用户跟 AI Agent 交互的默认入口。

腾讯选择了一条务实的路——不自己从头造轮子，而是拥抱最火的开源方案。OpenClaw 有社区、有生态、有开发者。腾讯把它接进微信，相当于直接继承了这个生态。

这个策略聪不聪明？我觉得非常聪明。

但也有风险。OpenClaw 是开源项目，腾讯对它的掌控力有限。如果未来 OpenClaw 的发展方向跟腾讯的利益冲突，事情会变得复杂。而且，开源社区对大厂”拿来主义”一直有微妙的情绪——腾讯能不能成为一个好的开源贡献者，而不只是使用者，也会影响这个合作的长期走向。

从商业角度看，腾讯的算盘也很清楚：微信是入口，云是后端，AI Agent 是中间层。 ClawBot 吸引用户使用 AI Agent，AI Agent 的计算需求拉动腾讯云的消费。这是一个非常完整的商业闭环。

写在最后

2026 年 3 月 22 日，可能是一个值得记住的日期。

微信正式支持 OpenClaw，意味着 AI Agent 从极客的终端工具，变成了 13 亿人手机里的聊天对象。

这个变化的速度，比大多数人预期的要快。半年前，OpenClaw 还是一个需要在终端里敲命令的开发者工具。现在，你妈可能都会用它了——只要她会在微信里打字。

当然，ClawBot 还很早期。灰度发布、功能限制、安全隐忧——这些问题都真实存在。但方向已经很清楚了：AI Agent 正在从”专业工具”变成”基础设施”。
