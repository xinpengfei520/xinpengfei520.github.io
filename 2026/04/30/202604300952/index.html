Anthropic 把 Claude Code 日均成本估算从 $6 改成 $13，官方说"这不算涨价"

发表于 2026-04-30 更新于 2026-07-02 分类于 AI 阅读次数： 本文字数： 4k 阅读时长 ≈ 4 分钟
解读 Anthropic 4 月 15 日悄改文档把 Claude Code 日均成本估算从 6 美元提到 13 美元（涨 117%）、90% 用户天花板从 12 涨到 30 美元的事件，结合 2% 测试和 Head of Growth 坦言订阅计划撑不住使用强度，判断订阅制正被 agentic 工作流打穿、调价只是时间问题。

大家好，我是飞飞。

昨天写完那篇省 token 的文章之后，我以为这周关于 Claude 的内容就告一段落了。

今早起床第一件事是被群里截图刷屏。Business Insider 昨天下午的一条报道：Anthropic 在 4 月 15 日悄悄把官方文档里”每个开发者每个活跃日的平均花费”从 $6 改成了 $13。

这件事的发现者叫 Ed Zitron，他是 EZPR 的 CEO，前 Business Insider 专栏作者。他在 X 上的原话是”Anthropic stealth-tweaked their Claude Code docs”。stealth 这个词他用得很准：没有公告，没有 changelog 条目，只是把那一页文档默默改了。

Anthropic 的官方回复也来得很快。发言人对 Business Insider 说：”这不是定价变化，也不是产品变化。Opus 4.7 现在是 Claude Code 的主力模型，我们更新这些数字只是反映从 2025 年 2 月那次更新到现在用户使用习惯的变化。”

我读完一上午的报道之后想写这篇。这件事单看是一个数字调整的小新闻，但放进过去十天 Anthropic 在做的一系列动作里看，它和上周的”2% 测试”、和 Amol Avasare 那句被很多人忽略的话，连起来才是完整的故事。

改了什么具体数字

Claude Code 的官方文档里有一页叫 Costs（code.claude.com/docs/en/costs），讲企业团队规划部署预算时怎么估算开销。

这一页的旧版（4 月 15 日之前）写的是：

  • 平均每个开发者每个活跃日：**$6**
  • 90% 的用户活跃日花费在 $12 以内

新版（4 月 16 日之后）写的是：

  • 平均每个开发者每个活跃日：**$13**
  • 90% 的用户活跃日花费在 $30 以内
  • 企业部署里每个开发者每月 $150 到 $250

平均成本从 $6 涨到 $13，是 117% 的涨幅。90% 用户的天花板从 $12 涨到 $30，是 150% 的涨幅。

这个文档页 Anthropic 自己确认上一次大改是 2025 年 2 月，整整 14 个月没动过。这一次不发公告就把”参考预算”翻倍多。从企业 IT 视角看，这相当于年度预算审批要重新做一遍。

Anthropic 的解释：因为默认模型从 Sonnet 3.7 换到了 Opus 4.7

发言人给 Business Insider 的解释里有一句关键：”Opus 4.7 is now the frontier model in Claude Code.”

这句话翻译过来是：旧版的 $6 估算，是基于当时 Claude Code 主推 Sonnet 3.7 模型计算出来的。新版的 $13 估算，反映的是用户现在主要用 Opus 4.7。

我自己跑过这两个模型，从单价层面对比，Sonnet 3.7（已弃用）当时是 $3/M 输入、$15/M 输出，Opus 4.7 是 $5/M 输入、$25/M 输出。Opus 比 Sonnet 大概贵 67% 左右。

如果只是模型默认换了，单纯的数学是 $6 × 1.67 ≈ $10。

但官方实际改成了 $13。多出来的 $3 缺口，按 Anthropic 的解释是”使用强度也在涨”。这个说法和 Amol Avasare 那句”engagement per subscriber is way up”是配套的。

所以从 Anthropic 的逻辑来说：单 token 价没动，模型选项没动，”涨价”这个词技术上他们说得过去。但是开发者每天实际烧的钱确实翻倍了。这个就是这篇报道之所以会引发反弹的核心。

上周的”2% 测试”现在连起来看

上周（4 月 21 日左右）我写过一篇关于 Copilot 停注册 + Claude Code 从 Pro 套餐里被拿掉的文章。那时候 Pro 用户登录定价页发现 Claude Code 没有了，社区炸了，Anthropic 当天回应说是”对 2% 新用户的 A/B 测试”。

当时我自己的判断是：这个”2% 测试”里的”测试”两个字是给媒体听的。它更像在为后面的真改动做预演。

今天这条新闻让那个判断更接近确认。

理由很简单：4 月 16 日改 cost 文档（公开但低调），4 月 21 日做”2% 测试”（更激进的产品改动），4 月 28 日 Avasare 在 X 上明说”现有订阅计划不适应目前的使用强度”，4 月 29 日 Business Insider 把 cost 文档变化爆了出来。

这十几天的节奏不像偶然。它更像一家公司在用市场可以接受的速率做价格预期管理：先调”参考预算”，再做小范围用户测试，然后等舆论消化，最后才动真正的订阅价。

如果 Pro $20 这一档将来从产品页彻底拿掉 Claude Code，或者订阅价直接调高到 $40、$60 量级，那今天看似”只是一个文档数字调整”的动作就是序章。

Amol Avasare 那句话才是真正的重点

这一节我想单独拎出来。

Amol Avasare 是 Anthropic 的 Head of Growth。他上周在 X 上发的那条原文是：

Engagement per subscriber is way up. We’ve made small adjustments along the way (weekly caps, tighter limits at peak), but usage has changed a lot and our current plans weren’t built for this.

中文：每个订阅者的使用强度大幅上涨。我们一直在做小调整（周限额、高峰期收紧），但使用习惯变化太大，我们当前的方案不是为这种使用设计的。

这句话被很多报道当作背景一笔带过。但它其实是这一系列动作里最坦诚的一句。它没有说”我们要涨价”，但它说的是”现在的订阅模型已经支撑不下去了”。

订阅制（不管是 Pro $20 还是 Max 5x $100/月）本质上是一个”打包定价”的赌博：Anthropic 赌大部分订阅者每月烧的 token 价值低于订阅费。这个赌博在 Sonnet 3.5、Sonnet 3.7 那个时代成立，那时候单次会话烧个几千 token 是常态，agentic 工作流还没普及。

到了 Opus 4.7 + 1M context + agent 全员日跑这个时代，事情变了。我自己上周写的那篇文章里有数据：我一周烧 28M token，按 API 裸价是 $106。我开 Max 5x $100/月，已经烧穿了订阅费的 100%。Disney 内部数据更夸张，Business Insider 之前另一篇报道里写过，他们有员工一天调 Claude 51000 次。

订阅制本来就是要让你”用得多就赚到、用得少就被赚”。但当一线开发者人均用得多，那 Anthropic 自己就被赚了。这是 Avasare 想说但不能说得太直白的事。

对 Claude Max 5x 这一档我目前的判断

我自己是 Claude Max 5x $100/月。今天读完这条新闻，我做了几件事。

最先做的是把昨天那篇省 token 的文章里的六个手法挨个再 review 一遍，确认 prompt caching、/compact 提前、sub-agent 隔离这三条最大的杠杆都在我的工作流里跑着。这件事在今天比昨天更重要，因为如果 Anthropic 的下一步是真的调订阅价，我用 Max 5x 多撑两个月的能力会直接转化为省下来的钱。

接着我把上周写的 Claude Desktop 开发者模式那篇文章里的 DeepSeek V4 接入路径再走了一遍。这是我的备份方案。如果 Max 5x 涨到 $200，我会立刻切走。

最后我把这周的几个备用模型再过一遍：DeepSeek V4（$1.74/$3.48）、MiMo-V2.5-Pro（¥7/¥21 分段）、GLM-5.1。同样的 28M token，如果走 DeepSeek V4 大概是 $50 上下，比 Anthropic 便宜一半。这个对比关系如果 Anthropic 调价，会瞬间被市场重新评估。

把这三件事拼起来看，我对 Anthropic 接下来三个月动作的判断是：要么砍掉 Pro 一档的 Claude Code、要么涨价、要么在 Max 5x 上加更严的”使用强度”限制（比如把现在的”5 小时窗口”换成”周用量上限”）。三选一。我个人倾向认为是后两者的组合。

这件事和”AI 使用成本上升”那个大叙事的关系

最后一节我想拉远一点看。

媒体报道里都把这件事归到”AI 使用成本全面上升”这个大趋势。这个表述不算错，但容易让人误以为是”AI 模型本身变贵了”。

实际上 Anthropic 这次没有动 token 单价。Opus 4.7 还是 $5/$25，Sonnet 4.6 还是 $3/$15。变贵的不是单价。

变贵的是”使用强度”。一个 agentic 工作流跑一次的 token 消耗，是一年前手动打字的 50 倍到 100 倍。当 50 倍这个倍率被加到原来按”低强度”打包定价的订阅模型上，订阅模型就崩了。

这就是为什么我这周连着写了七篇关于 AI 编程经济学的文章（Copilot 停 Pro 注册、Claude Code 从 Pro 里拿走、GPT-5.5 涨价、DeepSeek V4 涨价 8 倍、MiMo Token Plan 重构、Claude Desktop 开发者模式、token 省钱手法）。它们看上去是七件独立的事，但底下是同一条主线：订阅制这一代被 agent 打穿了，整个市场在重新定价”算力 + 模型能力”这个组合。

今天 Anthropic 把 cost 文档从 $6 改成 $13，是这条主线上的又一个观察点。下个观察点很可能在五月份。
