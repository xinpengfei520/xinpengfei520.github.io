躺在沙发上写代码：Claude Code Remote Control 如何让你用手机操控 AI 编程

发表于 2026-03-20 更新于 2026-04-10 分类于 阅读 阅读次数： 本文字数： 3.8k 阅读时长 ≈ 3 分钟

昨天晚上 11 点，我在终端里启动了一个大规模重构任务——把项目里 30 多个文件从旧的认证库迁移到新方案。

Claude Code 开始工作了。我盯着终端看了两分钟，突然觉得——我为什么要坐在这里等？

于是我输入了一条命令：/rc。

终端上弹出一个二维码。我用手机扫了一下，然后端着杯茶走到沙发上躺下。

手机屏幕上，Claude 正在一个一个文件地改。改完一批，问我要不要继续。我点了”批准”，翻了一页书。

30 分钟后，30 个文件迁移完成。我全程躺在沙发上。

这就是 Claude Code Remote Control——Anthropic 在 2026 年 2 月 24 日发布的新功能。一条命令，把你的手机变成 AI 编程的遥控器。

Remote Control 是什么：一句话解释

Remote Control 让你在终端启动 Claude Code 会话，然后用手机、平板或任何浏览器继续控制它。

注意关键词：继续控制。

你的代码不会被上传到云端。你的文件不会离开你的电脑。你的 MCP 服务器、环境变量、项目配置——全部留在本地。

手机只是一个”窗口”。你透过这个窗口，看到本地终端里正在发生的一切，然后发出指令。

Anthropic 的产品经理 Noah Zweben 说得很直白：**”目标是让你散步的时候也能继续工作流。”**

这不是把开发环境搬到云上。这是给你的本地环境装了一个遥控器。

为什么需要 Remote Control

在 Remote Control 出现之前，Claude Code 有一个让人抓狂的痛点：你必须坐在电脑前。

你让 Claude 做一个复杂任务——比如写 20 个测试用例，或者做一次跨文件的重构。这个过程可能要 10 分钟、20 分钟，甚至更久。

在这期间，Claude 会不断问你问题：这个文件要不要改？这个测试策略你同意吗？遇到冲突怎么处理？

你必须坐在那里，盯着终端，等着回复。不能离开。不能去开会。不能去倒杯水。

你被”绑”在了工位上。

一些极客想了各种办法绕过这个限制：

  • 用 SSH 从手机连回电脑
  • 用 tmux 保持会话，然后从另一台设备 attach
  • 用 ngrok 做端口转发
  • 甚至有人自己写 WebSocket 代理

一位开发者在 Reddit 上说：”我花了 6 个小时搭建 GKE 上的 WebSocket 代理、Firebase JWT 认证，让手机能连到 Claude Agent 虚拟机。然后 Anthropic 发布了 Remote Control。”

官方用一条命令解决了社区折腾了几个月的问题。

怎么用：2 分钟上手

前提条件

  • Claude Pro 或 Max 订阅（API key 不行）
  • Claude Code 版本 2.1.52 或更高
  • 手机装了 Claude App（iOS 或 Android），或者有浏览器

启动方式 1：新建远程会话

终端会显示一个会话 URL 和一个二维码。按空格键可以切换二维码的显示。

启动方式 2：把现有会话转为远程

如果你已经在一个 Claude Code 会话里聊了很久，不想丢失上下文：

或者简写：

当前的对话历史、文件状态、所有上下文——全部保留。只是多了一个远程连接入口。

连接方式

三种方式，选一种：

  1. 直接打开 URL：在任何浏览器访问 claude.ai/code
  2. 扫二维码：用 Claude App 扫终端上的二维码
  3. 在 App 里找会话：打开 Claude App 的 Code 标签页，远程会话会显示一个带绿色圆点的电脑图标

连上之后，手机和电脑上的对话实时同步。你可以从终端发消息，也可以从手机发消息，无缝切换。

始终开启

如果你每次都想用 Remote Control，不想手动开：

在 Claude Code 里输入 /config，把 Enable Remote Control for all sessions 设为 true。

以后每个会话自动带远程控制。

安全架构：代码到底安不安全

这是大家最关心的问题。我的代码会不会被传到 Anthropic 的服务器上？

答案是：不会。

Remote Control 的安全设计非常简洁：

组件 发生了什么 运行在哪里
代码和文件 留在你的机器上 本地
MCP 服务器 留在你的机器上 本地
环境变量 留在你的机器上 本地
聊天消息 通过加密通道传输 加密隧道
工具执行结果 通过加密通道传输 加密隧道
AI 模型推理 Anthropic API Anthropic 云端

关键的技术细节：

1. 仅出站连接。 你的电脑不会打开任何入站端口。没有端口暴露，没有防火墙配置。你的机器只是向 Anthropic API 发出 HTTPS 请求，然后轮询等待指令。

2. TLS 加密。 所有流量都走 TLS 加密通道——和你正常使用 Claude Code 时一样的传输安全级别。

3. 短期凭证。 使用多个短期凭证，每个凭证只用于单一目的，独立过期。会话结束后，远程链接立即失效。

4. 自动重连。 笔记本休眠？网络断了？没关系。机器恢复后自动重连，不丢失上下文和进度。

对于金融、医疗、政府等合规要求严格的行业来说，”代码永远不离开你的机器“这个架构，让合规对话简单了很多。

5 个真实使用场景

场景 1：走路开站会

早上开会前，你启动三个 Claude Code 会话：一个修 CI 管道，一个写集成测试，一个迁移旧 API。

开会的时候，掏出手机看看每个任务的进展。Claude 问了一个关于测试用例的问题，你在手机上回复，不打断会议节奏。

站会结束，三个任务都推进了 30 分钟。

场景 2：通勤路上修 Bug

下午 6 点，构建挂了。你启动 Claude Code 开始排查，但要赶火车了。

激活 /rc，扫码，上车。在地铁上继续引导 Claude 调试。到家的时候，修复方案已经准备好了。

场景 3：沙发上审代码

同事推了一个 PR，问你能不能看一下。你不想再开电脑了。

从手机上启动 Claude Code 分析这个 PR，审查变更，检查潜在问题——全程坐在沙发上完成。

场景 4：多 Agent 调度

Anthropic 的 Claude Code 负责人 Boris Cherny 说，他同时运行 5 个以上的 Claude Code Agent，每月产出 300 多个 PR。

有了 Remote Control，这种多 Agent 编排不需要坐在电脑前了。每个会话独立运行，你从手机上像调度员一样管理所有任务。

场景 5：凌晨 2 点的生产事故

以前：PagerDuty 响了 → 开电脑 → 连 VPN → 找代码 → 开始排查。

现在：PagerDuty 响了 → 打开手机上的 Claude Code → 开始排查。

从接到告警到开始响应，时间从 10 分钟缩短到 10 秒。

Remote Control vs OpenClaw：选哪个

如果你关注过 OpenClaw（GitHub 19.9 万星），你会发现它也能从手机控制电脑。两者有什么区别？

对比维度 Remote Control OpenClaw
启动方式 一条命令 自托管，需要端口转发或隧道配置
安全性 仅出站 HTTPS，无开放端口 WebSocket，曾出现 RCE 漏洞
平台支持 claude.ai/code、iOS、Android WhatsApp、Telegram、Slack 等 15+ 平台
定位 专注编程工作流 通用生活助手
费用 包含在 Pro/Max 订阅中 免费（自托管），但需自备 API key
断线重连 自动重连 手动重启

一句话总结：Remote Control 是专业开发者的遥控器。OpenClaw 是万能的生活管家。不同的工具，解决不同的问题。

如果你只是想在手机上继续编程会话，Remote Control 更干净、更安全、更省心。

当前的限制：研究预览阶段

Remote Control 目前还是研究预览（Research Preview），有一些已知限制：

  • 一次一个会话：每个 Claude Code 实例只支持一个远程连接
  • 终端必须保持打开：关了终端或停了 Claude 进程，会话就结束了
  • 10 分钟网络超时：机器无法联网超过 10 分钟，会话会超时
  • 权限审批仍需手动：--dangerously-skip-permissions 在远程控制下暂时不生效
  • 仅限个人订阅：Pro 和 Max 计划可用，Team 和 Enterprise 计划暂不支持

Simon Willison 在测试中还发现：会话崩溃时，错误提示不够清晰；重启进程后需要新建会话，不能重连旧会话。

这些问题大概率会在后续版本中修复。毕竟这才是第一个公开版本。

写在最后

Remote Control 解决的核心问题，用一句话概括就是：开发者不应该被绑在工位上。

AI Agent 越来越强大，能独立工作的时间越来越长。但每次它需要你确认、需要你做决策的时候，你必须坐在电脑前。这成了整个工作流里最大的瓶颈。

Remote Control 打通了这个瓶颈。你的角色从”写代码的人”变成了”指挥 AI 的人”。而指挥，不需要坐在固定位置。

DevOps.com 的评价很到位：**”在 AI 编程工具纷纷奔向云端的时候，Anthropic 反其道而行——代码留在本地，控制带在身上。”**

这个功能现在还很新，还有粗糙的地方。但方向是清楚的：未来的开发者，不是坐在电脑前写代码的人，而是随时随地指挥 AI Agent 的人。
