Claude Opus 4.7 还没发布，Figma 先跌了 6%

发表于 2026-04-15 更新于 2026-04-21 分类于 AI 阅读次数： 本文字数： 2.7k 阅读时长 ≈ 2 分钟

哈喽，我是飞飞。

昨天 The Information 发了一篇独家报道，说 Anthropic 正在准备发布 Claude Opus 4.7，可能最快本周推出，同时还有一个 AI 设计工具。

产品没人见过。截图没有。Demo 没有。连名字都不确定。

然后 Figma 当天跌了 6%。Wix 跌了 4.7%。Adobe 跌了 2.7%。GoDaddy 跌了 3%。

一份单一来源的报道，一个还没发布的产品，让设计软件公司的市值蒸发了几十亿美元。

2026 年的科技股，已经不需要产品来杀了。一条消息就够。

Opus 4.7 到底升级了什么

先说模型本身。

Opus 4.6 是 2 月 5 日发布的，到现在两个多月。按 Anthropic 今年的节奏——1 月以来每两周一次重大更新——4.7 现在出来一点都不意外。

根据目前已知的信息，Opus 4.7 的主要提升在三个方向：

多步推理更强了。以前的模型在解决复杂问题时，链条一长就容易断。4.7 在长链推理上的连贯性明显提升，处理需要十几步才能解决的 debug 或者多文件重构时更稳。

长时任务执行。4.6 已经有 100 万 token 的上下文窗口，4.7 保留了这个窗口但用得更好——在超长文档和整个代码库里做多针召回测试时，深处的信息找得更准。

多 Agent 协调。这是新加的能力。4.7 可以把任务分给多个 Agent，合并它们的输出，在长时间自主工作流中保持统一的计划。配合刚发布的 Managed Agents 和 Routines，这个能力才有了落地的场景。

需要说清楚的是：Opus 4.7 不是 Anthropic 最强的模型。上周发布的 Claude Mythos 才是，但 Anthropic 因为安全考虑没有对外开放，只给了 11 家机构用来做网络安全研究。Opus 4.7 是面向商业用户的旗舰模型。

真正的重头戏：Routines

昨天 Anthropic 还发布了另一个东西，新闻热度被 Opus 4.7 完全盖住了，但我认为这才是本周最重要的发布。

Claude Code Routines。

一句话解释：你配置一次——写好 prompt、指定代码仓库、连上工具——然后 Claude 按定时、API 调用或者 GitHub 事件自动执行，跑在 Anthropic 的云上，你的电脑关了也不影响。

以前我想让 Claude 每天凌晨自动帮我检查代码仓库的 issue，我得自己搞一台服务器、写 cron job、管脚本、处理日志。现在配一个 Routine 就完事了。

Anthropic 官方给了几个用例：

每晚 2 点自动从 Linear 拉最高优先级的 bug，尝试修复，开一个 draft PR，第二天早上起来直接 review。

GitHub 上有人开了 PR，Routine 自动按团队的 checklist 做代码审查。

Python SDK 改了一行代码，Routine 自动生成对应的 Go SDK 变更 PR。

社区的反应很直接。有人说这东西”杀掉了你一半的 Zapier 账户”。有人说这是”AI 员工”。The Register 给了一个冷幽默的标题：”Claude Code routines promise mildly clever cron jobs”——温和聪明的 cron job。

我觉得 The Register 说反了。Cron job 是你写好脚本让它按时跑。Routine 是你告诉 Claude 你想干什么，Claude 自己决定怎么做。这中间差了一整个大模型的推理能力。

目前 Routine 是研究预览版，有使用限制。Pro 用户每天 5 次，Max 用户 15 次，Team 和 Enterprise 用户 25 次。

对我来说，15 次/天已经够用了。我现在脑子里已经在想怎么把我的博客写作流水线里的一些环节改成 Routine——比如每天定时搜索 AI 行业新闻、整理成素材包存到 Obsidian。

那个让股票跌的设计工具

回到让 Figma 跌 6% 的那个 AI 设计工具。

根据 The Information 的报道，这个工具可以用自然语言创建网站、演示文稿、落地页和产品原型。面向技术和非技术用户。

具体什么样，没人知道。但市场已经给出了自己的判断。

这个判断的背景是：2026 年 S&P 500 软件和服务指数已经跌了近 26%。Figma 年内跌了 30%。上个月 Claude Cowork 发布时，全球 SaaS 股票就已经集体跳水过一次了。

现在市场的逻辑是：Anthropic 每发布一个新产品，就会有一批软件公司的股票跟着跌。不管产品实际上做不做得到。

有个分析师的评论我觉得很到位：Figma 面临的威胁不是 Anthropic 今天抢走它的用户，而是这条消息发出的信号——如果模型厂商开始在自己的模型上做垂直工具，两年来一直困扰创业公司的”AI wrapper”质疑，就要轮到大公司身上了。

Anthropic 已经做了几件事铺路：和 Figma 合作把 AI 生成的代码转成可编辑设计文件，把 Claude 集成进了 Microsoft Word 和 PowerPoint。设计工具是这条线的自然延伸。

AI 史上最忙的一周

这周不只 Anthropic 一家在发布东西。

OpenAI 据说要发 GPT 5.4 Cyber。Meta 在办 LlamaCon。加上 Anthropic 的 Opus 4.7 + 设计工具 + Routines。

有人说这是 AI 史上最忙的一周。我不知道是不是，但今年这种密度已经让人麻木了——1 月到现在，每两周就有一个大新闻，每个月都有人说”这是 AI 最重要的一周”。

对我个人来说，Opus 4.7 的意义要等实际用了才知道。4.6 到现在我用了两个多月，体感上已经很强了，特别是在 Claude Code 里跑复杂任务的时候。如果 4.7 的多 Agent 协调真的能让我同时分派几个子任务并行跑，那确实会改变我的工作方式。

但 Routines 的影响我已经能预见了。

我现在每天手动跑一遍”搜索资料→整理素材→写文章→检测 AI 味→配图→生成多平台版本”的流水线。如果这条线的前两步——搜索和整理——能变成一个每天凌晨自动跑的 Routine，我起床就能看到整理好的素材包等着我，直接进入写作环节。

Claude 从”我找它”变成了”它等我”。这个转变挺大的。

不过我也有一个担忧：Routine 每天 15 次的限制够不够用？如果我配了 5 个 Routine（新闻监控、代码审查、issue 追踪、RSS 更新、写作素材），每个每天跑 3 次就到上限了。Max 用户 200 美元/月拿到 15 次/天，算下来每次 Routine 成本大约 0.44 美元。比自己租服务器跑脚本便宜，但比 cron job 贵无数倍。

这也是 Anthropic 一直在做的事：用便利性换取付费意愿。你可以自己搭，但 Routine 帮你省掉了服务器、脚本、日志、错误处理。你买的是省心。

你最期待 Opus 4.7 的哪个能力？还是你更感兴趣 Routines？评论区说说你打算怎么用。
