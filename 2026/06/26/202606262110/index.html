超级个体第 8 课：用 AI 给自己上线一张互联网名片，从页面到收录我替你把坑都标了

发表于 2026-06-26 更新于 2026-07-01 分类于 教程 阅读次数： 本文字数： 2.8k 阅读时长 ≈ 3 分钟
这是超级个体第 8 课，给小白一条照着就能走完的建站链路：用 Bolt 一句话生成个人主页、Cloudflare 成本价买域名、Cloudflare Pages 或 GitHub Pages 免费部署绑域名、再去搜索引擎提交 sitemap 催收录。我用运营 vance.xin 的真实经验讲，包括 PAT 过期让部署静默挂一整天、GEO 改造后一周 4713 次 AI 抓取只换回 10 次回流（回流率 0.21%）这种只有亲手跑过才知道的事。

哈喽，我是飞飞。

上一课我们聊的是怎么让 AI 把代码写出来。这一课往前再走一步：代码也好、页面也好，做出来之后，怎么让它真上线，让别人在浏览器里输个网址就能找到你，让搜索引擎和 AI 也认得你这个人。

说白了，就是给自己上线一张互联网名片。

这件事我自己整条链路都趟过。我这个博客 vance.xin 就是一个静态站，从写完到上线全是自动跑的，后面还专门给它做过一轮给 AI 看的改造。所以这一课我不打算照着通用教程念一遍流程，我就把我踩过的坑、磨过的时间、翻过的车，挨着给你标出来。这套打法我也还在迭代，不一定每一步都最优，但能保证你照着走能上线。

页面这一步，现在真不用你自己写一行代码

放在两年前，做个人主页这步能劝退一大半人：你得会 HTML、会 CSS，还得调半天才不丑。现在这一关基本被 AI 抹平了。

你打开 Bolt.new，在对话框里用大白话描述一句，比如「帮我做一个个人主页，左边放头像和一句简介，右边放我的几个作品链接，配色用深蓝，中文」，它就能直接把整个网页的代码生成出来，还在浏览器里实时给你看效果。Bolt 是 a16z 投的那家 StackBlitz 做的，底层用了一个叫 WebContainers 的东西，等于把跑代码的环境塞进了浏览器，你电脑上什么都不用装。

跟它一类的还有两个你会反复听到的名字。v0 是 Vercel 出的，更擅长出好看的界面，你要的就是一张漂亮的静态名片，它很对路。Lovable 偏向把一整个能用的应用做出来。我给你一个最省事的分法：你只想要一个能直接上线的个人主页，用 Bolt；你特别在意视觉、想多调几版样子，用 v0。

这一步的产出，要么是一个它帮你部署好的临时链接，要么是一份它生成的网页代码。临时链接能用，但它是平台给的、长得乱、随时可能变，撑不起「名片」两个字。所以真正把它变成你的东西，得靠后面两步。

域名是你这张名片上印的那串地址，别在这步多花冤枉钱

域名就是别人在浏览器里敲的那个网址，vance.xin 就是我的。这是整张名片上唯一一个长期属于你、能跟着你走很多年的东西，值得认真挑一个。

挑的时候盯两点就行，一个是简短好记，另一个是后缀。.com 最通用，.xin、.me、.dev 这些也都能用，看哪个顺眼、哪个还没被注册。

真正想提醒你的是别花冤枉钱。同一个域名，在不同地方买，价格和套路差很多。老牌的 GoDaddy（就是大家叫的狗爹）一个 .com 首年看着才几美元，但你想要隐私保护得另外掏将近一百块一年，续费的时候价格还往上跳。我现在更推荐 Cloudflare 自己的域名注册，它是按成本价卖的，几乎不在注册局价格上加钱，续费不涨价，隐私保护还免费送。要用支付宝付的话，NameSilo、Porkbun 这几家也便宜，支持支付宝。

有一个小坑你记一下：Cloudflare 免费版不让你自己改 Nameserver，少数像 .cn 这种国家后缀它也不一定能注册。你要是没这些特殊需求，它就是性价比最高的那个。

部署这步真免费，几分钟就能让全世界打开你的网站

页面有了、域名买了，剩下的就是把页面放到一个能被全世界访问的地方，再把你的域名指过去。这就叫部署。听着唬人，其实现在小白也能五分钟搞定，而且不花钱。

最适合上手的是 Cloudflare Pages。你把前面生成好的网页文件传上去，它立刻给你分配一个免费的默认网址，这时候你的站其实已经能打开了。接着在项目设置里找到自定义域名，把你买的那个域名填进去、点激活，它会自动去做解析校验，等个几分钟，输你自己的网址就能访问了。整套下来一分钱不花，域名也就几块到几十块一年。

如果你像我一样用 GitHub 管代码，GitHub Pages 也是同一路子，原生支持把 main 分支自动构建上线，HTTPS 证书它帮你自动签、自动续。我的博客主站走的是另一条 Docker 流水线，镜像站就挂在 GitHub Pages 上，平时我只管往 main 推，剩下的 hexo 构建加部署全自动跑。

但我要泼一句冷水，全自动不等于你能彻底撒手。前阵子我就栽了个跟头：负责把镜像站推到 GitHub Pages 的那个授权令牌悄悄过期了，结果部署连着失败了一整天。最坑的是，构建那一步明明是绿的，只有最后推送那一下认证红了，整条流水线看上去还在转，没人会盯着它看，我是隔了一天自己去翻状态才发现镜像站已经挂了一天没人知道。这件事给我提了个醒：自动化最贵的成本不是算力，是没人盯的时候，失败会悄无声息地堆着。你哪怕做得再自动，也得给「失败」配一个你能第一时间看见的提醒。

上线只是开始，让搜索引擎和 AI 找得到你才是真功夫

网站能打开了，是不是就算有名片了？还差一口气。这时候你把网址发给朋友他能看到，但你在百度、Google 里搜自己的名字，大概率搜不到。因为搜索引擎还不知道你这个站存在。让它知道、并且把你收进去，这件事叫 SEO，给小白翻译一下就是「让搜索引擎找得到你」。

入门动作其实就几下。去 Google 的 Search Console、百度的搜索资源平台，各自验证一下这个网站是你的（按它给的步骤传个文件或贴段代码就行），然后把你网站的 sitemap.xml 交上去。sitemap 就是一份你站点所有页面的清单，等于主动把家里的房间图递给搜索引擎，省得它自己一间间摸。交完之后，对你最在意的那几个页面，再用它的「请求收录」按一下，催它快点来抓。Bing 一般一天内就收了，Google 提交过比干等着快不少。

想看自己到底被收录了没，最土的办法最好用：在搜索框里输 site: 加你的域名，能搜出来就说明进库了。

这里有个新手最容易自己埋的雷：很多建站工具里有个「不让搜索引擎索引本站」的开关，建站期间为了不被提前抓到，常常是开着的。你上线之后一定记得回去把它关掉，不然你前面所有功夫白费，搜索引擎被你亲手挡在门外。

多说一句给 AI 看的那层，和这两年才有用的真账

传统 SEO 是拼在搜索结果里排第几，但这两年多了一层新玩法，叫 GEO，目标是让 ChatGPT、Kimi、豆包这些 AI 在回答问题时把你的内容引用进去。我去年专门给 vance.xin 做了一轮，放开了 AI 爬虫的准入、加了一份专门写给大模型看的站点说明、给每篇文章配了机器能直接读的结构化数据。

我把真实的账摆给你看，省得你对这件事有不切实际的期待。改造之后那一周，各家 AI 爬虫一共来抓了我四千七百多次，里面 GPTBot 自己就占了快两千次。但真正从 AI 答案里点回我博客的，整整一周只有十次，算下来回流率是 0.21%。

这个数字是不是低得有点扎心。但实话讲，它恰恰是最真实的一手经验：被 AI 收录，跟被 AI 推荐给真人，中间隔着很长一段路。爬虫愿意来抓只是第一步，回流是个慢慢长起来的滞后量。所以你别一上线就盯着流量，先把链路跑通，让它爬得动、读得懂，剩下的交给时间。

绕回开头那张名片。把它做出来其实只是开头那一句话的事，难的是上线、是被找到、是熬过没人来的那段。你要是真打算给自己立这么一张名片，你最先想放上去的，是你的作品、你的简历，还是干脆一句你想让别人记住的话？评论区告诉我。
