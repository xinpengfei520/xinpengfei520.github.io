别睡了，起来跑代码！Claude 错峰双倍活动实战：3 天零成本撸完一个项目

发表于 2026-03-18 分类于 阅读 阅读次数： 阅读次数： Valine：

你是不是一直有个想法——做个小工具、搞个 Side Project、重构一下老项目——但一直没动手？

理由无非两个：没时间，没额度。

现在两个问题同时解决了。Anthropic 搞了个”Spring Break”活动，3 月 13 日到 27 日，非高峰时段 Claude 全线产品额度翻倍，包括 Claude Code。不需要任何操作，自动生效，Free 用户也能参加。

活动还剩不到 10 天。别等了，今天就开始。

这篇文章不讲理论，直接给你一个可执行的行动计划：怎么在活动结束前，用 Claude Code 零成本（或接近零成本）跑完一个微型项目。

先搞清楚规则：什么时候翻倍？

翻倍不是全天候的，只在非高峰时段生效。

时区 高峰时段（正常额度） 非高峰时段（双倍额度）
美东 ET 8:00 AM - 2:00 PM 2:00 PM - 8:00 AM
美西 PT 5:00 AM - 11:00 AM 11:00 AM - 5:00 AM
北京时间 晚 8:00 - 凌晨 2:00 凌晨 2:00 - 晚 8:00

重点来了： 北京时间早上 2 点到晚上 8 点，整整 18 个小时都是非高峰。你白天正常上班的时候，Claude Code 就是双倍状态。

周末更爽： 全天 24 小时都是非高峰，双倍额度不间断。

还有一个关键细节：翻倍部分的用量不计入你的周限额。 这意味着就算你在非高峰时段把双倍额度用完了，高峰时段的正常额度丝毫不受影响。

简单说就是：白嫖，真的白嫖。

哪些人该抓住这个机会？

免费用户

你可能之前试过 Claude，发几条消息就被限流了。这两周不一样。非高峰时段额度翻倍，你能做的事情比平时多得多。这是体验 Claude 真正实力的最好窗口。

Pro 用户（$20/月）

Pro 的额度一直是个痛点。写代码写到一半，提示”你已达到 5 小时限额”。翻倍后，一个完整的功能开发从”做一半等限额重置”变成”一口气写完”。

而且，新用户现在注册 Pro 有 30 天免费试用。活动期间注册，等于白嫖双倍 Pro 额度。

Max 用户（$100-200/月）

你本来就有 5 倍或 20 倍的额度，翻倍后变成 10 倍或 40 倍。这个量级足够你跑一个中型项目的完整重构了。

3 天跑完一个项目：行动计划

说了这么多，到底做什么项目？下面给你 3 个难度递增的选项，每个都能在活动窗口期内完成。

方案一：个人工具（1 天完成）

项目：AI 每日摘要生成器

一个脚本，每天自动抓取你关注的信息源（RSS、Hacker News、Twitter List），用 AI 总结成一段简短的每日摘要，发到你的邮箱或 Telegram。

用 Claude Code 的步骤：

Claude Code 会帮你创建项目结构、写代码、配置依赖。你只需要提供 Telegram Bot Token 和 AI API Key。

预计用量： 大约 30-50 条 Claude Code 消息。Pro 用户非高峰时段完全够用。

方案二：全栈应用（1 个周末完成）

项目：个人书签管理器 + AI 标签

一个网页应用，保存你的书签，AI 自动给每个链接生成标签和摘要。支持搜索和分类浏览。

技术栈建议：Next.js + SQLite + Tailwind CSS。部署到 Vercel，免费。

周六的计划：

  • 上午：让 Claude Code 搭建项目骨架、数据库模型、基础 CRUD
  • 下午：实现书签保存、AI 标签生成、搜索功能
  • 晚上：调整 UI，修 Bug

周日的计划：

  • 上午：添加浏览器扩展（一键保存当前页面）
  • 下午：部署到 Vercel，测试
  • 晚上：修复问题，收尾

周末全天都是非高峰，双倍额度 48 小时不间断。一个完整的全栈应用，两天足够。

方案三：老项目重构（3-5 天）

项目：给你现有的项目加上 TypeScript + 测试

你手头一定有那么一个”能跑但不敢动”的项目。JS 写的，没类型，没测试，改一行炸三行。

这是 Claude Code 最擅长的事情。

让 Claude Code 用 Plan 模式先分析，确认方案后再执行。大上下文（1M token）+ 双倍额度 = 一口气把整个项目理清楚再动手。

最大化利用的 6 个技巧

1. 提前准备好 CLAUDE.md

在开始之前，花 5 分钟写一个 CLAUDE.md 放在项目根目录。把技术栈、项目结构、编码规范写清楚。这样 Claude Code 不需要反复问你基本问题，节省大量额度。

2. 用 Plan 模式开头

按 Shift+Tab 进入 Plan 模式。先让 Claude 分析项目、制定方案，你确认后再执行。不要上来就让它直接写代码——有计划的执行比盲目动手省 3 倍额度。

3. 批量操作安排在周末

周末全天双倍，没有高峰限制。把最吃额度的操作集中在周末：代码库分析、大规模重构、批量生成测试。

4. 工作日利用好下午和晚上

北京时间下午 2 点到晚上 8 点，双倍额度在线。如果你白天要上班，下班后到晚 8 点之前还有几个小时的双倍窗口。

5. 一次说清楚需求

不要一句一句地跟 Claude Code 对话。把需求一次性说清楚：要做什么、技术栈、文件结构、期望输出。一次清晰的 Prompt 比十次模糊的提问高效十倍。

6. 不要浪费额度在简单任务上

改个变量名、调个颜色——这些事自己做更快。把 Claude Code 的额度留给真正有价值的任务：架构设计、复杂逻辑、跨文件重构、自动化测试。

免费用户的”零成本”方案

如果你连 Pro 都不想买，照样可以参与：

方案一： Claude 免费账户 + 非高峰双倍额度。能做简单的脚本和工具。

方案二： 新用户注册 Claude Pro 有 30 天免费试用。在活动期间注册，白嫖 Pro 双倍额度。试用到期前取消，一分钱不花。
