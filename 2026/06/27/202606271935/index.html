一人公司第 8 课：产品上线那天后台访客全是我自己刷的，第一批 100 个用户是怎么一个个捞回来的

发表于 2026-06-27 更新于 2026-07-08 分类于 产品 阅读次数： 本文字数： 2.4k 阅读时长 ≈ 2 分钟
一人公司最苦的一关不是做不出产品，是上线那天后台访客全是你自己刷的。第一批 100 个种子用户没有自动来的，全靠你顺着公众号博客小红书知乎朋友圈微信群一个个手动捞。这篇把每个渠道第一步发什么和一份 100 用户行动计划摆给你，也老实告诉你冷启动有多熬人。

哈喽，我是飞飞。

这是「一人公司成长指南」的第 8 课。前面几课我们聊过怎么找方向、做出第一版产品。今天这课只回答一个问题，也是最扎心的一个：东西做出来了，上线了，然后呢？没人用。

我先把自己最难看的一段摆出来。我有个小产品上线那天，我守着后台数据看了一晚上。访客数确实在涨，我还美了几分钟，后来一对时间戳，全是我自己。换个浏览器刷一下，加一；用手机点一下，再加一。那一晚上的「流量」，是我一个人左手倒右手刷出来的。真实的外部用户，零。

如果你正卡在这一步，别慌，也别觉得是自己废。这是每个一人公司都要单独闯的一关，而且基本是最熬人的一关。

没有用户的产品，比没做出来还危险

做不出产品，你至少知道自己卡在哪，再学个技术、再磨几天就过去了。可产品做出来了没人用，是一种很要命的悬空状态：你不知道是东西不行，还是没人知道它存在。

大多数个人产品不是死在做不出来，是死在做出来之后没人用，作者自己先撑不住，慢慢就不更新了。我见过太多 GitHub 上停在 v0.1 的仓库，小程序后台常年个位数的日活。它们栽的地方大多跟技术无关，就卡在从来没闯过第一批用户这道坎。

所以冷启动不能当成上线之后顺手做做的事。它本身就是你产品的一部分。你得把「怎么把头 100 个人弄进来」当成跟写代码一样正经的活来排。

冷启动的真相：种子用户是手动捞的，不是自动来的

我知道你想听的是某个一键起量的招。我没有，没人有。

种子用户这东西，前 100 个几乎是一个一个手动聊回来的。你私信一个人，发一句「我做了个东西能解决你那个问题，要不要试试」，对方回了，进来了，这就是 1。再重复 99 次。这个过程没有任何自动化能替你跑，也别指望算法会突然把你推上去，冷启动阶段你根本没有数据喂给算法。

听着像体力活，它就是体力活。但这件体力活有个巨大的好处：你每捞一个人，都直接跟他说上话，他卡在哪、嫌哪里难用、为什么不愿意付钱，全是当面问出来的。这批人给你的反馈，比你后面有了一万用户之后看任何报表都值钱。所以别嫌手动捞慢，这一程的慢是有复利的。

第一批 100 个人，先问他们现在在哪刷手机

捞人之前先想清楚一件事：你的人现在在哪。别泛泛说「在互联网上」，要具体到他此刻正盯着哪个 App 的哪个页面。

把这几个常见的地方过一遍，每个我只说第一步发什么，你照着做就行。

公众号：别一上来推产品。先写一篇你做这个东西的过程，遇到什么问题、怎么解决的，文末轻轻带一句「我把它做成了个小工具，有需要私信我」。内容是钩子，产品是钩子上的饵。

博客：把同一件事写成一篇更长更细的技术文，挂上你的关键词，让它能被搜到、被 AI 引用。这是慢钱，但博客这块地是你自己的，文章会一直在那儿替你捞人，哪怕几个月后。

GitHub：如果你的东西沾代码，开源一部分，写一份让人五分钟跑起来的 README。一个清爽的 README 本身就是最好的落地页，star 是免费的信任背书。

小红书：别发硬邦邦的产品介绍，要发「我用这个解决了什么具体的小烦恼」，配一张干净的效果图。这里的人吃的是真实场景，越不像广告越有人点。

知乎：找三个跟你产品强相关的问题，认真答，把你的方案揉进答案里，最后留一个原文回链。知乎的答案沉得久，半年后还有人顺着搜索点进来。

朋友圈：这是你最该先用、却最容易因为不好意思而跳过的地方。你那几百个微信好友里，一定有几个正好需要。发一条，配上能用的图，让真认识你的人先帮你试，先帮你转。

微信群：你潜水的那些群，别硬广。先在群里持续答几次相关的问题，混个脸熟，等有人问到你产品能解决的事，再自然地接一句。一个群里捞回三五个真用户，已经很好了。

一篇内容铺满所有渠道，是一个人扛得住冷启动的关键

你可能已经发现问题了：上面七个地方，每个都要发东西，一个人哪忙得过来。

我的办法是一篇内容铺到所有渠道。你正在看的这篇文章，本身就是这么分发出去的：写一遍，自动产出公众号、博客、知乎、掘金、CSDN 各平台的版本，每个版本按平台的脾气调好格式。我背后挂的是一条写作流水线，里头有个专门做分发的环节替我干这件事。一个人之所以能同时在七个渠道露脸，靠的就是这个，不然光复制粘贴改格式就能把人耗死。

配图也是一样。每个渠道都得有像样的视觉，封面、效果图、流程图，全靠人画一个人根本配不齐。我这边是一句话触发自动出图，主题丢进去图就回来了，每个渠道的脸面才能都撑住。冷启动阶段你的时间是最稀缺的，凡是能让一份力气覆盖多个渠道的，都值得先搭起来。

把它排成一份 100 用户行动计划

光知道渠道没用，得落成一张你明天就能开干的表。我建议你这么排。

先定一个周期，比如三十天，目标就是手动捞够 100 个真实用户，自己刷的不算。把 100 拆到每天，平均每天三到四个，听着不多，但你要真去聊就知道不轻松。

然后给每个渠道派一个固定动作：公众号一周一篇过程文，博客同步一篇长文，知乎一周答三个相关问题，小红书隔天一条场景图，朋友圈每次有进展就发一条，GitHub 把 README 一次性写好，微信群每天去一个群冒个泡。把这些写进日历，当成上班打卡，别凭心情。

最后留一栏专门记反馈。每捞进一个人，把他说的最扎心那句话记下来。攒到二三十条，你会发现产品该往哪改，比你自己拍脑袋清楚得多。这份表跑完一个周期，你手里就有了第一批种子和第一版真实反馈，这才算真正活过了冷启动。

最后泼盆冷水

我不想让你看完觉得照着发几天就能起量。不会的。

我自己的博客铺了那么多渠道，做了 GEO，真实回流也就 0.21%，一千个看到的人里两个点进来。冷启动就是这么个投入产出比，前期几乎全是付出看不到回报，靠的是你认准了这事值得，硬扛过那段没人理你的安静。

前 100 个用户基本没有捷径，就是一个一个聊出来的。但你只要真把这 100 个手动捞够，就跨过了大多数人跨不过去的那道坎，后面才谈得上让系统接手。

你现在的产品卡在第几个用户？你最不好意思发的是哪个渠道，朋友圈还是群里？评论区告诉我，我们一起想想第一句话该怎么发。
