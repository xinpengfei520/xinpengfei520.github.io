AI 时代的个人竞争力，正在从「会干活」挪到「会指挥一队 AI」

发表于 2026-06-28 更新于 2026-07-21 分类于 观点 阅读次数： 本文字数： 2.6k 阅读时长 ≈ 2 分钟
Anthropic CEO Amodei 在 Code with Claude 大会赌 2026 年会出现第一家只有一个人的十亿美金公司，给了七到八成把握。我借这事讲清一个判断，AI 把写代码、写文章这些执行做得又快又便宜之后，个人竞争力的重心正往发现问题、把活拆给一队 AI、判断哪个产出能放行这层挪，这套高阶课第一篇就从这讲起。

哈喽，我是飞飞。

先说个我这两天一直在琢磨的事。Anthropic 的 CEO Amodei，去年五月在他们自己的 Code with Claude 开发者大会上，被问「你觉得第一家只有一个人的十亿美金公司什么时候出现」，他几乎没犹豫，张口就是「2026 年」。后来记者追问，他才稍微收了口风，说大概有七到八成把握。

这话我放心里转了好久。前面两套课，我跟你聊的是怎么把 AI 用顺手、怎么把零散的工具搭成一套自己的系统。这一套要往上再走一层：当执行这件事被 AI 做得又快又便宜，一个人到底靠什么挣钱、靠什么攒下属于自己的资产。说白了，就是从「会用 AI」，走到「用 AI 给自己开一家公司」。

2012 年的十亿美金公司要 13 个人，现在有人赌只要 1 个

你可能对「一个人十亿美金」这个数字没概念，给你找个老坐标。2012 年 Facebook 花十亿美金买下 Instagram 的时候，Instagram 全公司就 13 个人。这事当年已经被当神话讲了好多年，13 个人撑起十亿估值，少得吓人。

Amodei 现在赌的，是把这个 13 直接干到 1。

我得先泼自己一盆冷水。Amodei 是卖 AI 模型的，他站台说一个人能干十亿的活，里头多少有点生意人的吆喝，这话你不能照单全收。可抛开吆喝，底下那股趋势是真的。Carta 那份 Solo Founders 报告里，单人创始的新公司占比，从 2019 年的两成三出头，涨到了 2025 上半年的三成六。去年六月还有个叫 Maor Shlomo 的人，一个人攒了个 AI 产品 Base44，做了半年，被 Wix 八千万美金收走。这种事搁以前想都不敢想，一个人，半年，八千万。

我现在干活，已经是一个人盯着一队 AI

说这些不是给你画饼。这股趋势离我没那么远，因为我自己每天就活在它的初级版里。

你正在读的这篇课，说出来你可能不信，我没怎么动手敲。它是我一个人，指挥着一队 AI agent 弄出来的。我这条写作流水线上，挂着十几个 skill：有专门去拉 AI 资讯的，有查重的，有配图的，有把成稿推到公众号的。每一个都是我照着 Claude Code 的脾气，一点一点磨出来的。

光说不够直观，给你拆一遍这篇课的诞生过程。早上我先让一个 skill 去把当天 AI 圈的动静拉回来，另一个 skill 按我定的那套维度给每条选题打分、排个序，我扫一眼，拍板挑哪个。选题定了，后面的调研、起草、降 AI 味、配图、转成各平台的版本，是一串 agent 接力跑完的。我从头到尾干的，是开头那一下拍板，和中间一道道关卡的验收。

就拿配图说。我从来不自己画封面。我那个叫 content-artist 的 skill，会自己读完文章、琢磨该画个什么画面、调 Nano Banana 出图、再传到我自己的图床上。我这头只管丢一句「给这篇配张封面」，回头图就在那儿了。有时候连配色我都懒得讲，丢个主题它自己拿捏，八九不离十。这套出图的活我挂着跑了大半年，封面没上百也有几十张了，我自己一笔没画。

你发现没有，我在这条流水线上真正花心思的地方，根本不在动手写、动手画。我干的是另一件事：盯着方向，做判断。哪个选题值得写，agent 拉回来一堆我挑哪个；图出来了我看顺不顺眼、要不要重来；稿子润完了我读一遍，决定它能不能发。活是一队 AI 干的，方向盘攥在我手里。

以后还值钱的，到底是哪一层能力

好，重点来了。会写代码、会写文章，这些本来是吃饭的本事，正在慢慢滑成一道门槛。门槛的意思是，它还重要，但它只是张入场券，不再是你比别人强的地方了，因为 AI 也能干，而且越来越便宜。

那比别人强的地方挪哪去了？往上那层去了。一个值不值得做的问题，你能不能先看出来；看出来之后，你能不能把这件大事拆成一个个 AI 接得住的小活；活干完了堆你面前，你能不能判断哪个能放行、哪个得打回去重来。这层能力，AI 暂时替不了你，因为它总得有个人替它定方向、替它兜底。

这层的债我天天都在还。给 agent 派活之前，我会下意识掂量一下值不值得动用 Opus，杂活就手动切个便宜模型省点额度，这种用之前先在心里过一遍秤的绷着感，就是判断。哪个 agent 适合干哪类活、贵的模型留给哪一步、便宜的顶哪一段，这套分配也得我来定，没人替我拿主意。还有更惨的，前阵子我博客那条自动部署的流水线，因为一个 token 过期，悄无声息挂了一整天，构建明明是绿的，只有最后那步推送红了，没人盯就没人知道。那天我学到的是，自动化堆得越高，「有个人盯着、出了事能判断」这层就越金贵。机器能把活干完，但它不会替你着急。

「一人公司」不是劝你裸辞，是先做到「够用」

聊到这，得赶紧按一下刹车。超级个体、一人公司，这些词太热血了，热血到容易被熬成鸡汤，撺掇人冲动裸辞去单干。我想讲的完全不是这个。

有本书叫《Company of One》，作者 Paul Jarvis，2019 年写的，我挺认他那套底子。他通篇在质疑一件事：公司是不是非得越做越大。他的答案是不必。先做到他说的「最小可行利润」，用最小的那个版本先跑起来，别一上来就惦记着融资、扩张、招人。过早地求大，反倒是把自己拖垮的那个东西。

把这套搬到 AI 时代，我的理解是这样：你不用真去注册一家公司，你要练的是「一个人调动一整支队伍」的手艺。只不过你这支队伍，现在整个换成了一队 AI。你不必什么都自己会，你要会的是整合，把合适的活派给合适的工具，让它们替你把事办成。Maor Shlomo 那种一个人卖八千万的，是这条路上跑得最远的极少数，多数人到不了那儿。可哪怕你只是想给自己开一条小小的、能挣点钱的副线，这套「指挥一队 AI」的能力，都是地基。

我还没靠这套挣到大钱，但有件事现在就能攒

最后我得跟你交个底，免得你把我当成什么财务自由的导师。我不是。实话讲，我到现在还是个 builder，手里这套指挥 AI 的流水线，跑得挺顺，但我还没把它变成一家真正赚大钱的一人公司。方向我信，路我也在走，结果还没到。这话得说在前头，不然就是骗你。

但有件事，不用等你财务自由，今天就能开始攒，就是上面说的那两样。一个是判断力，多逼自己想想「这件事到底值不值得做」，而不是埋头先干起来再说。另一个是把活拆给 AI、再回头验收 AI 的那点手艺，这玩意儿练得越熟，你一个人能扛的事就越多。这两样攒厚了，等真有一天机会砸下来，你才接得住。

我自己也还在半道上，这套课往后我会一篇篇接着写，把我趟过的坑都摆给你看。今天先撂个问题给你：要是明天你手底下真有一队随叫随到的 AI，你最想先让它们替你把哪件一直没空做的事给办了？
