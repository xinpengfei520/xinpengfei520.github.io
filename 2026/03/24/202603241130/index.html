一句话生成全栈应用：Google AI Studio 的 Vibe Coding 到底能做什么

发表于 2026-03-24 更新于 2026-03-30 分类于 工具 阅读次数： 本文字数： 3.5k 阅读时长 ≈ 3 分钟

3 月 19 号深夜，Google 悄悄干了一件大事。

AI Studio 原来是个 Gemini 模型的试玩场——你写个 prompt，它给你生成个前端页面，关掉浏览器就没了。没数据库，没用户登录，没后端逻辑，顶多算个交互式的 demo 生成器。

现在不一样了。Google 直接把 Antigravity 编程 Agent 和 Firebase 后端塞进了 AI Studio，搞出了一个全栈 Vibe Coding 平台。你用一句话描述你想要什么应用，它帮你从前端到后端、从数据库到用户认证、从开发到部署全链路搞定。

这一刀下去，Bolt.new、Lovable、Replit Agent 全成了被瞄准的靶子。

先搞清楚：什么变了

旧版 AI Studio 能干的事很有限——生成个好看的前端页面，加点动画效果，仅此而已。关掉浏览器标签页，你做的东西就消失了。想加个数据库？自己搞。想要用户登录？自己搞。想部署上线？还是自己搞。

新版 AI Studio 补齐了所有短板：

数据库和用户认证自动接入。 这是最大的变化。当你的 prompt 里提到”用户注册””保存数据”之类的需求时，Agent 会主动问你要不要启用 Firebase。点一下”Enable Firebase”，它自动帮你配好 Cloud Firestore 数据库和 Firebase Authentication 登录系统，连登录页面的代码都帮你写好了。

能调外部 API 了。 想接 Google Maps 拿实时定位？想接支付接口？现在可以了。AI Studio 新增了 Secrets Manager，你把 API Key 存进去，Agent 会自动检测什么时候需要用到，然后安全地调用。

多人实时协作。 这个能力比较骚——Google 官方的 demo 里直接做了一个多人激光枪对战游戏，实时同步、排行榜、AI 对手全齐了。还有一个 3D 粒子协作画布，多人同时操作，每个人的鼠标都会生成不同的粒子效果。

支持 React、Angular、Next.js。 不再只是生成静态页面，支持服务端逻辑，能通过 prompt 安装 npm 包。说一句”加上 Framer Motion 动画”，Agent 自己去装包、写代码。

关掉浏览器不丢进度了。 听起来是个基本功能，但对 Vibe Coding 工具来说，这一步很多竞品都没做好。

我昨晚花了十分钟试了一下，让它做一个带用户注册的待办清单应用。从输入 prompt 到看见带 Google 登录的完整页面，大概三分钟。Firebase 的数据库配置、认证逻辑它全自动搞定了，我一行配置代码都没碰。说实话，有点被震到。以前光配 Firebase 就得折腾半小时，现在点一下按钮就完事了。

Antigravity：从 Windsurf 到 Google 的那个 Agent

Antigravity 这个名字你可能听过。它是 Google 去年 11 月跟 Gemini 3 一起发布的桌面 IDE，底层技术来自 Windsurf 团队——Google 花了 24 亿美元收购的。

这次 AI Studio 里用的不是完整的 Antigravity IDE，而是把它的 Agent 能力抽出来，嵌进了浏览器端的 Vibe Coding 工作流。这个 Agent 的核心能力是：理解整个项目结构，执行多步骤的代码修改。不是那种改一行代码的补全工具，是真正能理解”把登录模块改成 Google OAuth，同时把用户数据存到 Firestore 里”这种复杂指令的。

背后跑的模型是 Gemini 3.1 Pro。Google 说内部团队已经用这套系统构建了”数十万个应用”——虽然大部分大概率是测试用的一次性项目，但至少说明基础设施经过了压力测试。

不过也有早期用户反馈，Antigravity 在处理超过一千行代码的项目时稳定性会下降。这一点值得关注，毕竟”能跑 demo”和”能上生产”之间隔着一条太平洋。

Google 同时杀掉了 Firebase Studio

这个细节很多报道没怎么提，但我觉得是最值得关注的一个信号。

Google 在同一天宣布：Firebase Studio 将于 2027 年 3 月 22 日下线。 从今年 6 月开始，你就不能在 Firebase Studio 里创建新项目了。

Firebase Studio 是什么？是 Google 去年 4 月在 Cloud Next 上才刚发布的全栈开发环境。一个产品从发布到宣布退役，总共活了不到一年。有人在 X 上算了一下——Firebase Studio 超过一半的生命周期是在”夕阳期”里度过的。即使按 Google 杀产品的标准来看，这速度也挺狠的。

为什么要杀？因为 Google 手里出现了三个高度重叠的开发者工具：AI Studio（浏览器端 Vibe Coding）、Antigravity（桌面 IDE）、Firebase Studio（全栈开发环境）。三个工具做的事越来越像，留着只会让开发者困惑。砍掉 Firebase Studio，把定位理清楚：AI Studio 负责快速原型和轻量开发，Antigravity 负责严肃的工程项目，Firebase 作为底层后端服务贯穿两者。

三变二，逻辑清楚了。但对那些刚在 Firebase Studio 上安顿下来的开发者来说，迁移成本是实打实的。

竞品该紧张了吗

浏览器端 Vibe Coding 这条赛道已经不是新鲜事了。Bolt.new 打了先锋，Lovable 紧跟其后，Replit Agent 也在做类似的事。Vercel 的 v0 更是把”prompt 生成 UI”这个概念推到了主流视野。

但这些工具有一个共同的短板：前端很行，后端很弱。 生成一个好看的界面没问题，但要加用户系统、加数据库、接第三方 API，你还是得自己动手。很多 Vibe Coding 的项目就是卡死在这一步的——界面做得漂漂亮亮，一涉及后端就歇菜了。

Google 这次的打法就是冲着这个痛点来的。Firebase 是 Google 自家的东西，集成深度是 Bolt 和 Lovable 没法比的。数据库、认证、云函数、部署——全在 Google 的生态里，不需要你去对接第三方服务。

Product Hunt 上有个评论说得很到位：”auto-detect for login, APIs and library installs is a big deal. That setup phase is where most vibe coding sessions fall apart for me.”（自动检测登录、API 和依赖安装才是重点，这个配置阶段正是我每次 Vibe Coding 翻车的地方。）

当然，Google 的如意算盘也很明显：免费层很慷慨，但生产环境部署要走 Gemini API 或者 Vertex AI，是要花钱的。这条路径设计得很巧妙——免费体验把你钩进来，真要上线就得掏钱用 Google Cloud。是开发者友好的便利还是精心设计的付费漏斗？取决于你怎么看。

同一天，OpenAI 也出手了

巧的是，同一天 OpenAI 也宣布要把 ChatGPT、Codex 和 Atlas 浏览器合并成一个桌面”超级应用”，还收购了 Python 工具链公司 Astral。

Cursor 也没闲着，同一周发布了 Composer 2，成本降了 86%。

AI 编程工具的竞争在 2026 年 3 月进入了白热化。Google、OpenAI、Anthropic、Cursor——都在抢同一个位置：成为开发者的默认工具。

对开发者来说，这可能是近几年最划算的时候。去年同一时间 Cursor Pro 还要 20 美元一个月，现在 Composer 2 成本降了 86%；Google AI Studio 的 Vibe Coding 免费就能用；Claude Code 也在不断降价。巨头们打架，开发者捡便宜。唯一的烦恼是选择太多了——你得花点时间搞清楚哪个工具适合哪个场景。

我的判断

Google AI Studio 这次更新，最大的价值不是某个单独的功能，而是把全栈开发的门槛拉到了零。

以前你想做一个带用户系统的 Web 应用，哪怕用最简单的技术栈，也得配数据库、写认证逻辑、处理部署——光这些脚手架工作就能吃掉一个下午。现在你说一句话，Agent 三分钟帮你搞定。

但 Google 官方说的”生产级应用”，我觉得要打个折。昨晚我试着让它加一个付费功能，生成的 Stripe 集成代码没有做 webhook 验证，安全性上是不过关的。所以我的判断很明确：拿来给客户演示概念、做内部管理工具、hackathon 抢时间——现在就能用，而且非常好用。但如果你要做一个处理支付或者存敏感数据的 SaaS 产品，生成的代码必须逐行审查，不能直接上。

方向是确定的。Google 用 Firebase 生态补齐了后端这块最大的拼图，其他 Vibe Coding 工具如果不尽快补上全栈能力，很快就会被甩开。
