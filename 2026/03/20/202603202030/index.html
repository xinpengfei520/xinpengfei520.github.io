外媒曝 OpenAI 内部全面恐慌！被 Claude Code 逼急的奥特曼，还能留住初级程序员吗？

发表于 2026-03-20 更新于 2026-04-13 分类于 观点 阅读次数： 本文字数： 3.2k 阅读时长 ≈ 3 分钟

Sam Altman 盘腿坐在办公椅上，盯着天花板。

这是 WIRED 记者描述的场景。当被问到「为什么 AI 领域最大的名字，却在 AI 编程革命中掉队了」时，这位 OpenAI 的掌门人沉默了一会儿，说了一句：「先发优势很重要。我们在 ChatGPT 上有过这个优势。」

言下之意——在 AI 编程这场仗上，这个优势属于 Anthropic。

上周，WIRED 发布了一篇万字长文，标题直接了当：《Inside OpenAI’s Race to Catch Up to Claude Code》（OpenAI 追赶 Claude Code 的内幕）。与此同时，India Today 报道 OpenAI 内部进入了「Code Red」状态，The Wall Street Journal 则披露高管要求员工放下「支线任务」，全力聚焦核心产品。

一家估值 3000 亿美元的公司，在追赶一家「叛逃者」创办的竞争对手。而这场追赶的代价，可能是一整代初级程序员的职业前景。

OpenAI 到底怎么掉队的

故事要从 2021 年说起。

那年，OpenAI 就已经做出了第一个 Codex——一个能把英语指令翻译成代码片段的模型。他们把它授权给了微软，做成了 GitHub Copilot。产品大火，几个月内用户过百万。

然后呢？Codex 团队被拆散了。有人去做 DALL-E 2，有人去训练 GPT-4。公司内部的逻辑是：编程能力会自然地融入未来的大模型里，不需要单独搞一个产品。况且，「这块有 GitHub Copilot 在做了。」

2022 年底 ChatGPT 爆火，一切都停下来了。全公司围着消费级产品转。整个 2023 和 2024 年，OpenAI 没有一个专门的 AI 编程团队。

而 Anthropic 做了一个完全不同的选择。

2024 年初，Anthropic 开始在真实的、混乱的代码仓库上训练 Claude 模型。同年 6 月，Claude Sonnet 3.5 发布，编程能力让开发者惊艳。Cursor 接入这个模型后，用量一路飙升。再之后，Anthropic 内部开始测试 Claude Code。

2025 年 2 月，Claude Code 以「研究预览」的形式发布。5 月正式上线。OpenAI 的回应？还在「组建冲刺团队」。

WIRED 的数据非常扎眼：2025 年 9 月，Codex 的使用量只有 Claude Code 的 5%。 到 2026 年 1 月，这个数字追到了 40%——进步很大，但仍然是在追。

收入差距同样明显。Claude Code 的年化收入超过 25 亿美元，占 Anthropic 总营收近五分之一。同期 Codex 的年化收入刚刚超过 10 亿美元。

一系列追赶操作

面对差距，OpenAI 做了什么？

收购 Windsurf——失败。 Altman 试图以 30 亿美元收购 AI 编程创业公司 Windsurf，希望一步到位获得成熟产品和客户。但微软搅了局——作为 OpenAI 的最大投资方，微软想要 Windsurf 的知识产权。交易拖了几个月，最终在 2025 年 7 月流产。Windsurf 的创始人去了 Google，团队被 Cognition 收了。

收购 Astral——成功。 2026 年 3 月 19 日（昨天），OpenAI 宣布收购 Astral，一家做 Python 开发工具的公司，旗下有 uv、Ruff 等工具。Astral 团队将并入 Codex 部门。这次 OpenAI 学聪明了——通过收购嵌入开发者工作流的基础设施来加固 Codex。有趣的是，Anthropic 去年 12 月也做了类似的事：收购了 Bun JavaScript 运行时。

内部进入「Code Red」。 India Today 引述消息称，OpenAI 高管 Fidji Simo 告诉员工，公司的运作方式要「当作是 Code Red」。所有支线项目暂停，编程工具和企业产品成为最高优先级。The Verge 报道 OpenAI 正在把 ChatGPT、Codex、Atlas 浏览器合并成一个桌面「超级应用」——对标的正是 Anthropic 已有的 Claude Desktop 一体化体验。

疯狂更新 Codex。 Codex 发布了独立桌面应用、GPT-5.3-Codex 模型、安全 Agent（发现了 14 个真实 CVE）、Skills 和 Worktree 支持。周活用户从年初至今翻了三倍，达到 200 万。用量增长了五倍。

追得很用力。但 Claude Code 也没停——VS Code 插件安装量 520 万，评分 4.0（Codex 是 490 万和 3.4）。在小公司和创业团队中，Claude Code 的渗透率高达 75%。

真正让人不安的：初级程序员怎么办

两家公司打得头破血流，开发者可以坐享其成。但有一群人笑不出来——初级程序员。

先看数据：

  • LinkedIn 数据：2026 年前两个月，新增软件工程岗位比去年同期减少 15%
  • Resume.org 调查：六成美国企业计划在 2026 年裁员，四成计划用 AI 替代员工
  • 斯坦福研究：初级开发者是「AI 曝露度」最高的职业之一
  • 纽约时报报道：过去四年，超过 70 万科技工作者被裁

这些数字背后的逻辑很简单。

以前，公司雇初级程序员做什么？写基础函数、做简单功能、修小 Bug、跑测试。这些活儿是「学徒工」的必经之路。但现在，一个 Senior 带着 Claude Code，能做原来 Senior + 2 个 Junior 的活。公司的算法很直接：为什么花钱雇一个需要 3-6 个月才能上手的新人，如果 AI 能即时上手？

Cisco 的首席产品官 Jeetu Patel 对员工说的话非常有代表性：「不，你不会因为使用这些工具而丢掉工作。但我保证，如果你不用，你一定会丢掉工作。因为你会变得不再有价值。」

这段话翻译成大白话就是：工具不会取代你，但会用工具的人会取代你。

但问题是——如果初级岗位消失了，谁来培养下一代中级和高级工程师？

这是一个被忽视的悖论。AI 可以替代初级程序员的工作，但它不能替代初级程序员变成高级程序员的过程。 如果这条路径断了，五年后整个行业可能面临高级人才的断层。

程序员该怎么办

不贩卖焦虑，说几条实际的。

1. 别再只学写代码了。 Morgan Stanley 的研究认为，AI 会创造更多软件工程岗位，但这些岗位需要的是「系统思维者」而不是「代码编写者」。架构设计、性能优化、安全审计、复杂系统的权衡取舍——这些 AI 短期内做不了。

2. 学会「编排」而不是「编写」。 2026 年最稀缺的技能不是写 React 或 Python，而是：给 AI Agent 下精准的任务指令，审查 AI 生成的代码质量，在多个 Agent 之间做编排决策。Pragmatic Engineer 的调查显示，使用 AI Agent 的工程师对 AI 的态度积极度是不使用者的两倍。

3. 两个工具都用。 500 多个 Reddit 开发者的真实反馈显示：2026 年的最优策略不是选 Claude Code 还是 Codex，而是都用。Claude Code 擅长深度推理和大型重构，Codex 擅长并行任务和快速迭代。很多人的工作流是：Claude Code 做架构和提交，Codex 做审查和调试。

4. 如果你是初级程序员，现在比以前更需要主动学习。 不要等公司的培训计划——很多公司已经砍掉了。自己用 Claude Code 做项目，但要刻意练习去理解 AI 生成的每一行代码。你的竞争对手不是 AI，而是那些比你更会用 AI 的同行。

写在最后

OpenAI 和 Anthropic 的这场战争，本质上是在争夺一个「多万亿美元的市场」——这是 Altman 自己的原话。

对我们普通开发者来说，竞争是好事。Codex 在拼命追赶的同时，Claude Code 也在疯狂发新功能。价格在降，能力在涨，体验在好转。

但别忽略硬币的另一面。

Greg Brockman 说了一段很诚实的话：「当你成为这支由成千上万个 Agent 组成的舰队的 CEO，它们在执行你的目标、你的愿景的时候……你不再深入了解具体问题是怎么被解决的。」他说这种新的工作方式有时候让他觉得——「自己正在失去对问题的脉搏感知。」

一个连 OpenAI 总裁都在担心的事情。值得每一个程序员深思。

你觉得 AI 编程工具会彻底取代初级程序员吗？还是会像之前每次技术革命一样，淘汰旧岗位的同时创造新岗位？评论区聊聊你的看法。
