OpenClaw从入门到精通：如何让小龙虾自我进化

发表于 2026-03-17 更新于 2026-03-18 分类于 阅读 阅读次数： 阅读次数： Valine：

你有没有想过，如果有一个 AI 助手能 24 小时待命，在你的 WhatsApp、Telegram、Discord 里随时响应，还能帮你执行脚本、管理文件、自动化工作流？这不是科幻，这就是 2026 年初在开源社区爆火的 OpenClaw。

这个项目在短短几周内从一个不起眼的 GitHub 仓库飙升到 24 万+ Star，甚至催生了一个叫 Moltbook 的”AI 社交网络”。它的吉祥物是一只小龙虾🦞，但它的能力远超你的想象。今天我们就来聊聊，OpenClaw 到底是什么，以及如何让这只”小龙虾”为你工作。

OpenClaw 是什么？为什么突然火了

从 Clawdbot 到 OpenClaw 的爆红之路

OpenClaw 的故事很有意思。2025 年 11 月，奥地利开发者 Peter Steinberger 发布了一个叫 Clawdbot 的项目，名字来源于 Anthropic 的 Claude。结果 2026 年 1 月底，Anthropic 发来商标投诉，他只好改名叫 Moltbot（保持龙虾主题）。三天后又觉得这名字”念起来不顺口”，最终定名 OpenClaw。

就在改名的同时，创业者 Matt Schlicht 推出了 Moltbook——一个专门给 AI Agent 用的社交网络。你没看错，AI 之间可以互相加好友、发帖、聊天。这个脑洞大开的项目迅速走红，连带着 OpenClaw 也火了。

到 2026 年 3 月初，OpenClaw 在 GitHub 上已经有 24.7 万 Star 和 4.77 万 Fork。硅谷和中国的公司都在用它，腾讯甚至基于它推出了一整套 AI 产品，还能接入微信。

它到底能做什么

简单说，OpenClaw 是一个本地运行的 AI Agent 网关。它不是一个聊天机器人，而是一个能主动做事的智能助手。

核心能力包括：

  • 多平台接入：支持 WhatsApp、Telegram、Discord、Slack、Signal、iMessage 等 20+ 个消息平台
  • 本地优先：所有数据存在你自己的设备上，不上传到云端
  • 技能系统：通过 Skills 扩展能力，可以执行 Shell 命令、管理文件、自动化浏览器操作
  • 模型无关：可以接入 Claude、GPT、DeepSeek 等任何大模型
  • 语音交互：支持唤醒词和持续语音对话（macOS/iOS/Android）
  • 自主执行：不只是回答问题，还能主动完成任务

为什么开发者都在关注它

传统的 AI 助手，比如 ChatGPT、Claude，你问一句它答一句。但 OpenClaw 不一样，它是一个自主 Agent。

举个例子：你可以让它”每天早上 9 点检查我的邮箱，把重要邮件总结后发到我的 Telegram”。它会真的去执行，而不是只给你一个方案。

更重要的是，它是开源的。这意味着：

  • 你可以看到所有代码，知道它在做什么
  • 可以自己修改和扩展功能
  • 数据完全在你的控制之下
  • 不用担心服务商突然涨价或关停

这种”本地优先 + 自主执行 + 开源透明”的组合，正是开发者们梦寐以求的。

进阶玩法：让小龙虾自我进化

技能系统：OpenClaw 的灵魂

OpenClaw 的强大之处在于它的技能系统（Skills）。每个技能就是一个独立的能力模块，存储在 .claude/skills/ 目录下。

技能的结构很简单：

SKILL.md 文件包含：

  • 技能名称和描述
  • 触发条件
  • 使用的工具
  • 执行逻辑

OpenClaw 内置了 100+ 个预配置技能，涵盖：

  • 文件系统操作
  • Shell 命令执行
  • 浏览器自动化
  • API 调用
  • 数据处理

多 Agent 路由：不同场景用不同助手

OpenClaw 支持多 Agent 配置。你可以为不同的场景创建专门的助手：

  • 工作助手：处理邮件、日程、文档
  • 学习助手：整理笔记、搜索资料、生成总结
  • 生活助手：提醒事项、查询信息、娱乐推荐

通过配置文件，可以让不同的消息平台或账号路由到不同的 Agent。比如工作相关的消息发到 Slack，个人消息发到 Telegram，各自由专门的 Agent 处理。

实战案例：OpenClaw 能帮你做什么

案例 1：自动化客户线索管理

小型企业和自由职业者用 OpenClaw 自动化销售流程：

  • 监控潜在客户的网站
  • 自动生成审计报告
  • 整合到 CRM 系统
  • 发送跟进邮件

这个流程原本需要人工花费大量时间，现在完全自动化了。

案例 2：多平台内容同步

内容创作者用 OpenClaw 实现：

  • 在一个平台发布内容
  • 自动转换格式
  • 同步到其他平台（Twitter、LinkedIn、微信公众号）
  • 收集各平台的反馈和数据

案例 3：智能家居控制中枢

通过 OpenClaw 连接各种智能设备：

  • 语音控制家电
  • 根据日程自动调节环境
  • 异常情况及时通知
  • 能源使用分析和优化

需要注意的安全问题

OpenClaw 很强大，但也带来了安全风险。

DM 访问控制

默认情况下，OpenClaw 对陌生人的私信采用”配对模式”。陌生人发消息时会收到一个配对码，你需要手动批准：

如果要开放公开访问，需要显式配置 dmPolicy="open"。但这样做风险很大，因为任何人都能让你的 Agent 执行命令。

Prompt 注入攻击

OpenClaw 容易受到 Prompt 注入攻击。恶意用户可能在消息中嵌入指令，诱导 Agent 执行危险操作。

防范措施：

  • 限制 Agent 的权限范围
  • 对敏感操作设置人工确认
  • 定期审查 Agent 的行为日志
  • 不要给 Agent 访问敏感数据的权限

技能安全

OpenClaw 的技能仓库缺乏严格审核。Cisco 的安全团队测试发现，有第三方技能会在用户不知情的情况下窃取数据。

建议：

  • 只安装来源可信的技能
  • 审查技能代码再使用
  • 定期更新和检查已安装的技能

OpenClaw 的一位维护者 Shadow 在 Discord 上警告：”如果你连命令行都不会用，这个项目对你来说太危险了。”

监管动态

2026 年 3 月，中国政府要求国有企业和政府机构禁止在办公电脑上运行 OpenClaw，理由是潜在的安全风险。

这提醒我们：自主 Agent 的能力越强，责任越大。使用时必须谨慎。

OpenClaw 的未来：AI Agent 时代的序章

OpenClaw 的爆火不是偶然。它代表了一个趋势：AI 正在从”回答问题”进化到”主动做事”。

2026 年 2 月，OpenClaw 的创始人 Peter Steinberger 宣布加入 OpenAI，项目将移交给开源基金会。这意味着 OpenClaw 会有更稳定的发展和更广泛的社区支持。

类似的项目还有很多，比如 AutoGPT、LangChain Agents。但 OpenClaw 的独特之处在于：

  • 真正的本地优先
  • 完整的多平台支持
  • 活跃的社区生态
  • 开箱即用的体验

对于开发者来说，现在是入场的好时机。你可以：

  • 贡献代码和技能
  • 基于 OpenClaw 开发商业产品
  • 探索 AI Agent 的各种可能性

对于普通用户，OpenClaw 还需要一定的技术门槛。但随着工具的成熟和文档的完善，未来会有更多人能用上这样的个人 AI 助手。

想象一下，每个人都有一个 24 小时在线的智能助手，能理解你的需求，主动帮你处理事务，还完全在你的控制之下。这不就是我们一直期待的未来吗？
