内容工厂能一键发公众号了，但我故意没让它直接群发

发表于 2026-06-03 更新于 2026-06-25 分类于 产品 阅读次数： 本文字数： 2.3k 阅读时长 ≈ 2 分钟
内容工厂0.28.1版上线公众号一键发布，绑定认证公众号API Key后可把排好版的稿子连封面直接送进草稿箱，但作者刻意不做直接群发，因为订阅号一天只有一次群发且推送不可撤回，把对外发布那不可逆的最后一步留给人自己确认。

哈喽，我是飞飞。

昨天那篇文章的末尾，我老实交代过一件事：一键发布、多平台一稿多发这些功能，我公测前临时下线了，正在打磨，没敢上线。当时还补了句，不想让你满怀期待进来，结果发现没有。

结果今天，它就上线了。

内容工厂 0.28.1 更新，补上了一键发布这条。你在编辑器里写完、配完图，点一下，就能把整篇稿子连排版带封面，直接送进你自己公众号后台的草稿箱。

但我做这个功能的时候，留了个心眼：它只发到草稿箱，不替你直接群发。

昨天还在打磨的那一步，今天接上了

内容工厂原本这条线是顺的。内容雷达帮你盯热点选题，创作页多模型把长文扩出来，编辑器左边写右边手机实时预览，再一键配图。

但走到最后就断了。稿子在内容工厂里排得好好的，你还得手动复制，切到公众号后台，粘进去，再对一遍样式。这一段我自己每天都在走，烦了很久。前面那几步内容工厂都给我包圆了，偏偏卡在最后这道复制粘贴上，每次都像跑完全程，在终点线前被绊了一跤。

0.28.1 就是把最后这段接上。从雷达里看到一个选题，到稿子躺进公众号草稿箱，中间不用再离开内容工厂。说白了，以前是写完自己搬，现在是写完它帮你送进门。

先绑号：填一次 API Key，验证过了就行

要一键发布，得先让内容工厂知道发到哪个号。

进设置，有个新的发布账号页。把你公众号的 API Key 填进去，它会去验一下。验证通过，这个页面就标成已绑定，下面列出你可用的公众号。想换个号或者不想绑了，旁边有换绑和解绑。

我自己拿「飞飞的AI实验室」这个号试的，填完 Key 点验证，几秒钟就绿了。这一步是一次性的，绑过之后就不用再碰，往后每次发文都认这个号。

有一点得提醒：这个能力走的是微信认证公众号的接口，个人主体没认证的号接不进来。这是微信那边定的规矩，内容工厂这边松不了。

写完弹个窗就发，不用再切回后台

绑好号，回到编辑器，真正省事的地方在这儿。

稿子写完、图配好，点发布，弹出来一个窗。里头能选发到哪个号，填封面图、作者、摘要，确认一下标题。都对了，点那个发布。

我上一篇写 Mac 清理的那篇文，就是这么发出去的。在编辑器里改完最后一个字，弹窗里把封面链接一贴，几秒钟，系统回了句发布成功。我切到公众号后台的草稿箱，那篇 Mac 清理就静静躺在那儿，标题、正文、封面全在，连绿黄红那几个小标题、命令行那几段都没乱。

那感觉是真省心。以前我从内容工厂复制一篇长文粘进后台，遇上代码块、加粗、引用多的，样式偶尔会跑位。

上周排一篇带四五段代码的文章，粘过去之后加粗全成了普通字，代码块的灰底也掉了，我对着后台编辑器一处处补，磨了快二十分钟。现在这一段彻底没了。

但它只发到草稿箱，没做直接群发

这是我做这个功能时想得最久的地方。

技术上，让它发完直接群发、推给你所有粉丝，并不难。但我没做，只做到草稿箱这一步就停了。

我做的时候，脑子里过了一遍最坏的画面：我正低头调模型参数，手一抖触发了发布，一篇只写了一半、标题还挂着草稿记号的东西，直接推到了几千个订阅这个号的人手机上。这玩意儿撤都撤不回来。想到这儿，我就把直接群发那条砍了。

把这事掰开，有两层。一层是微信本身的规矩：订阅号群发，一天只有一次机会，而且推出去就收不回来。你想想，要是 AI 哪天抽了风，把一篇还没改完的稿子替你推了出去，这一天的群发就废了，粉丝那头还得平白吃一条尴尬的推送。

另一层是我自己的想法。发到草稿箱，意味着稿子进了你的后台，但推不推、什么时候推，还是你说了算。你可以在微信原生后台再扫一眼排版，确认无误，自己点那一下群发。说到底，那个不可逆的动作，我留在了你手里。

那最后一下，我不敢替你按

写上一篇 Mac 清理的时候，我聊过一个观点：AI 给的建议得可看、可逆、可控，别让它自己闷头跑一条删除命令。

发文章这事，其实是一个道理。

AI 能帮你把选题、写稿、配图、排版、连送进后台这一长串都提速，但「对外发出去」这一下，性质不一样。它面对的是你那几千上万个读者，是你这个号的脸面。这种一锤子的事，我觉得就该攥在人自己手里。

我自己现在的流程是，内容工厂把稿子发进草稿箱，我趁泡水的功夫，在手机上把那条推送预览拉出来，从头到尾扫一遍，确认没问题了，才回后台点群发。多这一步，我心里才踏实。

所以内容工厂的边界我划得很清楚：它负责把稿子做好、送到你后台门口，门里那一下，你自己开。这条线我暂时不打算挪。

还没做完的，我也照实说

发布弹窗里，你可能会瞄到发布类型那栏，还列着「小红书」。

实话讲，这个我得交代清楚：公众号这条我跑通了，小红书那条还在做，现在点了不一定顺。我自己试过把一篇公众号长文原样塞进小红书那条，发出来又臭又长，话题标签也没带上，根本不是小红书该有的调性。这条得单独打磨，急不得。

我没把它藏起来，是想让你知道这是接下来的方向。但你也先别拿它当现成功能用，免得试了发现不对劲，回头怪我虚标。

这是我做产品的一条底线。能用的我才说能用；还在做的，我也不藏着，直接告诉你它还在做。0.28.1 这次，真正能放心用的，就是公众号一键发到草稿箱这一条，我自己天天在用。

打开 aigc.vance.xin，绑上你的号，下一篇写完就能直接送进草稿箱试试。

你会放心让 AI 替你按下群发那一下吗

把群发那一下留给人，是我现在的判断。但我也挺好奇你怎么想。

如果有个工具，能在你设好规则之后，写完自动就把文章群发出去，一个字都不用你再点，你会用吗？还是说，对外发布这种事，你也跟我一样，想把最后那一下留给自己按？

评论区告诉我。你的答案，可能直接决定内容工厂下一步要不要做「定时自动发布」。
