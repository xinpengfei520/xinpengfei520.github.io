微信 ClawBot 来袭：微信终于正式支持原生 OpenClaw 了

发表于 2026-03-22 分类于 AI 本文字数： 3.3k 阅读时长 ≈ 3 分钟

你有没有想过，有一天你可以在微信里直接跟一个 AI Agent 对话，让它帮你查资料、写文档、处理日常琐事？

不用想了。2026 年 3 月 22 日，腾讯正式上线了微信 ClawBot 插件——微信原生支持 OpenClaw。

腾讯公关总监张军在社交媒体上官宣了这个更新。这意味着，全球用户量最大的即时通讯应用，终于拥抱了当下最火的开源 AI Agent 框架。

这件事比你想的要大得多。

OpenClaw 是什么？为什么这么火？

先说背景。

OpenClaw 是由 Peter Steinberger 创建的开源 AI Agent 框架。它的前身叫 Clawdbot，后来改名 Moltbot，最终定名 OpenClaw。截至 2026 年 3 月，它在 GitHub 上拿下了 24.7 万颗星，4.77 万个 Fork——这个数字放在整个开源史上都排得上号。

NVIDIA CEO 黄仁勋称它为”下一个 ChatGPT”。

OpenClaw 之所以火，是因为它解决了一个很实际的问题：让 AI 不只是聊天，而是能真正帮你干活。 它可以操控你的电脑、浏览网页、执行任务，甚至串联多个工具完成复杂工作流。

程序员会消失吗？我和 Claude Code 共事 30 天后的答案

发表于 2026-03-21 更新于 2026-03-22 分类于 观点 本文字数： 3.4k 阅读时长 ≈ 3 分钟

一个月前，我开始全面切换到 Claude Code 作为主力开发工具。

不是偶尔用用，是真的把日常开发的大部分工作交给它——写功能、修 bug、重构代码、写测试、生成文档。我打开终端的次数比打开编辑器多。我写的提示词比写的代码多。

30 天后，我对”程序员会不会消失”这个问题有了一个非常具体的答案。

不会消失。但你认识的那个”程序员”，已经不存在了。

第一周：兴奋，然后是恐慌

第一天用 Claude Code 做了一个完整的功能模块。从 API 设计到数据库迁移到前端组件，两个小时。这个工作以前至少需要一天半。

兴奋了大概 15 分钟。然后一种很奇怪的感觉涌上来——如果一个 AI 两小时就能干完我一天半的活，那我存在的价值是什么？

这不是矫情。这是 2026 年，每一个认真使用 AI 编程工具的程序员，都会在某个时刻问自己的问题。

Business Insider 的一篇报道给这种感觉起了个名字：**”身份危机”**。很多人成为程序员，是因为喜欢”造东西”的感觉。当你的工作从亲手写代码变成监督 AI 写代码，这份工作变得”更简单”了——但也更不让人满足了。
