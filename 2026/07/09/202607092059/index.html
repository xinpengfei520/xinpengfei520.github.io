GPT-5.6 发布，但真正的动作是 OpenAI 把 ChatGPT 改成了工作台

发表于 2026-07-09 更新于 2026-07-14 分类于 观点 阅读次数： 本文字数： 2.9k 阅读时长 ≈ 3 分钟
OpenAI 发布 GPT-5.6 三个能力档位 Sol、Terra、Luna，但更关键的动作是把 Codex 并回 ChatGPT 桌面 App，一个入口里放 Chat、Work、Codex 三种模式，还加了能把成果变网页的站点。信号是 AI 正从聊天框走向能派活的工作台。

哈喽，我是飞飞。

昨天晚上刷推，OpenAI 又发东西了。Sam Altman 甩了条推，说这是他们「有史以来最好的模型」，还配了一篇「写得最好的博文」。点开一看，GPT-5.6 来了，一口气三个模型。

评论区吵的全是跑分、定价、又比 Claude 强了没有。我盯着看了半天，心思却没全落在模型上。真正让我坐直的，是 OpenAI 顺手把 ChatGPT 和 Codex 两个桌面 App 合成了一个，里面塞进三种模式，还多了一个能直接把东西变成网页的入口。

这步棋的分量，不在模型又强了多少，在 OpenAI 悄悄把 ChatGPT 从一个聊天框，改成了一个能派活的工作台。

先说清楚，我是天天用 Claude Code 干活的人，手里十几个 skill 全长在它上面。这回不吹 GPT-5.6 也不踩，就是从我这个隔壁生态重度用户的位置，聊聊 OpenAI 这个产品结构的大动作。最近我写了一串 Claude Code 自己的功能，这回不一样，主角是 OpenAI，Claude 只是我随身带的那把尺子。

真正的动作在那个 App

大部分人这次的注意力，被 GPT-5.6 那三个模型吸走了。这很正常，新旗舰嘛。

但你把 OpenAI 这次发的东西摊开看，会发现模型只是其中一块。同一天他们还干了三件事：把 Codex 那个独立桌面 App 并回了 ChatGPT，给桌面版理出 Chat、Work、Codex 三种模式，再上线一个叫「站点」的功能。

模型是给你换了台更好的引擎，后面这几样才是在改车的形状。我更想聊后面这块，因为它透露的方向，比多几个跑分点有意思得多。

三个模型换了套名字

先把模型这块交代完。GPT-5.6 这次不叫 mini、标准、pro 了，换成三个名字：Sol、Terra、Luna。

Sol 是顶配旗舰，Terra 是中间那档管均衡，Luna 主打最快最便宜。OpenAI 的说法是，这三个名字代表能各自按自己节奏迭代的能力档位。说人话就是，以后不用等一个大版本齐步走，每一档能单独往前跑。

我更新完 App，开屏头一下就弹出这个 Sol 的介绍，说它能扛最复杂的代码改动和最有雄心的活，还特意提了句可以先用低推理强度、遇到难题再往上调。这句我看着挺受用，等于把「省着点用」写进了引导。

价格拉得很开。按每百万 token 算，Sol 输入 5 美元、输出 30 美元；Terra 砍到 2.5 和 15；Luna 更狠，1 美元和 6 美元。免费和 Go 用户默认给 Terra，付费用户能在三档里自己挑，Pro 和企业版还能上更贵的 Sol Pro。

看到这套分档，我立马觉得眼熟。我平时用 Claude 就在手动干这件事：正经活动 Opus，杂活切便宜模型省额度，用之前先在心里过一遍秤。OpenAI 现在等于把这道过秤摆到了台面上，让你自己按活儿挑档。这块我之前专门写过，点一下就够。

一个入口三种模式

真正的重头戏在这。原来 Codex 是个单独的桌面 App，写代码用它，聊天用 ChatGPT，两个东西分开开。

这次 OpenAI 把 Codex 并进了新的 ChatGPT 桌面 App，Mac 和 Windows 都有。一个外壳，里面并排三种模式：Chat 还是那个聊天，Work 管跨应用干活，Codex 继续专门写代码。而且这三种模式，所有计划都能用，包括免费版。

我更新完弹出来的头一屏就是这个，标题直接写「Codex 现已成为 ChatGPT 应用」，底下一句「工作不止于代码」。你看左边那条栏，聊天、站点、插件全排在一起，Codex 就这么被收进了 ChatGPT 的大壳子里。

这个动作看着不起眼，其实挺关键。以前你想让 AI 干活，进门前得先掂量该开哪个 App；现在进来点一下切个模式就行，脑子省了一道弯。

对老用户也留了后路。你要是习惯了 Codex，可以把它设成默认视图，图标还留在 Mac 上，不硬逼你改习惯。

Work 是让你派活不动手

三种模式里，Work 是这次最新、也最能看出野心的一个。

它比普通聊天框走得远得多。OpenAI 给它的定位是跨应用的自主 agent：能从你连上的各种 app 和工作流里扒信息，自己生成表格、幻灯、文档、甚至网页应用，还能把一个复杂项目拆成一小步一小步，自己接着往下干，一干就是几个钟头。背后是 GPT-5.6 在驱动。

顺带说个数字。OpenAI 公布现在每周超过 500 万人在用 Codex，其中超过 100 万是拿它干非写代码的活。这说明让 AI 自己接活这件事，早就不只是程序员的玩具了，Work 就是想把这股势能接到更宽的人群里。

那个站点也是一样的思路。它现在还是公测，能把你在 Work 里做出来的东西，直接变成一个能分享的交互网页或小应用。产出就不用一直躺在对话框里当一段文字，能甩个链接给别人当成品用。

我看 Work 的心情有点复杂。点一下它自己跑几个小时这种爽，我太懂了。但放手 agent 干活的坑，我也踩得够深。

我在一个终端里手动切

说到这我得坦白，新版 App 我更新装上了，上面那两张开屏就是我在自己电脑上截的，但还没正经拿它跑过一轮活。我是 Claude 生态里的人，下面这些是我从自己活法里照出来的对照，不是评测。

OpenAI 这个一个入口切三种模式的设计，我看着特别眼熟。因为我天天在 Claude Code 那个终端里，手动干着几乎一样的事：一会儿切模型，一会儿调不同的 skill，一会儿起一个后台 agent 让它自己跑。我把这套自动化循环分过四档，从我死死盯着的，到完全放手的都有。OpenAI 现在做的，就是把「按活儿切模式」这件我手动在干的事，包装成了一个人人能点的按钮。

具体到写一篇文章，我手里那条流水线就串了十几个 skill：拉选题的、查资料的、写稿的、润色的、配图的、分发的，全靠一个编排 skill 把它们接起来。哪个该自动触发、哪个得我自己手打调用，我脑子里得记着一张表，稍不留神就会漏掉一环。OpenAI 那个模式切换器，说白了就是想把这张表从你脑子里搬到界面上，让你少记点东西。这个省心，我是真认。

但正因为我手动趟过，我对 Work 那种放手几小时格外警惕。我那条最放手的循环，有次部署令牌悄悄过期了，构建每次都成功，只有最后一步认证红了，没人盯着，结果静默失败堆了整整一天，我隔天才发现镜像站挂了。放手 agent 最贵的成本，从来不在它跑得多快，在没人看着的时候它出了错，你多久才知道。Work 让人派完活就走的门槛越低，这个问题就越得先想清楚。

从对话框走向工作台

绕了一圈，回到开头那句。

大家盯着 GPT-5.6 比 Claude 强了几个点，但 OpenAI 这次真正在改的，是 AI 产品的形状。从一个你问它答的聊天框，往一个你派活它干的工作台挪。三种模式一个入口、Work 能连干几小时、站点把成果直接变网页，这三样拼在一起，指的是同一个方向。

这个方向对不对，我现在下不了结论。作为一个在 Claude 这边把活法磨顺了的人，我甚至有点抵触把这么多模式塞进一个 App，我更习惯自己在终端里一样样拼。但我也承认，对大多数不想折腾的人，一个入口切模式确实省心，OpenAI 这步棋踩的是更大的那块地。

我会继续在 Claude 这边盯着看，看这套工作台到底是让更多普通人真用起了 agent，还是又一轮把功能堆进一个 App 的热闹。

你更喜欢哪种活法：像 OpenAI 这样一个 App 里切模式，还是自己在终端里一样样手动拼？放手让 agent 自己跑几个小时的活，你现在敢交给它吗？
