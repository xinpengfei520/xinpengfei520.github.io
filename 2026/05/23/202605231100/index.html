I/O 2026 ICYMI：除了 Antigravity 2.0，Google 把 idea 到 production 这条链全填上了

发表于 2026-05-23 更新于 2026-06-18 分类于 AI 阅读次数： 本文字数： 5.5k 阅读时长 ≈ 5 分钟
I/O 2026 keynote 不只是 Antigravity 2.0 那一个事。AI Studio 手机 app、Antigravity SDK、Managed Agents API、Android CLI、WebMCP、Chrome DevTools for agents、Science Skills，Google 这次把开发者从 idea 起点到生产终点整条链都占位了。每一节我按"链路位置 + 对国内开发者的实操价值"梳一遍。

哈喽，我是飞飞。

前两天我刚写过 Antigravity 拆成 IDE 和 2.0 这件事，本来以为 I/O 2026 keynote 我已经看明白了。

周末重新把 keynote 录像拉了一遍，又把 developers.googleblog.com 上当天发的 9 篇配套博文一篇一篇过完，我发现自己看漏了一大半。

Antigravity 拆 5 个 surface 那件事只是棋盘中心一格。

外围 Google 还动了 AI Studio、Managed Agents API、Android CLI、WebMCP、Chrome DevTools、Science Skills，至少 7 个方向。

说白了，Google 这次不是发了一个产品，是把开发者**从”手机上想到一个 idea”到”agent 在生产环境跑”**整条链的所有空位都填上了。

这一篇 ICYMI 我按”链路位置 + 对国内开发者的实操价值”把每个新东西梳一遍。

链路顺序我画一下:AI Studio（抓 idea）→ Antigravity IDE/2.0（写代码）→ Antigravity SDK（自部署 agent）→ Managed Agents API（产品上线）→ Android CLI / WebMCP / Chrome DevTools（移动+web 补齐）→ Science Skills（垂直领域 skill 包）。

AI Studio 这一头：从手机上把 idea 直接接进 Antigravity

AI Studio 这次的更新我个人觉得比 Antigravity 2.0 更值得国内开发者关心。

它现在干了 4 件事一起放出来。

原生 Kotlin Android 支持:在 aistudio.google.com 里 prompt 一句话直接生成完整 Android app 代码，Workspace API 能直接调，Cloud Run 一键部署，Firebase 服务原生接，甚至能从 AI Studio 直接发布到 Google Play 的 Test Track。我自己测了一下，让它生成一个”用 Gemini API 做翻译的 Android demo”，从 prompt 到模拟器跑起来不到 5 分钟。

手机 app 即将上线:iOS 和 Android 版的 AI Studio 移动 app 这周开放预注册，Google 自己描述是”路上想到 idea 直接抓住，到桌面打开就有一个能用的原型”。这对内容创作者 / PM 这种”灵感跑得比键盘快”的人是个真东西。

导出到 Antigravity:在 AI Studio 里写到一半的项目可以一键导出到本地的 Antigravity，包括所有上下文。这点 Google 在 keynote 里没强调，但其实是新链路的关键。它把 AI Studio 从孤岛实验场变成了一条进 IDE 的正式入口。

Workspace API 原生集成:agent 可以直接调 Gmail、Calendar、Drive、Docs 这些 Workspace API，不需要再走 OAuth dance。做企业内部工具的同行，这一项直接省掉了一大堆鉴权重复劳动。

我自己接下来要试的就是 AI Studio app 装到手机上，看它能不能让我把通勤路上想到的 idea 在到家之前就跑出原型。

Antigravity SDK 是给”想自部署”的人留的口子

Antigravity 2.0 desktop app 适合普通开发者，但有一批人是不希望 agent 跑在 Google 服务器上的。

国内做企业内部工具的、外贸 ERP 的、合规要求高的 SaaS 团队都属于这一批。

Antigravity SDK 就是给这批人留的口子。

它把 Google 自己产品里在用的同一套 agent harness（plan / tool-use / sandboxing / git policy / credential masking）抽成 SDK，你可以装到自己的服务器或者 Kubernetes 集群里跑。

实话讲，SDK 这条线的真正价值不在”开源”。它仍然是闭源 SDK，调用 Gemini API。

真正价值是”你可以在 prompt / tool 注册 / memory 这一层完全自定义”。

OpenAI 的 Codex CLI 走的是类似路线但更轻；Anthropic 的 Claude Agent SDK 这边的位置类似但 harness 更”硬”（自带 Hooks 系统）。

Google 这次拉出 SDK，等于明确给开发者一个选择:要么用 Antigravity 桌面应用、要么用 SDK 自部署，但 agent harness 是同一套。

Managed Agents in Gemini API 是这次最重要的一颗棋

如果非要选 I/O 2026 一个”对未来 12 个月最重要”的发布，我会选 Managed Agents。

简单讲，它干的事是:一次 API 调用，你就拿到一个已经配好工具、已经隔离好沙箱、已经能联网、能跑代码的 agent。

这一个 endpoint 后面起一个 Linux 容器，里面装好 Antigravity agent harness，挂载 sandboxed shell + file system + browser，每次 interaction 是一个可恢复的 session，files 和 state 都留着，下一次 API 调用接着上一次跑。

底下默认是 Gemini 3.5 Flash。

Google 之前发布的 Deep Research 就是基于这个架构跑的，这次是把它开放给所有开发者。

本质上这是 OpenAI Assistants API + Code Interpreter + Browser 三件事打包，但比 OpenAI 那一套整合度更高（一次调用拿全套，不需要先创建 assistant 再 attach tools）。

对国内 SaaS 团队的实操价值很直接:你想做”输入一段需求让 agent 跑出来一个数据分析报告 + 生成 PPT”这种产品，原来你要自己维护 Linux 沙箱 / Docker orchestration / 浏览器自动化 / 文件 IO 安全策略，现在一个 API 全交给 Google。

定义自己 agent 的方式也很轻量:写两个 markdown 文件，AGENTS.md（告诉 agent 它是谁 / 怎么思考）和 SKILL.md（告诉它有哪些工具），注册一下就能跑。

跟 Claude Agent Skills 的格式几乎完全对齐。

Android CLI 1.0 把 Android Studio 的能力开给所有 agent

这是给 Android 开发者的一颗糖。

Android CLI 是一个独立 binary，可以让任何 agent（Claude Code / Codex CLI / Antigravity CLI 都行）直接调用 Android Studio 里的能力。

包括 android studio analyze-file（用 IDE 的静态分析跑代码诊断）、android studio find-declaration（找符号定义）、android studio render-compose-preview（让 agent 真的渲染出 Compose UI 预览图）。

这件事真的反直觉:Android Studio 是 Google 自家 IDE，但 Android CLI 这次明确支持 Claude Code 和 OpenAI Codex CLI 调用。

Google 这次没有把 Android Studio 锁死给 Antigravity，是开放给所有 agent 的。

包管理也铺得很全:apt-get、winget、homebrew 都能装，国内开发者最容易踩坑的”装不上”问题这次基本不会有。

同时开源了一套 Android skill，覆盖 Jetpack Compose 迁移、CameraX 迁移、Display Glasses + Compose Glimmer for XR、Adaptive UI 等场景，github.com/android/skills 上能看全清单。

对国内做 Android app 的同行最大变化:不需要再用 Gemini in Android Studio，可以保留你熟悉的 Claude Code / Cursor 工作流，只是多加一个 android 命令让 agent 调。

WebMCP 在 Chrome 149 上 origin trial 开了

WebMCP 是 web 这边今年最值得跟的一件事。

它的设计想法是:网站作者主动暴露一组结构化工具给浏览器里的 AI agent 用，而不是让 agent 像现在一样去 click 来 click 去模拟用户。

比如 Expedia 把”搜索航班 / 选座 / 改签”做成 WebMCP tool，你在 Chrome 里跟 Gemini 说”帮我订一张周六回北京的票”，Gemini 直接调 Expedia 的 backend API，不用模拟点击界面。

Chrome 149 已经开了 origin trial，Expedia、Booking.com、Shopify、TurboTax、Redfin、Etsy、Instacart、Target 都在实验阶段。

这是开放 web 标准，Anthropic 那边的 Claude for Chrome 应该也会支持。

对国内 web 开发者的具体动作:你的网站如果想被 agent 友好访问，明年这个时候就要开始想 WebMCP tool 的设计了，跟 SEO 一样会变成”agent-SEO”标配。

Chrome 那边同时还放了 Chrome DevTools for agents。agent 可以直接 attach 到 DevTools，拿 console log、network trace、accessibility tree 来做自主 debug。

Google 在 keynote 里给了一个数据，日本 LY Corporation 用这个做自动 performance audit，节省了 96-98% 的手工分析时间。

Science Skills:DeepMind 把领域知识做成 agent skill 包

Science Skills 这条容易被忽略但其实是个未来 anchor。

DeepMind 这次开源了一个针对生物 / 医学 / 化学领域的 agent skill 包，把 30 多个数据库（UniProt、AlphaFold Database、AlphaGenome API、InterPro 等）打包成 skill，agent 可以直接调。

效果是把”一个结构生物信息学分析”从原来的几小时人工压到几分钟全自动。

Google 自己内部测试时用 Science Skills 跑了一个 AK2 基因突变的疾病机理分析，给出了新的潜在机理推断。

Science Skills 本身只是起点，关键是 Google 用它开了”specialized skills bundles”这个产品形态。下一波很可能是 Medical Skills、Legal Skills、Finance Skills、Game Dev Skills。

每一个 skill bundle 都是 Google 跟领域专家合作做的”垂直 agent capability pack”。

国内做垂类 AI 应用的团队应该认真看这个范式，它是 horizontal model + vertical skill pack 的一个 reference 实现。

拼起来看，这次 Google 在卷的是 idea 到 production 整条链

把上面 6 个发布拼起来看，链路就清楚了。

手机抓 idea:AI Studio 移动 app

桌面写代码:Antigravity IDE（写）+ Antigravity 2.0（agent 跑）

自部署 agent:Antigravity SDK

生产上线:Managed Agents API

移动场景补齐:Android CLI 给所有 agent 用

Web 场景补齐:WebMCP + Chrome DevTools for agents

垂直深度:Science Skills 和后续 specialized bundles

这次 Google 干的事情远不止补一个产品的缺。整条链上的每个节点都被填了一个自己的产品占位。

这跟 OpenAI 一直只在 API + ChatGPT + Codex 这三个点上加码、Anthropic 在 Claude.ai + Claude Code + Claude API 三件套上往深走的策略完全不一样。

不夸张地说，Google 这次把”端到端 stack”这张牌打到最满。

要不要押 Google 这条链是各人选择，但链路完整这件事本身已经是个值得国内开发者认真考虑的信号。agent-first 时代的开发栈不会停在某一个工具上，整条链都要一起重画。

今晚跑 Managed Agents demo，明早注册 WebMCP origin trial

发到生产之前我自己还要再试两件事。

一个是 Managed Agents API，我会用 Interactions API 起一个 agent 让它跑”读我桌面 desktop 上一个 CSV 然后生成一份 markdown 总结”这个最简单的 demo，看每次 API 调用的延迟、Token 消耗、session resume 是不是真的能跨调用保留状态。

另一个是 WebMCP origin trial 注册申请，我有一个之前给读者做的小工具站，想把”搜索文章 / 复制摘要 / 导出 markdown” 三个动作暴露成 WebMCP tool 试一下，看 Chrome 149 + Gemini in Chrome 调用起来手感怎样。

两件事跑完之后我会再写一篇实测。
