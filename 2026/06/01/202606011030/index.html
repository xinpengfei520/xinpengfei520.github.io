Copilot 把包月改成按 token 计费惹翻开发者，我这种早跳车的来算笔账

发表于 2026-06-01 更新于 2026-06-24 分类于 观点 阅读次数： 本文字数： 3k 阅读时长 ≈ 3 分钟
GitHub Copilot自6月1日起从包月改成按token计费，10美元Pro变成1000个AI信用点用完即停，有用户账单预估从39美元暴涨到902美元，作者对比Claude Code、Cursor等订阅加配额模式的确定性优势，建议先把超额预算上限设为0并学会为算力做预算。

哈喽，我是飞飞。

这两天我刷 GitHub 社区和 X，老看到同一类截图在转：一个开发者把官方的账单预览工具点开，四月份按老规矩算是 39 美元，换成新规则重新一算，902 美元。差不多翻了二十三倍。评论区第一句火出圈的话是「What a joke」，开个什么玩笑。

事情的起因是 GitHub Copilot 从今天，6 月 1 号开始，把计费方式从「包月固定价」改成了「按 token 用多少算多少」。我装 Copilot 那会儿还是它刚火的时候，从那之后我陆陆续续把各家 AI 编程工具的定价都摸了一遍。所以这事我想跟你好好聊聊。先把账算明白，再说说我自己为什么庆幸早早换了阵营。

先把这笔账算明白，Copilot 到底改了什么

老规矩很简单。Copilot Pro 一个月 10 美元，你这个月写多少代码、跟它聊多少次，都是这个价。心里有底。

新规矩换了个内核。还是 10 美元，但这 10 美元现在变成了 1000 个「AI 信用点」，1 个点等于 1 美分。你每发一次请求，它按你这次烧掉的 token 扣点。

token 你可以理解成模型处理文字的最小计价单位。你发进去的、它吐出来的、缓存住的，全算钱。模型越贵，同样一句话扣得越狠。那种会「深度思考」的推理模型，单价能比普通模型高十几倍到几十倍。

套餐价格表面上一分没涨。Pro 还是 10 美元，Pro+ 还是 39，企业版还是那些数字。变的是这笔钱买到的东西，从「一段时间随便用」变成了「一笔额度，用完即止」。点数烧完，Copilot 直接停下不干了，除非你自己手动去后台开「超额预算」。

有一条值得说清楚：最基础的代码补全和下一处编辑建议，还是免费的，不计点。当年让 Copilot 火起来的就是这个边敲边跳出灰字的功能，这块没动。被计价的是那些会自己跑、自己改、自己折腾半天的 agent 能力。

晒天价账单的人冤不冤，这事得分两头说

先站到 GitHub 那一边替它说句话。

那些晒出 50 美元变 3000、39 变 902 的人，里面确实有一批是在「凭感觉糊代码」。开一个 agent 任务，让它自己跑几个小时、自己派生出几十上百个子任务在那空转，token 哗哗地烧。这种用法在老规矩下平台替你兜着，换成按量计费，账单爆炸一点都不冤。

微软给 Copilot 这套订阅到底垫了多少钱，外人看不到。但能让人这么造，本身就不是个长久的生意。

可话又说回来，锅也不全在用户。是微软自己这几年一个劲儿地鼓励大家放开了用，把烧 token 这件事做得越来越顺手，一个请求点下去能跑半天。大家都是顺着它铺好的路在走，现在突然把计价器装上了。社区那条公告下面 400 多条评论、将近 900 个踩，骂的就是这个「先请你上车，开到一半再装计价器」。

我自己的判断是，一次正经的 agent 编程任务，烧掉三四十美元的点数很常见，那是 Pro 用户一个月额度的三四倍。也就是说，你认真干一天活，月额度就见底了。这才是大家真正慌的地方。

订阅制和按 token，差的根本不是钱

聊到这我得说个我琢磨了好几天的想法：订阅制和按 token，差的不是价格，是确定性。

我装 Copilot 那阵子，10 美元一个月，账单页面我从来没点开过。没必要看，这个月不管我写到几点、改了多少版，月底都是固定那个数。这份「不用算账」的安心，才是我那 10 美元真正买到的东西。说白了，平台帮我把忽高忽低的成本波动给摊平了。

按 token 之后，这份安心没了。它把成本的不确定性原封不动地塞回到了你手里。从此你每按一次回车，心里都得过一道秤：这条指令值不值这点点数，要不要换个便宜的模型，这个 agent 任务跑出去会不会又是几十美元。说真的，多花的那点钱倒还在其次。真正累人的，是这杆悬在头顶、按一次掂量一次的秤。

我用 Antigravity 那段时间也算超过额度，但体感完全不一样。配额每周刷新，超了顶多给我降速，我骂一句等明天接着用就完了。我从来没担心过会突然收到一张计划外的大额账单。这种「最坏也就是慢一点」的踏实，跟「最坏可能是九百刀」的提心吊胆，是两种心情。

为什么我还赖在「订阅 + 配额」这边不走

我主力是 Claude Code，走的是 Claude 的订阅。Pro 20 美元一个月，往上是 Max，100 或者 200。它也有用量限制，但那个限制管的是「这段时间内你能用多少」，而不会真金白银从你卡里一笔笔扣。用超了就等窗口刷新，账单永远是那个固定的订阅数。对我这种需要心里有数的人，这点太重要了。

Cursor 我也常开，$20 一档，往上 60、200。它官方文档自己都写了，每天重度用 agent 的人，实际花销更接近每月 60 到 100，入门那个 20 根本打不住。但好歹，它的贵是「订阅档位的贵」，你选定一档，这个月封顶就是这个数，不会失控。

Codex 跟着 ChatGPT 的订阅走，不过它其实比 Copilot 还早一步，今年四月初就转成了按 token 的算法。所以这不是 Copilot 一家在试水。

把这几家排一排，分界线就清楚了。Claude Code、Cursor、还有 Codex 的订阅档，本质上还是「订阅加配额」这一挂的：用超了，要么降速，要么等下个周期，账单稳稳的。Copilot 新规矩是另一挂：固定额度之外，多用一点就按 token 实打实扣钱。这两挂给人的体感，差着一整个量级的安全感。所以这次我没有任何要回 Copilot 的念头，反而更庆幸早就在订阅这边扎下了根。

说到底，是 AI 的免费午餐吃到头了

骂归骂，说实话，我倒不觉得微软是单纯想多挣钱。这背后有件更大的事在发生。

前面说了 Codex 四月就转了 token。再往外看，《华尔街日报》报道说，美国不少公司因为 AI 成本涨得太凶，已经开始给员工的 AI 用量「配给」了，定额度、分层审批，跟当年发饭票似的。你把这些信号串起来看，方向就一个：AI 工具正在从「赔本补贴抢用户」的阶段，回到「该花多少成本就收多少钱」的阶段。

OpenAI、Anthropic、Google 的 API 一直都是按 token 收费的，这不新鲜。新鲜的是，这套算力的真实成本，第一次这么直白地摆到了普通开发者的月账单上。我们是最先体感到「token 是要花钱的」这批人。Copilot 不过是第一个把计价器装到你面前、还装得这么显眼的而已。

所以与其纠结要不要换工具，不如先把这门以前不用上的功课补上：学会为算力做预算。

具体到马上能做的，有这么几件。

先去后台把超额预算的上限设成 0。这样点数一烧完它就停，绝不会偷偷给你刷出一张大额账单。这是最稳的一道保险丝。

再就是分清活儿的轻重。免费补全能搞定的小事，别动不动就开 agent 烧点数；大改造、跨文件重构这种硬活，才值得放它出去跑一趟。

要是你天天靠 agent 干活，那就认真算一笔。回到 Claude Code 或者 Cursor 这种订阅封顶的阵营，可能比盯着 Copilot 的计价器过日子舒服得多。

写在最后

绕了一圈再回到开头那张账单截图。39 变 902 这个数字是挺吓人，可真正变的东西，是我们头上凭空多出来的那杆秤：从此每按一次回车都得掂量一下。包月时代那种「写到爽、月底一个价」的日子，大概是真的要翻篇了。

我会接着盯各家后面怎么走，尤其是 Claude 和 Cursor 会不会也顶不住成本，慢慢往按量那边挪。要是连订阅封顶这条退路都没了，那才是真要重新算大账的时候。

最后想问问还在车上的你：你现在主力用的是哪家，是已经被按 token 的账单吓到了，还是跟我一样躲在订阅封顶这边？评论区报个数，我们对对账。
