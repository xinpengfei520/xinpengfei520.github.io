腾讯 WorkBuddy 来了：是 OpenClaw 套壳，还是大厂的正确打法？

发表于 2026-03-23 更新于 2026-04-16 分类于 产品 阅读次数： 本文字数： 3.3k 阅读时长 ≈ 3 分钟

3 月 9 日，腾讯深圳总部门口排了近千人的长队。不是抢演唱会门票，是排队免费装 OpenClaw。

队伍里有背双肩包的程序员，有穿校服的小学生，还有退休的航空工程师。

就在”全民养虾”的狂热还没散去的几天后，腾讯一天之内连发两款产品：先是自研的 WorkBuddy 正式上线，随后 QClaw 内测消息曝光。

社区立刻炸了锅。支持的人说：”大厂终于下场了，体验比原版好。”质疑的人说：”这不就是 OpenClaw 套壳吗？”

到底是套壳还是自研？这个问题背后，藏着一场关于 AI Agent 入口的巨大博弈。

先回答核心问题：到底是不是套壳？

爱范儿对 WorkBuddy 负责人汪晟杰做了一次采访。他的回答非常直接：

“百分之百自研，没有用过任何一行 OpenClaw 的源码。”

他解释说，腾讯去年下半年就已经推出了 AI 编程产品 CodeBuddy，WorkBuddy 正是搭建在 CodeBuddy 的产品架构之上。早在 1 月 17 日——那是一个周末——他和几个工程师通宵两天，把 0.01 版本搞上线给内部使用。

那时候 OpenClaw 还没有在中国爆火。

2 月 6 日，WorkBuddy 启动内测。3 月 9 日正式上线——恰好赶上了 OpenClaw 的全民热潮。

时间线说明了一些事情：WorkBuddy 不是看到 OpenClaw 火了才临时赶工的产品。但 OpenClaw 的爆火，确实为它提供了一个完美的上线时机。

36 氪的一篇文章用了一个很精准的说法：

“正如 OpenClaw 是基于 AI 编程的基础做的一个智能体产品，WorkBuddy 正是基于’国产类 Cursor 产品’ CodeBuddy 的基础，做的’龙虾化’改造。”

所以准确地说：WorkBuddy 不是 OpenClaw 的代码套壳，而是功能对标、生态兼容。 它走的是另一条技术路线，但对用户来说，干的是差不多的事。

WorkBuddy 到底能干什么？

身世聊完了，说说这东西到底能干什么。

WorkBuddy 官方定位是”全场景职场 AI 智能体桌面工作台”——说白了就是一个能听懂指令、操控你电脑的 AI 员工。

我装了之后试了一圈，几个能力值得提：OpenClaw 社区的 skills 可以直接导进来用，内置了 20 多个技能包（发票处理、报告生成、数据分析这些），支持 MCP 协议接外部工具，而且模型可以切换——混元、DeepSeek、GLM、Kimi、MiniMax 随便选，不绑死一家。最让我眼前一亮的是多端遥控：企业微信、QQ、飞书、钉钉都能连，出门在外掏出手机发条消息，它就在你办公室的电脑上自动干活。

跟原版 OpenClaw 相比，WorkBuddy 有两个明显的差异点：

第一，不需要部署。 OpenClaw 的云端部署对普通人来说门槛不低，你得会用终端、会配置环境。WorkBuddy 直接下载安装就能用。

第二，安全机制不同。 OpenClaw 的工作原理是”透传”——它直接操控你的系统，信息可能暴露在公网上。WorkBuddy 用的是自研的沙箱机制，任务在隔离环境中执行。汪晟杰说他们”刻意规避了 OpenClaw 的透传工作原理”。

这一点很重要。CrowdStrike 曾警告说，配置不当的 OpenClaw 会暴露完整的配置文件，包括所有凭证。SecurityScorecard 的数据显示，全球有超过 13.5 万个 OpenClaw 实例 暴露在公网上。安全，是 WorkBuddy 最想打的差异化牌。

市场反应：暴涨与质疑并存

WorkBuddy 上线当天，腾讯股价飙升 **7.3%**，创年内最高单日涨幅，单日市值暴涨 3400 亿港元。花旗银行发报告说，这”可能代表中国从’聊天 AI’向’执行 AI’的重大转变”。

但硬币有另一面。

上线当晚，WorkBuddy 就扛不住了。用户访问量远超预期，核心服务压力过大，登录和服务不稳定。CodeBuddy 团队紧急发布致歉信，宣布紧急扩容 10 倍。

36 氪的测评也很实在：

“作为一个’一周上线’的产品，腾讯交出了及格的答卷。好处是，它帮没有使用龙虾经验的用户开辟出实际应用场景。缺点是在稳定使用和体验细节的优化上，还有太多能够打磨的空间。”

另一篇 36 氪文章则提出了更尖锐的质疑：

“OpenClaw 本身是 MIT 协议的开源项目，任何人都能自由使用。腾讯的 WorkBuddy、飞书的插件、MiniMax 的 MaxClaw，本质上都在开源项目的基础上，加个壳做产品化封装而已。回顾历史，这种大厂套壳开源的结局，往往不太好看。”

这就是 WorkBuddy 面临的核心争议：你说你是自研，但你兼容 OpenClaw 的生态、对标 OpenClaw 的功能、赶着 OpenClaw 的热度上线。用户分不清楚，也不在乎——他们只关心好不好用。

腾讯的龙虾全家桶

WorkBuddy 不是腾讯在 AI Agent 领域的唯一动作。马化腾在朋友圈里亲自预告了完整阵容：

“自研龙虾、本地虾、云端虾、企业虾、云桌面虾、安全隔离虾房、云保安、知识库……还有一批产品陆续赶来。”

目前已经亮相的产品矩阵：

产品 定位 状态
WorkBuddy 桌面办公 AI Agent 已上线
QClaw 一键部署 + 微信/QQ 直连 内测中
ClawBot 微信原生 OpenClaw 插件 灰度中
Lighthouse 开发者云端部署方案 已上线
SkillHub 技能市场 + 安全审核 已上线

拆一下：QClaw 和 ClawBot 负责拉人——微信的流量管道直接灌进来。WorkBuddy 负责留人——企业级体验把人留住。Lighthouse 给开发者用，SkillHub 做技能市场。消费端、企业端、开发者，每个口子都堵上了。

大厂”捞虾”背后的真正战争

腾讯不是唯一在抢这个位置的大厂。阿里推了企业级 AI Agent 平台”悟空”，百度直接在搜索 APP 里集成了 OpenClaw 给 7 亿用户用，字节和智谱升级模型支持 Agent 工作流，小米搞了 Miclaw Agent，华为弄了”鸿蒙版龙虾”小艺 Claw。

一窝蜂地下场，抢的是同一样东西：AI Agent 的入口。

这一波里我最看好腾讯和百度。原因很简单——腾讯有微信，百度有搜索，这两个是真正的日活入口，13 亿用户每天打开无数次。把 AI Agent 塞进微信，等于在最大的流量池里养鱼。阿里的”悟空”和小米的 Miclaw，说实话我还没看到差异化在哪里，更像是怕错过就先占个坑。

这跟十年前抢移动互联网入口是一回事。当年争的是”用户打开手机第一个用什么 APP”，现在争的是”用户跟 AI 对话的第一个界面是什么”。

值不值得现在就用？

我自己的体验是：发票处理技能确实开箱即用，比在 OpenClaw 上折腾 MCP 配置快多了；但报告生成跑到一半卡过一次，重启之后进度全丢。整体来说，处理结构化的、步骤少的任务已经可以用了，复杂长链路任务还不太稳。

如果你之前觉得 OpenClaw 部署太麻烦，WorkBuddy 确实降低了门槛——直接去官网下载安装就行。开发者可以关注 MCP 协议和 SkillHub 生态，你给 OpenClaw 写的 skills 能无缝跑在 WorkBuddy 上，不会被锁死。企业用户的话，沙箱安全机制确实比裸跑 OpenClaw 靠谱，但上线当天紧急扩容 10 倍这件事本身就说明了它的成熟度——先在非核心业务上试水比较稳。

写在最后

套壳还是自研？这个问题，其实可能问错了。

更好的问题是：在 AI Agent 时代，”自研”还重要吗？

OpenClaw 是开源的。它的价值不在于代码本身，而在于它定义了一种新的交互范式——让 AI 从”聊天”变成”干活”。谁能把这个范式做得更好用、更安全、更适合中国用户，谁就能赢。

腾讯龙虾负责人说了一句话我觉得很值得玩味：

“龙虾不是一个产品，龙虾是一种概念。而且，龙虾不等于 OpenClaw。”

不管你信不信这句话，有一件事挺清楚的：WorkBuddy 会活下来，但不是因为它比 OpenClaw 强，而是因为它背后站着微信。 在中国市场，渠道就是产品。腾讯总部门口那条千人长队，和上线当天暴涨的请求量，说的都是同一件事——人们想要一个能帮自己干活的 AI，而且他们想要的是现在。
