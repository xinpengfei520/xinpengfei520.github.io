Cursor 3 把 IDE 变成了备用选项，开发者炸了

发表于 2026-04-08 更新于 2026-04-13 分类于 工具 阅读次数： 本文字数： 2.6k 阅读时长 ≈ 2 分钟

大家好，我是飞飞。

上周 Cursor 发布了 3.0，我第一时间升级体验了一下。打开之后愣了几秒——那个熟悉的文件树不见了，取而代之的是一个看起来像任务调度台的界面。

这不是一次普通的版本更新。

IDE 的位置变了

Cursor 3 最核心的变化，是把”智能体控制台”（Agents Window）设为默认界面，把传统的代码编辑器降级成了一个可以随时切换回去的”备用模式”。

官方的说法是：**”centered around agents, while keeping the depth of a development environment”**。

翻译成大白话就是：我们认为你以后大部分时间是在调度 Agent、审查代码，而不是自己一行一行地写。编辑器还在，但它不再是主角了。

这个判断是对的，还是太激进了？我觉得值得好好聊聊。

Cursor 3 到底发布了什么

先说具体功能，不然容易被标题带跑偏。

Agents Window（智能体窗口）是这次的核心。你可以同时跑多个 Agent，跨多个代码仓库，跑在本地、worktree、云端、远程 SSH 上都行。所有 Agent 的状态统一显示在一个侧边栏里——包括从手机、网页、Slack、GitHub、Linear 触发的任务，全都汇聚在这里。

Cloud Handoff（云端接力）是我觉得最实用的功能。你在本地跑着一个 Agent 任务，中途要关电脑，可以把这个 session 无缝转移到 Cursor 的云端继续跑。等你回来，再把它拉回本地做最后的调试和测试。反过来也行——云端开始，本地收尾。

Design Mode（设计模式）是给前端开发者的。在 Agents Window 里，你可以直接在浏览器里框选 UI 元素，告诉 Agent “就改这里”，不用再费劲描述”左上角那个按钮”。

Agent Tabs（多标签并排）让你可以同时看多个 Agent 的对话，侧边并排或者网格布局都支持。

还有一个背景信息值得提：Cursor 在 3 月份还发布了自研模型 Composer 2，基于 Moonshot AI 的 Kimi K2.5 微调，在他们自己的 CursorBench 上跑出了 61.3 分，比 Claude Opus 4.6 的 58.2 分高一点。当然，这是 Cursor 自己的评测，参考就好。

为什么是现在

这次发布的时机很有意思。

Cursor 的年化收入在 2026 年 2 月刚刚突破 20 亿美元，三个月内翻了一倍。听起来很猛，但同期 Claude Code 的年化收入是 25 亿美元，企业客户超过 30 万家。

更扎心的是，社区里开始出现大量”从 Cursor 换到 Claude Code”的帖子。有 Cursor 的投资人公开说，他投资组合里的几家创业公司已经在和 Cursor 脱钩了。

然后 Cursor 在一个月内连发三个大动作：3 月 5 日发布 Automations（自动化触发 Agent），3 月 19 日发布 Composer 2，3 月底开放自托管云端 Agent，4 月 2 日发布 Cursor 3。

这个节奏，不像是按计划推进，更像是在应对压力。

The New Stack 的一篇分析说得很直接：**”这是一家年化 20 亿美元的公司，相信自己的品类正在被重新定义，然后把整个产品界面都押上去了。”**

各家的路线分歧

Cursor 3 发布之后，我重新梳理了一下各家的思路，发现大家对”Agent 控制台该放在哪里”这个问题，给出了完全不同的答案——而且分歧大得出乎意料。

Anthropic 的选择是最极端的：没有 IDE，直接用终端。Claude Code 的控制台就是命令行，后来加了网页版和桌面 App，但核心还是 CLI。OpenAI 走了另一个极端，Codex 到处都是——桌面 App、CLI、VS Code 插件、网页版，没有一个固定的”主界面”，你在哪里都能调度 Agent。

Google 的 Antigravity 跟 Cursor 3 最像，也是在 IDE 内部做双模式：Editor View 写代码，Manager Surface 调度 Agent，两个视图平等共存。Cursor 3 比 Google 更激进一步——Agent 控制台是默认，编辑器是备用。

The New Stack 用了一个比喻我觉得很贴切：云计算时代，AWS 控制台出来之后，没人停止用 SSH，但 SSH 变成了偶尔调试用的工具，控制台才是做决策的地方。

Cursor 3 在赌的是：代码编辑器会变成那个”偶尔用的 SSH”。

开发者的反应

Hacker News 上 Cursor 3 的帖子，置顶评论不是在夸功能，而是一句话：

“I wish they’d keep the old philosophy of letting the developer drive and the agent assist. I still want to code, not vibe my way through tickets.”

Cursor 的工程师当天就回复了：IDE 还在，随时可以切回去。

这个反应很能说明问题。有一批开发者，他们用 Cursor 是因为它是一个好用的 AI 辅助编辑器，不是因为他们想要一个 Agent 调度台。Cursor 3 的默认界面变了，对这批人来说是一种冒犯。

但也有另一批人——尤其是在做大型项目、需要同时跑多个任务的开发者——他们等这个功能等了很久了。

两种需求都真实存在。Cursor 的选择是：把 Agent 控制台设为默认，把编辑器留着但降级。这个选择会不会流失一部分用户，现在还不好说。

我的感受

用了几天之后，我的感受是：Cloud Handoff 是真的好用，Agents Window 需要适应期。

Cloud Handoff 解决了一个我之前一直有的痛点——跑一个耗时的 Agent 任务，不能关电脑。现在可以了。这个功能单独就值得升级。

Agents Window 的并行 Agent 管理，理论上很强，但实际用起来需要你先想清楚”我要同时跑哪几个任务”。如果你还在单任务模式下工作，这个界面反而会让你觉得信息过载。

我觉得 Cursor 3 代表的方向是对的——未来的开发工作确实会越来越像”调度 Agent + 审查输出”，而不是”自己写每一行代码”。但这个未来到底有多近，每个人的感受不一样。

现在还在单任务模式下工作的人，打开 Agents Window 大概率会觉得信息过载，切回 IDE 就好了。已经在同时跑多个 Agent 的人，这次升级是真的值得——尤其是 Cloud Handoff，解决了一个我之前一直有的痛点：跑一个耗时任务不能关电脑。现在可以了。

你现在用的是哪个 AI 编程工具？有没有考虑过从 Cursor 切换到 Claude Code，或者反过来？评论区聊聊，我很好奇大家现在的工作流是什么样的。
