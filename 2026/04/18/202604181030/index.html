Anthropic 深夜发了 Claude Design，Figma 盘中跌了 7%

发表于 2026-04-18 更新于 2026-04-24 分类于 观点 阅读次数： 本文字数： 5.1k 阅读时长 ≈ 5 分钟

大家好，我是飞飞。

昨晚 23 点多我本来在收尾一篇稿子，顺手刷了下 X。Anthropic 官号在 9 分钟前发了一条推——“Introducing Claude Design”。我点进去的时候，官网的演示视频还没加载完。

北京时间 2026 年 4 月 17 日 23:00，旧金山早上 8 点，Anthropic Labs 把 Claude Design 推上来了。

订阅 Pro 以上的用户打开 claude.ai/design 就能用。一句话生成 prototype、slide deck、landing page、marketing 素材，画布上直接改、导出成 PDF / PPTX / HTML，或者一键塞给 Claude Code 让它把代码写出来。

昨晚美股开盘，Figma（NYSE: FIG）盘中最低跌了 7.82%，收盘跌 6.89%，从前一天的 20.32 美元跌到 18.92 美元。

一周前我写过一篇——Opus 4.7 还没发布，Figma 就先跌了 6%。那篇的结尾我留了一个问号：Figma 接下来会怎么走。一周过去，答案就是这根收盘价 18.92 美元的阴线。

Claude Design 到底是个什么东西

官方说法是：research preview，Pro / Max / Team / Enterprise 订阅里直接开。底下跑的是前天（4 月 16 号）发的 Opus 4.7——那个视觉分辨率涨到 2576 像素、能一口吃下一整屏 Figma 截图的版本。

从一句话到一个 prototype 的路径是这样的：

输入框里敲”帮我做一个付费订阅落地页，主色是深紫，参考 Linear 的风格”。

Claude 在画布中央开始画，直接画出一个能交互的 HTML 页面——按钮能点、tab 能切、表单能填。

画完之后你可以在任意元素上留 comment，或者让 Claude 给你一个”调节滑块”——比如它会现场生成一个”行距”slider，你拖一下画布上所有段落的 line-height 都跟着变。这个交互我之前真没见过。

更关键的一步在导出：Claude 会把整个 design bundle 打包，一条命令丢给 Claude Code，它就把代码写出来了。从 idea 到 prototype 到上线代码，全在 Anthropic 自家产品里转。

Brilliant 的设计师在官方发布页里留了一句话——“以前在其他工具里要 20 个 prompt 才能还原的复杂页面，在 Claude Design 里 2 个 prompt 就够了”。Datadog 那句更刺——“以前一周的 brief + mockup + review 来回，现在一次对话解决”。

我昨晚摸黑试了一下

我订阅的是 Max。页面灰度到我那里的时候已经凌晨快 1 点了。

打开 claude.ai/design 看到的就是这个界面：

左边是新建 prototype / slide deck / template 的入口，右下是 Set up design system——读你公司的代码和设计文件，把品牌色、字体、组件抽出来变成一套自动应用的 design system。

试的第一个东西是”做一个博客文章封面编辑器，能上传图片、加文字、选字体、导出 PNG”。我给 Claude 的就这一句话。

它 40 秒给了第一版，页面打开是一个左画布+右侧边栏的布局——画布显示上传的图，右边是字体/颜色/位置的控件。我上传了一张我之前博客的封面图进去，试了一下加文字——能导出 PNG。

这已经是一个能跑的 prototype 了。

然后我开始改。我说”把右侧边栏的颜色改成浅灰”、”字体默认换成思源黑体”、”加一个对齐辅助线”——这三个改动全是通过对话说出来的，Claude 在画布上直接改，不用重新生成整个页面。

这种改法比 Figma 里拖图层快太多。Figma 里我要找到那个元素、选中、改属性，每一步都要用鼠标点。Claude Design 里我一句话，它帮我找到、改好。

当然也踩了坑。我让它加一个”历史记录”功能，它给我加了一个假的——画布上有 UI 有按钮，点下去没反应，因为这毕竟是 prototype 不是真应用。想真能用得送去 Claude Code 补后端。

凌晨 2 点我关了电脑。第二天早上起来股价已经出来了。

设计师 in the loop 这个假设没了

Figma 的底层假设一直是：设计这件事里永远有一个训练过的设计师。

Figma 好用在”一群设计师围着一个文件协作”这件事上，它的护城河是十年积累的协作网络、几百万个社区组件、和每个公司都迁移过去的设计系统。所有功能都围绕”设计师”这个角色设计。

Claude Design 不做这个假设。

它的目标用户写在官方博客里：founders、PM、marketers。这些人是 Figma 的非客户——他们从来没打开过 Figma，也没打算学。之前他们想出一个 landing page 要么找设计师排队，要么用 Canva 模板拼，要么自己在 Notion 里拉文档应付。

现在这些人打开 Claude，一句话拿到一个能点的 prototype。这块饼 Figma 以前根本吃不到——founder、PM、marketer 这批人本来就不会打开 Figma。

这种错位打击比正面对抗更难受。Figma 的所有产品优势——协作、社区、精细度——在”founder 一个人出 demo”这个场景里都派不上用场。VentureBeat 那句评得挺到位：Figma 的产品设计一直默认有设计师在 loop 里，Anthropic 的工具把这个默认条件取消了。

一周内 Figma 掉了 13%

时间线拉长来看更有意思：

  • 4 月 11 日：The Information 首次报道 Anthropic 在做 design 工具——Figma 预先跌了大约 6%
  • 4 月 14 日：Anthropic CPO Mike Krieger 辞去 Figma 董事会席位，当天 Figma 再跌 6%
  • 4 月 16 日：Anthropic 发 Opus 4.7
  • 4 月 17 日：Claude Design 正式上线——Figma 盘中跌 6.8%，收盘跌 7.28%

一周时间 Figma 掉了大约 13%。自去年夏天 IPO 以来，Figma 股价已经从高点跌掉 80% 以上。

Mike Krieger 这步棋值得单独说一下。他是 Instagram 联合创始人，2024 年加入 Anthropic 做 CPO，不到一年前又去坐 Figma 董事会。4 月 14 日他把董事辞了，SEC 披露的当天，The Information 的 design 工具报道就出来了。

这个时间差不是巧合。一个董事如果知道自家公司马上要发一个跟所在董事会公司直接竞争的产品，他必须先辞。辞完的当天消息才能放出来。

这个时序是 Anthropic 自己压住的。

连带效应是整个设计生态

4 月 14 号那天 Krieger 辞职的消息一出来，跌的不止 Figma。

Adobe 跌 2.7%，Wix 跌 4.7%，GoDaddy 跌 3%。

华尔街看待这件事的方式是：Claude Design 动的不只是 Figma 一家的饭，整个”做一个视觉成品”的工作方式都在被改写。

Adobe 靠 Photoshop 和 Illustrator，假设你会用专业软件。
Wix 和 GoDaddy 靠模板拖拽，假设你要一个网站但不想写代码。
Canva 靠易用性，假设你需要快速出社交媒体图。

三个赛道的公共假设都是”用户打开一个工具去做”。Claude Design 把这个假设改成”用户打开一个对话框去说”。

Canva 的反应最有意思——没跌。CEO 直接在 Anthropic 发布页里出镜，说”我们很高兴和 Claude 的合作能让人把想法从 Claude Design 带到 Canva 里继续协作”。Canva 做了一个动作：认怂 + 绑定。Claude Design 支持”一键导出到 Canva”，等于把 Canva 绑成了下游编辑层。

Figma 没接这个合作，所以开盘就吃了这根 7% 的阴线。

Design → Code 这条线才是最致命的

看完 Claude Design 我最受震动的其实不是”能做 prototype”这件事——Figma、Canva、Framer 很多工具都能做 prototype。真正让我停顿的是它对代码库的打通：你可以把整个 repo 挂进来，然后直接问”账号删除入口应该放在哪”——它读你的代码，看你现有的导航、设置页结构，再给设计建议。

同样的逻辑反过来也成立。Claude Design 里做完一个页面，右上角有一个”Send to Claude Code”。点下去，它把 design bundle（HTML、design tokens、组件定义、交互描述）打包成一条 prompt，送进 Claude Code。Claude Code 接到之后，开 worktree，写代码，跑测试，出 PR。

这条流水线，从产品经理开会时的想法到 merge 上 main 分支，全部在 Anthropic 的产品矩阵里跑完。

Reddit 上有人已经在把这个矩阵往外延伸：Claude Design 出 UI → Magic Hour 做动画 → Kling 转视频 → ElevenLabs 配音。一个 founder 一个人，当天能出一个能发布的产品 demo。

Anthropic 从模型公司变产品公司这件事，去年 Claude Code 出来的时候已经起步了。今年一口气发 Opus 4.7、Claude Design、Claude Cowork 这几个，把”开发”、”设计”、”协作”这三条线都占住了。2 月那轮 300 亿美元融资、3800 亿美元估值，现在看不是泡沫——是把钱烧进这几条产品线的票据。

那 Figma 是不是就完了

也没那么快。

Claude Design 现在还是 research preview，不能替代 Figma 里那些超复杂的产品设计流程——几千个组件的 design system、几百人协作的大团队、每一个像素都要精细抠的品牌项目，这些场景 Figma 还是目前的最优解。

Reddit /r/investing 下面有人说得挺实在：”我试了 Claude Design，它很酷，但说它是 Figma 的竞品还早——它只是 Figma 功能面积的一小片。”

这话我认同一半。

另一半是：Claude Design 的增长斜率跟 Figma 不一样。Figma 的用户池是全球设计师，大概几百万量级；Claude Design 瞄准的是全球所有 Claude Pro 以上订阅用户，里面有大量从不碰设计工具的 founder、PM、marketer。这个池子比 Figma 的核心市场大一个数量级。

当一个 founder 习惯了”我想到什么就说一句，Claude 给我做出来”——他之后根本不会有动力再去学 Figma。

Figma 十年堆出来的协作网络能留住老设计师，留不住下一批从 Claude 入门的创业者。

我现在的判断

昨晚我躺床上想了一会儿，想清楚几件事。

专业设计师短期不用恐慌。几千个组件的 design system、几百人团队的大项目，这些场景 Figma 还是最顺手的工具——Claude Design 的复杂度远没到那一层。但手里要不要继续把未来两三年押在 Figma 生态里，这个赌注已经没那么稳了。

订了 Claude Pro 的，今天就打开 claude.ai/design 试一下，别等灰度。我昨晚 40 分钟的体验已经让我改了习惯：下一次做产品原型，我先开这个，Figma 放一边。

设计工具的投资人不用听别人分析，过去一年 Figma 的股价走势就够说明问题了。

昨晚 23 点我刷到 Anthropic 官号那条推的时候，第一反应是”这一刀来得真快”。上周五 Krieger 辞董事我还在想 Anthropic 的动作顶多在一两个月内。结果三天后东西就上线了。

现在回头看这句挺精准。

你昨晚看到 Claude Design 消息了吗？有没有点开 claude.ai/design 试一下？我比较好奇一件事：做过 Figma 专业用户的人，用 Claude Design 的第一感觉是什么——是”这东西迟早吃掉我的工作”，还是”它离我用 Figma 做的事还远”？评论区聊聊。
