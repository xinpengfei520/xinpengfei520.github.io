Claude Sonnet 5 逼近 Opus 还便宜一半：我那本「用之前先过秤」的账，得重算了

发表于 2026-07-01 更新于 2026-07-21 分类于 观点 阅读次数： 本文字数： 2.6k 阅读时长 ≈ 2 分钟
Anthropic 在 2026-06-30 发布 Claude Sonnet 5，SWE-bench Pro 63.2% 逼近 Opus 4.8 的 69.2%，知识工作打平，价格只有 Opus 的六成。它第一次让 Opus 对多数活变成可选项。但 Sonnet 5 换了新 tokenizer，同样文本多吃 1 到 1.35 倍 token，标价便宜 40% 不等于账单便宜 40%，选型得按活的代价乘调用量乘单价重新分档。

哈喽，我是飞飞。

昨天 Anthropic 发了 Claude Sonnet 5，我第一反应没去翻跑分榜。先打开的，是自己那本模型分配的账本。

我最近连着写了几篇 Claude 的东西，聊过它进 Slack 当同事、聊过它被投毒的安全坑、也聊过怎么用 CLAUDE.md 攒复利。这篇跟那些都不搭界，只聊一件很具体、很俗的事：一个逼近 Opus 又便宜一半的模型出来后，我手头那些活，该重新分给谁跑。

这对我这种每个月给 Anthropic 交不少钱的人来说，是笔要认真算的账。

我有本模型分配账

先说我为啥有这么本账。

我整条写文章的流水线，挂着十几个 skill，全长在 Claude Code 上。每个 skill 干的活不一样：有的要它读一大堆资料提炼观点，有的只是帮我把格式转一转。

所以让 agent 动手之前，我脑子里都会下意识过一遍秤：这活值不值得动用 Opus？重的、容易出岔子的，上 Opus；那些机械的杂活，我手动切到便宜模型，省点额度。

举个我自己的例子。写稿那个 skill、还有润色查 AI 味的那步，输出质量直接决定文章成不成，我一直咬牙用 Opus，错一句我得回头返工，那点模型钱省不得。

可像把 Markdown 转成公众号排版、给标签分类这种死板活，我早换成便宜模型跑了，反正它错不到哪去。

中间还夹着一批我一直没想明白该归哪档的活，比如读一大摞资料提炼观点，用 Opus 觉得肉疼，用便宜的又怕它抓不住重点。

这种「用之前先在心里称一下」的绷着感，就是「贵」最真实的样子。你不会拿最贵的模型去跑一个改标点的活，就跟你不会开跑车下楼买瓶酱油一样。

Sonnet 5 这一出来，我账本里一直悬着、没想明白该归哪档的那批中间活，全跟着晃了。到底往哪挪，值得好好算一算。

Sonnet 5 到底强到哪

先看 Anthropic 官方自己给的那张对比表，数字我都核过。

在最硬的那项 agentic 编码 SWE-bench Pro 上，Sonnet 5 拿了 63.2%，前一代 Sonnet 4.6 是 58.1%，一步跨了五个多点。终端操作 Terminal-Bench 上 80.4%，电脑操作 OSWorld 上 81.2%，都比上一代明显往上走。

真正让我盯着看半天的，是它跟旗舰 Opus 4.8 贴得有多近。

官方这次没把宝押在某个跑分上。它反复强调的关键词是 agentic 可靠性：长任务链不容易丢上下文，工具调用失败了自己会纠错，跑一个长会话状态更稳。说实话，这几条对天天让它连轴干活的人来说，比多两个点的跑分实在得多。

价格这块才是重锤。促销期到 8 月 31 号，输入每百万 token 两美元，输出十美元。之后恢复到三美元和十五美元。而 Opus 4.8 是五美元和二十五美元。标准价一摆，Sonnet 5 便宜了差不多四成。

有人说得挺到位：这是头一回，Sonnet 和 Opus 落在了同一条性价比曲线上，Opus 对大多数活，第一次变成了可选项。

Opus 强在哪没退休

但我得替 Opus 说句公道话，别看完就把它打入冷宫。

同样是那张官方表，Opus 4.8 在几块硬骨头上还是领先的。SWE-bench Pro 它 69.2%，比 Sonnet 5 高六个点。这六个点在小活上你没感觉，可一旦是那种几十个文件互相牵扯的大重构、长链路的工程活，六个点就是「一次能不能跑对」的真实差距。深度推理、电脑操作这些，Opus 也都还压着。

有意思的是知识工作那项 GDPval，Sonnet 5 拿了 1618，反而微微超过 Opus 的 1615。

所以结论不能简单说成 Sonnet 5 全面赢了。两个模型各有各的地盘：硬核编码和深推理留给 Opus，价格和速度归 Sonnet 5，知识类的活两边打平。它俩谁也没吃掉谁，只是中间那道墙被推薄了。

账单没标价那么香

讲到这，最容易被忽略、但对我这种要过秤的人最要命的一条，来了。

标价便宜四成，不代表你的账单就降四成。

Sonnet 5 换了个新的 tokenizer，跟 Opus 4.7 那套一样。同样一段文字喂进去，它切出来的 token 数，可能是原来的一倍到一点三五倍。Anthropic 自己在脚注里也承认了，说促销价是特意校准到「大致成本中性」的，但等促销一过、恢复标准价，这个 token 膨胀就会把你名义上省的那四成，悄悄削回去一截。

这就是我说的过秤要秤准。你光看单价那个数字，会以为捡了大便宜；真正决定你月底掏多少钱的，是单价乘上你实际烧掉的 token 量。后面这个量，被 tokenizer 偷偷改过了。

这种账我太熟了，光盯着单价那个数字乐呵，月底账单一拉才发现根本没省下几个钱。所以别信 sticker price，拿你自己真实的活跑一批，看月底实际扣了多少，才算数。

我的活准备怎么重排

那具体到我这本账，怎么重排。

我不打算新模型一出就无脑全切，那是另一种偷懒。我准备挑出自己调用量最大的那两三个 skill，头一个就是刚才说的那个读一大摞资料提炼观点的中间活，各拿一批我手头真实的选题，让 Sonnet 5 和我现在用的模型分别跑一遍。

两头我就盯两个东西。一是产出质量：它提炼出来的观点，够不够我直接拿去写，还是又得我大改一遍。二是实际 token 账单：同一批活跑下来，两边到底差多少钱，注意是看真实扣费，不是看那个标价。跑完这一轮，哪些活迁、哪些活留，我心里就有数了，不用凭感觉拍脑袋。

判断的尺子我也想清楚了。一个活该用哪档模型，看三样东西相乘：这活错一点的代价有多大，我一天要跑它多少次，以及单价。选模型这事，真正该盯的从来不是「哪个模型最强」，是「这个活错一点的代价，乘上我要跑多少次，再乘单价」。

按这把尺子，那些错了代价大、但我一天跑不了几次的活，比如关键的大重构，我还是留给 Opus，多花的钱买的是安心。那些质量要求没那么顶、但一天要跑几十上百次的高频活，是 Sonnet 5 最该接手的地方。省下来的是实打实的额度。

报活我帮你分档

绕一圈回到那本账本。

Sonnet 5 真正改变的，不只是又多一个模型可选。它把「贵」和「够用」之间那个纠结地带，给填上了一大块。以前我总在「舍不得动 Opus」和「便宜模型不够聪明」之间反复横跳，现在中间有了个体面的默认选项。

但工具给你便利，算账这件事还得你自己干。哪些活敢交给它、哪些活省下的是真钱、tokenizer 会不会把你的账偷偷抬回去，这些没人能替你测，得你自己拿真实任务过一遍秤。

说到底，会换模型的人很多，能把自己每类活该花多少钱算明白的人不多。你现在手头哪个活最纠结用哪档模型？评论区报一下这个活大概在干嘛、一天跑多少次，我帮你分个档。
