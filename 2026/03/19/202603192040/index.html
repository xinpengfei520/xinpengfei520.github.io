不会写代码也能做 App？我用 OpenAI Codex 开发了一个插件，赚到了第一桶金

发表于 2026-03-19 更新于 2026-03-30 分类于 阅读 阅读次数： 本文字数： 4.4k 阅读时长 ≈ 4 分钟

凌晨两点半，我盯着屏幕上那行绿色的提示——“Build Succeeded”。

那一刻我的手是抖的。不是因为咖啡喝多了，是因为我——一个连 Python 都没学完的产品经理——居然真的把一个能用的插件做出来了。

三天后，这个插件上架了。第一周，47 个人付费下载。收入不多，但那笔 $141 到账的时候，我对着支付通知截了三次图。

这是我人生中第一次靠”写代码”赚到钱。但严格来说，我一行代码都没写。

写代码的是 OpenAI Codex。我只是那个提需求的人。

今天我把这个过程完整记录下来——不是教程，更像是一个”非程序员用 AI 做产品”的实验报告。踩过的坑、花过的时间、最终怎么赚到钱的，全部坦诚分享。

OpenAI Codex 是什么：不是补全代码，是替你写整个项目

先说清楚一件事：2026 年的 OpenAI Codex，和你印象里的那个”代码补全工具”完全不是一个东西。

2021 年那个 Codex，是 GitHub Copilot 背后的模型，帮你补全代码片段。2026 年的 Codex，是一个自主编程 Agent——你用自然语言描述你要什么，它自己规划、自己写代码、自己测试、自己修 Bug，最后给你一个能跑的项目。

具体来说，现在的 Codex 有三种使用方式：

  1. Codex 桌面 App：macOS 和 Windows 都有，独立应用，打开就能用
  2. Codex Web：在 ChatGPT 里直接用，云端运行
  3. Codex CLI：命令行工具，开源的，用 Rust 写的

桌面 App 是 2026 年 2 月发布的，搭载 GPT-5.3 Codex 模型——OpenAI 目前最强的编程模型。发布一个月，超过 100 万人下载，周活用户突破 160 万。

最关键的一点：免费用户也能用。 Sam Altman 明确说了，Codex 会保留免费额度。当然，免费的限制多一些，付费用户（ChatGPT Plus，$20/月）额度更高。

这意味着什么？意味着任何人——产品经理、设计师、运营、学生——只要能把需求说清楚，就能让 AI 替你把软件做出来。

为什么我要自己做一个插件

故事的起点很简单：我在工作中遇到了一个痛点，市面上没有好的解决方案。

我的日常工作需要频繁在 Notion 里整理会议纪要。每次开完会，我要手动把录音转成文字，然后提取待办事项，再分别填进 Notion 的不同数据库里——任务库、决策库、问题跟踪库。

一次会议，整理工作要花 30-40 分钟。一天三四个会，光整理就要两个小时。

我找了很多工具。有些能转录，但不能自动分类。有些能分类，但和 Notion 的集成很拉胯。没有一个能完整解决我的问题。

某天晚上刷到一条推文，有人用 Codex 做了一个 Chrome 扩展，自动化了他的某个工作流。我想：我能不能也做一个？

这个念头一旦冒出来，就再也按不下去了。

第一夜：从零开始，用自然语言”写”代码

晚上 9 点，我打开了 Codex 桌面 App。

我没有任何 Chrome 扩展开发经验。我甚至不知道 Chrome 扩展的文件结构长什么样。但我知道我要什么——我把需求写成了一段话：

“帮我创建一个 Chrome 浏览器扩展。功能：检测当前页面是否是 Notion 的会议记录页面。如果是，在页面右侧弹出一个浮窗，显示三个按钮——‘提取待办’、’提取决策’、’提取问题’。点击按钮后，用 AI 分析页面内容，提取对应信息，格式化后复制到剪贴板。”

按下回车。

然后我看着 Codex 开始工作。它先分析了需求，拆解成了几个步骤。然后开始一个文件一个文件地创建——manifest.json、background.js、content.js、popup.html、popup.js……

整个过程大概 8 分钟。它不只是生成了代码，还自己跑了一遍测试，发现一个权限配置的问题，自己修复了。

最后它告诉我：”项目已创建完成，你可以在 Chrome 开发者模式中加载这个扩展进行测试。”

我按照它的指引，打开 Chrome，加载扩展。打开一个 Notion 页面——浮窗出来了。

按钮能点。内容能提取。格式化后复制到剪贴板，粘贴到 Notion 的数据库里，完美匹配。

8 分钟。从零到能用。

当然，这只是第一版。还很粗糙。但那种”我居然做出来了”的感觉，真的很上头。

第二夜：迭代、打磨、踩坑

第二天晚上，我开始优化。

这时候 Codex 的”对话式开发”体验就体现出来了。我不需要读代码、改代码。我只需要告诉它哪里不好，它自己去改：

  • “提取结果的格式不对，待办事项应该包含负责人和截止日期”
  • “浮窗的位置挡住了 Notion 的侧边栏，帮我改成可拖动的”
  • “加一个设置页面，让用户自定义提取规则”
  • “支持快捷键触发，不用每次都点按钮”

每一轮修改，Codex 都会自己改代码、自己测试、告诉我结果。我就像一个甲方——只提需求，不碰代码。

但也踩了坑。

坑 1：上下文丢失。 到第 15 轮对话的时候，Codex 开始”忘记”之前的改动。它把一个我前面让它加的功能又改没了。解决方法：创建一个新线程，把当前代码和需求文档一起发给它，让它从最新状态开始。

坑 2：样式问题。 AI 写的 CSS 总是差那么一点。按钮大小不协调，颜色不统一，间距不一致。我花了比预期更多的时间在 UI 调整上。最后的解决方法是：给它一个参考截图，告诉它”照着这个风格来”，效果好了很多。

坑 3：Chrome 扩展的审核要求。 Codex 生成的代码能跑，但不一定符合 Chrome Web Store 的审核要求。比如它用了 eval() 函数，这在 Chrome 扩展的 Manifest V3 里是禁止的。我是提交审核被拒之后才发现的。然后我告诉 Codex：”Chrome Web Store 审核说我的代码用了 eval()，帮我改掉。”它一分钟就改好了。

整个第二夜，大概花了 4 个小时。其中一半时间在调 UI，一半时间在处理审核相关的问题。

第三夜：上架

第三天，我做了三件事：

第一，写了一个落地页。 还是用 Codex。我告诉它：”帮我用 HTML + CSS 做一个单页网站，介绍这个 Chrome 扩展的功能，包含截图展示区、功能列表、下载按钮。风格简洁，科技感。”10 分钟搞定，部署到 GitHub Pages，免费托管。

第二，准备 Chrome Web Store 的素材。 图标、截图、描述文案、隐私政策。图标用 Midjourney 生成的，截图直接截屏，文案让 ChatGPT 写的，隐私政策用模板改的。

第三，定价。 这个纠结了最久。最后定了 $2.99/月的订阅制。理由：足够便宜，用户不会犹豫；订阅制让我有持续收入；而且这个价格覆盖了我 ChatGPT Plus 的月费。

提交审核。等了两天。通过了。

变现：第一桶金怎么来的

上架后，我在三个地方做了推广：

  1. Twitter/X：发了一条推文，讲了自己作为非程序员用 AI 做插件的经历。这条推文意外地火了——3000+ 转发。大多数人不是对插件本身感兴趣，是对”非程序员做产品”这件事感兴趣。

  2. Reddit：在 r/Notion 和 r/productivity 发了帖子，重点讲这个插件解决了什么痛点。

  3. Product Hunt：上架当天排到了日榜第 8。

第一周的数据：

指标 数据
安装量 312
付费用户 47
收入 $140.53
退款 2

$140。不多。但对我来说意义重大。

这是我第一次不靠卖时间、卖技能，而是靠一个”产品”赚到钱。 产品在那里，24 小时运转，有人用、有人付费。我睡觉的时候，它也在帮我赚钱。

到现在（上架三周），付费用户 130+，月收入稳定在 $400 左右。不够辞职，但足够让我认真思考下一个产品了。

Codex 做产品的 5 条实战经验

回顾这三天三夜，我总结了 5 条经验：

第一，需求比技术重要 100 倍。 Codex 不缺技术能力。它缺的是你告诉它”做什么”和”为什么做”。我能做出来这个插件，不是因为我懂代码，是因为我太了解这个痛点了——我每天被它折磨。你的领域知识，就是你的护城河。

第二，用 Skill 固化流程。 Codex 现在支持 Skills——可复用的指令模板。我把”Chrome 扩展开发”的规范、审核要求、文件结构写成了一个 Skill。下次再做扩展，Codex 直接加载这个 Skill，不用我重复解释。

第三，分步骤，不要一口气。 不要试图用一个超长的 prompt 描述所有功能。先让 Codex 做出最核心的功能，确认能跑，再一步步加。每次只加一个功能。这样出了问题容易定位。

第四，多线程并行。 Codex 桌面 App 支持多个线程同时运行。我一个线程让它写核心功能，另一个线程让它写落地页，第三个线程让它写文档。三个任务并行，效率翻倍。

第五，AI 做 80%，你做 20%。 别指望 AI 做出 100% 完美的产品。代码能跑 ≠ 产品能用。UI 细节、用户体验、文案调性——这些”最后 20%”的打磨，还是需要人来做。但 AI 帮你跳过了前面 80% 最耗时的部分，这已经是巨大的杠杆。

2026 年，谁都能做独立开发者

回到更大的图景。

2026 年的 AI 编程工具市场，竞争异常激烈。Codex、Claude Code、Cursor、Windsurf、Kiro、Devin……每一家都在比谁更”自主”，谁更能替代程序员的工作。

但我觉得，这些工具最大的价值不是替代程序员，而是把编程能力给了那些不会编程但有好想法的人。

以前，有一个产品想法，你需要：找一个程序员朋友 → 说服他 → 等他有空 → 等他做完 → 改需求 → 再等。整个过程，少则一个月，多则半年。

现在呢？打开 Codex，描述需求，8 分钟出原型，三天上线。

社区里有人用 Codex 做了 macOS 桌面应用，有人做了 iOS App（通过 Xcode 集成），有人做了自动化交易机器人。一个 RAND 研究发现，市面上 80-90% 号称”AI Agent”的产品其实只是聊天机器人套壳。但 Codex 是真的能自主完成任务的——规划、编码、测试、修 Bug，一条龙。

Fortune 的报道提到，OpenAI 的 Codex 负责人 Thibault Sottiaux 说，Codex 未来的目标不只是编程工具，而是要成为”每个人都能用的自主 Agent”——从编码扩展到更广泛的知识工作。

编程正在从一门”技能”变成一个”指令”。 你不需要知道怎么写代码，你只需要知道你要什么。

写在最后

那天凌晨两点半看到”Build Succeeded”的时候，我其实不知道这个插件能不能赚到钱。

但我知道一件事：以前我觉得”做产品”是程序员的特权，现在不是了。

$141 不多。但它证明了一件事——一个不会写代码的人，用 AI 工具，可以从 0 到 1 做出一个有人愿意付费的产品。

这件事在 2025 年还很难。在 2026 年，它变得稀松平常。

如果你也有一个”要是有这个工具就好了”的想法，我的建议只有一个：今晚就打开 Codex，试试看。最坏的结果，不过是浪费几个小时。最好的结果——你可能会赚到你的第一桶金。

你有没有用过 AI 编程工具做过什么有意思的东西？有没有靠 AI 赚到过钱？如果给你一个 Codex，你最想做什么产品？评论区聊聊你的想法——也许下一个赚到第一桶金的人，就是你。
