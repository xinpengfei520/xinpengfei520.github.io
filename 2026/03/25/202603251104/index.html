阿里云 JVS Claw 全面开放：人人都能养一只云端"龙虾"了

发表于 2026-03-25 更新于 2026-04-13 分类于 工具 阅读次数： 本文字数： 3.5k 阅读时长 ≈ 3 分钟

你可能已经刷到过这个词了——“养虾”。

不是真养虾。是一群程序员、产品经理、甚至完全不懂代码的人，在手机上养了一只 AI “龙虾”，让它帮自己写代码、搞数据、做 PPT、甚至操作企业内部系统。上周我一个做运营的朋友还在问”MCP 是什么”，这周她已经让龙虾帮她自动生成日报了。

3 月 13 号，阿里云正式发布了 JVS Claw。一开始需要邀请码，很多人抢不到。现在，全面开放了。 访问 jvsclaw.aliyun.com 下载客户端，注册就能用，不用排队，不用邀请码。人人都能养一只自己的云端”龙虾”。

我昨天下载试了一下，说说真实感受。

先说清楚：JVS Claw 到底是什么

JVS Claw 是阿里云无影团队做的产品。你可以把它理解成一台”跑在云上的电脑”，但这台电脑上住着一只 AI Agent——也就是那只”龙虾”。

具体来说，JVS Claw 有两个核心模块：

ClawBot 是那只龙虾本身。它是一个 AI 助手，能理解你的自然语言指令，然后去执行各种任务。你跟它说”帮我整理这份 Excel 里的销售数据，按季度汇总”，它真的会打开文件、操作表格、把结果给你。不是给你一段代码让你自己跑，是它直接动手干。

ClawSpace 是龙虾住的地方。一个云端桌面环境，预装了各种开发工具和办公软件。你的龙虾在这个环境里操作，跟你在自己电脑上操作一样，只不过算力和存储都在云上。

说白了，你给它一句话，它能自己打开软件、点来点去、把活干完。不是那种”给你一段代码自己去跑”的偷懒式 AI。

“养虾”到底在养什么

如果你最近关注科技圈，会发现”养虾”这个词突然到处都是。最夸张的是深圳，对 OpenClaw 创业公司最高直接补贴 500 万——上一次见到这种力度还是当年补贴新能源车的时候。Jensen Huang 也在喊 OpenClaw 是”下一个 ChatGPT”，虽然老黄的话打个折听比较好，但这个赛道确实在肉眼可见地爆发。

JVS Claw 就是建在 OpenClaw 架构上的。OpenClaw 定义了一套标准协议——AI Agent 怎么跟外部工具交互、怎么访问数据源、怎么调用其他 Agent。你可以把 OpenClaw 想象成”龙虾的操作系统”，而 JVS Claw 是阿里云基于这个操作系统做出来的一只具体的龙虾。

市场上不只阿里在做。Kimi 有自己的 Claw，字节有 ArkClaw，腾讯有 WorkBuddy，微信还上线了 ClawBot 插件。大厂全部下场了。

但阿里这只龙虾有个目前独一份的东西：它自带一台云电脑。

其他家的龙虾更像是一个聊天机器人——你问它问题，它给你答案。JVS Claw 的龙虾是真的有一个操作环境的。它可以打开浏览器、运行代码、操作文件系统。这个区别很关键，因为很多实际工作不是”回答一个问题”就能搞定的，而是需要一连串的操作。

我试了一下，说说真实体验

昨天晚上我在 Mac 上下载了 JVS Claw 的客户端。安装过程没什么好说的，扫码登录阿里云账号就行。

进去之后，界面比我预期的干净。左边是跟 ClawBot 的对话框，右边是 ClawSpace 的云桌面。云桌面的启动速度大概 8 秒——比我预想的快，毕竟是云端虚拟机。

我先试了个简单的：让它帮我写一个 Python 脚本，把一个 CSV 文件里的重复行去掉。ClawBot 没有直接给我一段代码，而是在 ClawSpace 里打开了终端，自己创建了一个 .py 文件，写好代码，跑了一遍，然后告诉我”处理完了，去掉了 47 行重复数据，结果保存在 output.csv”。

整个过程我就说了一句话。

然后我试了个复杂一点的——让它帮我把一份 Word 格式的周报转成带图表的 PPT。这个它卡了一下，花了大概两分钟，中间问了我一次”图表要用柱状图还是折线图”。最后出来的 PPT 说实话不算惊艳，但能用。字体和配色都是默认的，不丑但也不好看。如果你要求不高，直接发出去没问题；如果要给老板看，还得自己调一下。

还有一个让我印象深刻的细节：它支持接入钉钉、企业微信、飞书。我试了钉钉的集成，设置完之后，我在钉钉里直接 @ 我的龙虾，让它查一下某个项目的最新进度，它真的去翻了钉钉里的聊天记录和文档，给我汇总了一份简报。这个能力放在企业场景里，想象空间很大。

不过也有翻车的时候。我让它帮我分析一个 GitHub 仓库的代码结构，它在 ClawSpace 里 clone 了仓库，但分析到一半报了个内存不足的错误。看了一下，免费版的 ClawSpace 配置不高——2 核 4G 内存，跑大项目确实吃力。这个后面肯定会开放更高配置的付费版本，但现在免费版就是这个水平。

跟其他”龙虾”比，阿里这只怎么样

说实话，我这两周把市面上主流的龙虾都试了一遍。

Kimi Claw 的强项是对话能力。你跟它聊天，它回答得很聪明，上下文理解也好。但它没有独立的操作环境，本质上还是一个对话 AI，只是套了 OpenClaw 的壳。

字节的 ArkClaw 跟豆包绑定，在内容创作方面做得不错，写文案、做短视频脚本很顺手。但代码能力就差了不少，让它写个稍微复杂点的脚本就开始出错。

JVS Claw 的定位不太一样。它不是最聪明的那只龙虾——单纯比对话质量，可能不如 Kimi。但它是最”能干活”的那只。因为有 ClawSpace 这个云端环境兜底，它能做的事情范围比纯对话 AI 大得多。你让它操作文件、跑代码、装软件，它是真的在一个完整的操作系统里执行，不是模拟出来的。

我做了个简单测试：同样让三只龙虾”把这份 CSV 清洗完生成柱状图”。Kimi 给了我一段 Python 代码让我自己跑，ArkClaw 说它不太擅长这个，JVS Claw 直接在云桌面里跑完了把图表文件甩给我。就这个区别——谁是真能干活的，一目了然。

阿里在下什么棋

JVS Claw 全面开放只是第一步。

阿里同时还在做 JVS Book（笔记本电脑）和 JVS Box（迷你主机）。这两个硬件产品还没发布，但方向很清楚：阿里想做的不只是一个 AI 助手 App，而是一整套”AI + 云计算”的终端生态。

想想看：一台轻薄笔记本，本地只需要最基本的配置，所有重活都扔给云端的 ClawSpace 去跑。你的龙虾住在云上，你走到哪里，打开任何一台设备，龙虾和它的工作环境都在。

这个故事听着耳熟——Google 十年前用 Chromebook 讲过一模一样的。结果呢？没人买账，因为一台只能跑浏览器的低配电脑，除了便宜没有任何理由让人放弃 MacBook。但现在不一样了，云上多了一只能帮你干活的龙虾。”本地不需要高配”这件事终于有了一个真正的理由，而不只是”省钱”。

不过我昨晚用的时候，ClawSpace 偶尔会有零点几秒的延迟，写代码时光标跳动有时不太跟手，那种”我按了键但字还没出来”的感觉挺膈应的。网络好的时候完全没问题，但如果你在咖啡店用公共 WiFi，这个体验会让你很难受。另一个更大的坑是数据安全——你的文件全在阿里云的虚拟机上跑，我随便问了两个做安全的朋友，第一反应都是摇头。阿里要啃下企业市场，这块儿还有硬仗要打。

我现在拿它干什么

我现在每周五下午的周报数据整理全扔给它了。以前我要花 40 分钟对着 Excel 复制粘贴，现在一句话丢过去，两分钟搞定。光这一个场景，我就觉得这玩意儿没白装。

写脚本也是个高频场景。前两天我需要一个脚本把某个文件夹里的图片批量压缩上传到 OSS，我完全不想自己写——丢给龙虾，它在 ClawSpace 里写好、测试通过、还告诉我怎么设定时任务。整个过程我就坐着看它操作，有种”监工”的快感。

最让我意外的是拿它学编程。我最近在啃 Rust，看文档看得头疼，就随手让它解释一段代码。它不光解释了，还直接在 ClawSpace 里跑了一遍，把每一步的变量状态都标出来了。这比干看文档直观太多，我现在遇到看不懂的代码直接扔给它跑。

办公自动化那些——格式转换、文档整理——也能干，但说实话没有前面几个场景惊艳。别指望它做创意型的活儿，比如设计一个好看的 PPT 模板，出来的东西只能说”能用”。

写在最后

养虾这件事，三个月前还是小圈子里的尝鲜行为，现在大厂全部下场，连硬件都开始配套了。阿里这次全面开放 JVS Claw，等于把门槛从”需要邀请码”降到了”注册就能用”。

我会继续用它，但现在只敢让它干那种搞砸了也不心疼的活。周报数据整理、小脚本、学习用途——这些没问题，放心交给它。但重要的客户 PPT？关键业务的数据分析？还是自己来吧，至少现阶段是这样。

免费的，注册就能用，花十分钟试一下你就知道它适不适合你。我的建议是先找一个你每周都在重复做的无聊工作，丢给它试试。如果它能接住，你就赚到了一个免费实习生。

你打算用云端龙虾干什么？已经在养的，聊聊你的龙虾都帮你干了哪些活？评论区见。
