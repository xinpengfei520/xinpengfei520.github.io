搭建你的第一个 AI Agent：把它拆成脑、手、记忆、规划四件套，照着搭一个研究助手

发表于 2026-06-27 更新于 2026-07-14 分类于 编程 阅读次数： 本文字数： 2.5k 阅读时长 ≈ 2 分钟
AI 超级个体进阶第 7 课，把 Agent 拆成四件套：大模型当脑、工具当手、记忆让它记得住、规划让它会拆步骤。我用天天在跑的 Claude Code 一一对上，脑是 Opus、手是 MCP 加文件读写、记忆是一个 memory 文件夹加 CLAUDE.md、规划是长任务自己拆成子任务，再带你照着搭一个能自己查资料、整理、出报告的研究助手，小白先用现成 agent 别急着自己写框架。

哈喽，我是飞飞。

进阶系列上一课我们聊完了 MCP，今天这一课更动手：我带你亲手搭出第一个能干活的 AI Agent。

很多人对 Agent 的印象还停在「一个很厉害的聊天机器人」，觉得它神秘、门槛高、得是大厂工程师才碰得了。我天天靠一个 Agent 吃饭，掀开盖子看进去，里面其实就四样东西。把这四样搞清楚，你再看任何一个框架、任何一篇架构图，都会突然通了。

一个 Agent，掀开盖子就是这四样东西

我先把这四件套摆给你，后面每一样都用我天天在跑的工具对一遍。

第一样，大模型，相当于脑子。它负责理解你说的话、想清楚下一步该干嘛。但它光会想，不会动手。

第二样，工具，相当于手。联网搜索、读写文件、调接口，这些让模型能真的去外面的世界里办事，而不是只在对话框里空想。

第三样，记忆，让它记得住。聊天机器人转头就忘，Agent 得记得你是谁、上一步干到哪了、查到的东西存哪了。

第四样，规划，让它会拆步骤。你扔给它一个大目标，它自己拆成一小步一小步，调工具、看结果、不对就改，转着圈干到完。

普通大模型和 Agent 的差别就在这。一个普通 LLM 是你问一句它答一句，答完这一轮就结束了。Agent 是把这个模型包上手、记忆、规划，让它能盯着一个目标自己转好几圈，直到把事办成。这个「转圈」的本事，才是 Agent 区别于聊天框的地方。

这套四件套不是我编的。我翻了几篇综述，从 2023 年 Wang 那篇被引最多的自治 Agent 综述，到 SuperAnnotate、Prompting Guide 这些 2026 年的指南，讲法高度一致：脑、手、记忆、规划，名字略有出入，骨架一模一样。

拿我天天在跑的 Claude Code，把四件套一个个对上

光讲概念你记不住，我用一个真实的 Agent 给你对一遍。Claude Code 是我每天干活的家伙，它正好这四样齐全，是个标准样本。

脑，是 Opus 这个模型。 我让它干正经活之前，会下意识掂量一下值不值得动用 Opus，杂活就手动切个便宜模型省点额度。这种「用之前在心里过一遍秤」的体感，就是脑子这一块最真实的样子，它贵、它强、它负责想。

手，是一堆工具。 Claude Code 给我的手包括联网搜索、读写文件、敲命令行，还有一种叫 MCP 的东西（上一课讲过，你就理解成给 AI 接外部能力的标准插口）。官方把它的扩展能力归成六样：CLAUDE.md、技能、子 agent、斜杠命令、钩子、还有 MCP。说人话就是，这只手能伸到的地方特别多，不只是回你几行字。

记忆，是一个 memory 文件夹加一份 CLAUDE.md。 我的家底说白了就是十几个自定义技能、一份越喂越厚的 CLAUDE.md、再加一摞 memory 文件。CLAUDE.md 里写着这个仓库的规矩，memory 里记着我这个人的偏好和过往。每次开新对话，它先把这些读进去，就不会跟我从零认识。这就是记忆这一块。

规划，是长任务它自己拆。 我给它一个大活，比如写一篇文章，它不会一口气闷头干完，而是拆成查资料、整理、起草、配图、分发好几步，还会派出几个子 agent 各干一摊、各自在自己的小窗口里跑、干完只把结论汇报回来。这个「自己拆步骤、分头干、再汇总」的动作，就是规划。

你看，四件套，一个不缺。我天天用的这套写作流水线，本质上就是一个攒了十几个技能的研究兼写作 Agent，全长在 Claude Code 的钩子和技能机制上。

想动手，先选脑，别上来自己造框架

知道了四件套，下一步就是给你的第一个 Agent 配齐它们。新手最容易犯的错，是上来就想自己写一套 Agent 框架，结果卡在工程细节里，热情磨没了。我劝你反着来：先用一个现成的、四件套已经装好的 agent，把流程跑顺，再谈定制。

脑子怎么选，看你在哪个圈。国际上，Claude Code 和 Codex 都是装好就能用的成熟 agent，开箱四件套齐全。国内的话，DeepSeek 是个稳妥的选择，能力够用、用的人多、遇到问题搜得到答案。

还有个我想专门提一句的，GLM-5.2，智谱（现在叫 Z.ai）6 月 13 号刚发的，1M token 的超长上下文，权重还按 MIT 开源放出来了，发布当天就支持接进 Claude Code 用。它是专门冲着「让 agent 干长活」调的，原生就会调工具。国内想找个能扛长任务、又能塞进现成 agent 壳子里的脑子，它现在是个挺实在的选项。

我得多嘴一句：这些工具的版本和跑分变得飞快，你真要用之前去官网核一眼最新的，别全信我今天写的数字。

照着搭一个研究助手，把四件套用活

光对概念太干，我给你一个能照做的小目标：搭一个 AI 研究助手，能自己上网查资料、整理、最后产出一份报告。这正好是四件套的最小实战。

脑，挑一个会调工具的模型。 上面那几个随便选一个，关键是它得支持调工具，不然手装不上。

手，先只给它一个搜索。 别贪多，第一个 Agent 你就给它配一个联网搜索加一个文件读写。能查、能把查到的存下来，研究助手的骨架就立住了。等这一步跑顺，再往上加别的工具。

记忆，给它一个文件夹。 建一个文件夹，让它把查到的资料、你定的研究方向、你的偏好都写进去。下次再开工，它读一眼就接着上回干，不用你重新交代。这一步特别像我那份越喂越厚的 CLAUDE.md，攒得越久，它越懂你这摊事。

规划，把目标交给它自己拆。 你不用手把手指挥，直接给它一句话的大目标，比如「帮我把某个主题查清楚，整理成一份带来源的报告」。它会自己拆成搜索、读取、筛选、起草几步，一步步往下走，中间查不到就换个关键词再来。

这四步搭完，你就有了人生第一个真能干活的 Agent。它已经从问一句答一句的聊天框，长成了收了目标就自己转圈去办事的小助手。我天天跑的那条写作流水线，最早也就是从「给它一个搜索、配一个文件夹」这么一点点长起来的。

回到开头那句话，Agent 不神秘，掀开盖子就是脑、手、记忆、规划这四样。你今天选个现成的脑，给它一只手、一个文件夹、一个目标，第一个研究助手就站起来了。剩下的，是你越用越想给它加东西。

你打算让你的第一个研究助手先帮你查清楚哪件事，是盯一个你一直想搞懂的技术，还是替你扒某个领域最近的动态？评论区告诉我，我帮你看看四件套该怎么配。
