OpenAI 说效率提升了才敢拆限速墙，我去它自家论坛翻了翻，没那么单纯

发表于 2026-07-14 更新于 2026-07-21 分类于 观点 阅读次数： 本文字数： 2.7k 阅读时长 ≈ 2 分钟
OpenAI 宣布临时取消 Codex/ChatGPT Work 的 5 小时限速，官方称是 GPT-5.6 Sol 效率提升所致，但这些数字均为自述、无第三方复核；OpenAI 自家开发者论坛和 GitHub issue 里真实反馈是消耗更快而非更慢。对照 Anthropic 5 月靠买 22 万块 GPU 永久翻倍限速，两条路径一个临时一个永久，不能简单当作殊途同归。

哈喽，我是飞飞。

你要是关心 AI 编程工具的限速政策，这两天大概率也刷到了这条推。OpenAI 那位管 Codex 的工程负责人 Tibo 昨晚发了条推，说过去 48 小时 Codex 和 ChatGPT Work 太忙了，接下来三件事：暂时取消 5 小时限速，让 GPT-5.6 Sol 更省用量，活跃用户破了 600 万。我第一反应是，行啊，这波挺良心。可我这人算账算习惯了，说实话，多看了两眼就觉得不对味。

三项更新一夜刷屏

GPT-5.6 这拨 Sol、Terra、Luna 三兄弟 7 月 9 号才正式放出来，上线没几天需求就顶到天花板，OpenAI 那几天连着重置了好几次用量。Tibo 这条推文，说白了是在这波涌入之后打的补丁。

三件事分别是：Plus、Business、Pro 三档订阅的 5 小时滚动限速暂时拿掉；GPT-5.6 Sol 整体效率提升，用量消耗会降下来，具体数字后面再公布；Codex 用户破 600 万，后续更新说 7 月 14 日冲到了 700 万。从 600 万涨到 700 万，中间隔了不到两天，这个增速本身就说明了那几天的流量是怎么个涌法。十几家科技媒体连夜跟进核实，这条推文是真的，事情也确实发生了。

Codex 和 ChatGPT Work 共用同一个 5 小时滚动用量池，本地消息和云端跑的任务都往这个池子里扣。你要是在 Codex 里跑一个复杂任务，跑到一半额度没了，就得干等窗口重置。这次相当于先把这道干等的墙搬走。

但有一个词我扫了三遍才确认没漏看：暂时。多家报道原话都写着这不代表彻底无限制，周限额还在，OpenAI 也没说这是永久政策。

官方数字听着挺美

官方给的效率数字确实好看。OpenAI 自己那篇 GPT-5.6 发布页上写，Sol 开最高推理档跑分能到 80，比上一代 Fable 5 高 2.8 分，输出的 token 却不到之前的一半。翻译成人话就是，同样一顿饭钱，官方说 Sol 现在能多干出一倍的活儿。

官方预览材料里还提到一项安全能力基准测试，Sol 用大约三分之一的输出量就追平了对手同级模型的表现。OpenRouter 那边挂的数据是，跟老版本 Codex 比，token 更省、速度快了大概 25%。到了 7 月 13 号下午，Tibo 又补了一句，说推理优化能多出约一成的可用量，还专门否认了在思考预算上动手脚。

这些数字摆在一起看起来挺唬人。但说真的，我盯着这几行看了半天，发现一个共同点：全是 OpenAI 自己给的。没有一个是独立机构测出来复核过的。跑分是自家发布页写的，速度提升是自家关联的模型市场挂的，那 10% 更是创始团队成员发推补的口头承诺。

这不是说他们在撒谎，只是这套说法目前只有一个信源，那个信源恰好是这次要证明自己更省钱的当事人。

自家论坛却在喊贵

真正让我改主意的是接下来这一步。你猜怎么着，我顺手去了趟 OpenAI 自己的开发者社区，想看看真用户怎么说。结果撞上一个帖子，标题直接就是在问 Codex 的 token 消耗为什么最近涨得这么快。Reddit 那边的 codex 板块也在讨论同一件事，说 5.6 这版用量掉得明显比以前快。

不是嘴上抱怨，GitHub 上挂着具体的 issue。有开发者用 Terra 开最高推理档跑一次常规的仓库预审，几分钟就把整个 5 小时窗口耗光了，活儿都还没干完。另一条 issue 记录的是 Sol 中档推理，Pro 账号的 5 小时额度消耗速度明显不正常，跟同一个人之前用老模型跑同类活的消耗对不上。两条 issue 底下都有别的开发者跟帖说自己也遇到同样情况，不是个例。独立开发者 Theo 说自己在 Sol 上已经烧了 20 万美元的 token，吐槽 200 美元一个月的 Codex Pro 套餐特别容易撞限速。

一边是官方说输出 token 砍了一半，一边是自家用户在论坛喊消耗涨得更快了。这两件事同时挂在网上，谁都能查得到。我没法替 OpenAI 解释这个矛盾，但至少能看出，这次取消限速，跟「我们变高效了、把红利分给你」这种单纯的解释对不上号，更像是先把眼前撞墙的抱怨压下去，给自己争取时间。

另一边在拼硬件

聊到这我想起另一边的 Anthropic，正好可以拿来对照着看。5 月 6 号，Anthropic 把 Claude Code 的 5 小时限速永久翻了倍，Pro、Max、Team 到按席位算的企业版全覆盖，还顺带砍掉了 Pro 和 Max 高峰时段的限流。这次能这么大方，底气是跟 SpaceX 和 xAI 签的 Colossus 1 算力协议，一次性砸进 300 兆瓦以上、22 万块以上的英伟达 GPU。这几个数字我在之前一篇文章里核过一遍，没跑偏。GPU 是买断的固定资产，装机那天起就一直在撑着这个限速水位，不用每个月再单独花一笔钱续命，这也是它敢把政策直接钉成永久的底气所在。硬件这东西，买了就是买了，跑得动就一直跑得动，不像一句效率承诺，说变就能变。

5 月 13 号 Anthropic 又把 Claude Code 的周限额提了 50%，一直促销到 7 月 13 号，正好卡在这次 Codex 事件爆出来的前一天到期。好几家分析文章把这次调整定性成对 Codex 的防守动作，理由是 Codex 号称同样的活消耗 token 最多能省到四分之一。

对照一看，两家最近这几个月都在放宽限速，方向确实一致。但支撑这个动作的东西完全不一样。

两笔账根本不对等

我这人押过一次错注，而且当时押得挺有把握。上个月我笃定 Anthropic 会趁着算力紧张涨价加限制，结果人家转头就把 22 万块 GPU 租进来，直接把限速翻了倍。那次我彻底明白，判断一家公司敢不敢放宽限速，光看它嘴上怎么说是不够的，得看它背后真掏了什么本钱。后来面对号称最强的那版 Claude，我也是先看了眼价格才把手缩了回来。这本账我一直算得很紧。

放在这次对比里，两笔账根本不对等。Anthropic 那边是永久政策，背后是签了协议、真金白银砸进去的算力，查得到具体数字，验证得了。OpenAI 这次是临时措施，背后是一句还没经过第三方复核的效率承诺，自家论坛此刻正在吵它到底灵不灵。这事跟良心不良心没关系，是一个有硬件兜底，一个还只停在软件口头承诺这个阶段。真正该盯的不在他们各自嘴上说了什么，在这笔账有没有人替你核过。我自己那十几个 skill 跟 Claude Code 的脾气磨了大半年才稳住，换个底座这些规矩大概率得重新踩一遍坑，这也是我没立刻心动去试 Codex 的原因之一。

这笔账我先记着

以后再看到「限速放宽了」这种消息，我给自己定了三句话当尺子：这是永久政策还是临时措施；有没有独立机构或者第三方数据能对上；自己实测的账单跟官方说法对不对得上号。三条都过了才算数，过不了就当官方通稿看看就好，别急着当真事记进心里。

这次的 Codex，我打算再等等，等官方那个具体数字真的公布、等社区里的吐槽平息下去再说。你手头正好在跑 Codex 或者 ChatGPT Work 的，这几天有没有觉得账单消耗跟官方说的对不上号？评论区说说你那边的实际感受。
