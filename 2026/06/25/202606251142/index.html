Anthropic 说阿里巴巴「偷」了 Claude 的能力，我天天靠 Claude 吃饭，但这次选择先不站队

发表于 2026-06-25 更新于 2026-07-14 分类于 AI 阅读次数： 本文字数： 2.7k 阅读时长 ≈ 2 分钟
Anthropic 给美国国会和白宫写信，指控阿里巴巴用近 2.5 万个欺诈账户、2880 万次交互蒸馏提取 Claude 的能力。这是一封单方指控信，不是法庭定案，证据没公开，阿里也还没回应。我作为天天用 Claude Code 又刚测过国产模型的工程师，聊聊为什么在尘埃落定前我选择先按住不站队。

哈喽，我是飞飞。

昨天刷到一条消息，让我盯着屏幕愣了一下。Anthropic 给美国参议院银行委员会和白宫写了封信，指控阿里巴巴对 Claude 发动了「迄今最大规模的蒸馏攻击」，原话用了 brazenly（明目张胆）和 illicitly（非法）这两个词。

我愣，是因为我每天的活都压在 Claude 身上。我整条博客发布流水线、十几个写作 skill、那份越喂越厚的 CLAUDE.md，全长在 Claude Code 上面，每个月给 Anthropic 的订阅费不算少。所以看到「我赖以吃饭的东西被指出能力被人薅走」，第一反应是关心。

但愣完那一下，我决定先把情绪按住。因为这件事，远没有一个标题能讲清楚的那么干净。

先把事实的边界划清楚，别被标题带跑

我花了点时间去核原始信源，想搞明白到底发生了什么、哪些是事实、哪些只是说法。

能核到的是这些：Anthropic 在 6 月 10 号给国会和白宫递了一封信，说阿里巴巴和旗下的 Qwen 团队，从 4 月 22 号到 6 月 5 号，用了将近 2.5 万个欺诈账户，跟 Claude 产生了超过 2880 万次交互，专门冲着 Claude 最值钱的两块能力去：软件工程和 agent 推理。手法叫「对抗性蒸馏」，说白了就是反复拿精心设计的问题去喂一个强模型，把它的回答收集起来，再拿这些回答去训练一个便宜的竞品，让弱模型逼近强模型的本事。

听起来证据确凿对吧？但我必须把另一半也说清楚。

这是一封信，不是一纸诉状，也不是法庭判决。它是 Anthropic 单方面递给美国政府的一份指控。Anthropic 发言人被追问细节的时候，原话是「拒绝讨论具体情况」。也就是说，对外公开的只有账户数、交互数、时间窗这几个数字，那些能坐实蒸馏的技术取证、日志样本，一样都没拿出来。

阿里那边呢？CNBC 问，阿里没立即回应；Bloomberg 问，阿里说无可奉告。截至我写这篇，阿里对这桩蒸馏指控一个字的正式回应都没有。有意思的是，同一周阿里刚起诉了美国国防部，要把自己从一份黑名单上摘下来，对那件事它是公开硬刚的。对这件事的沉默，你可以读出很多种意思，但任何一种都还只是猜。

所以到这一步，我能给的结论很朴素：目前公开的信息很稀薄，基本只有 Anthropic 这一方的说法，关键证据和细节都没摆到台面上。我不想用脑补的情节去填这个空。信息不够，本身就是这件事现在最真实的样子。

「蒸馏」这事，追到底大家手都不太干净

为什么我不愿意顺着标题就喊「阿里偷了 Claude」？因为「模型能力提取」这摊水，在整个 AI 行业里早就是一片谁都不太干净的灰色地带。

先说蒸馏到底是什么。它跟黑客撞库、拷走源代码是两码事。它是拿强模型的输出当老师，去教一个弱模型。你反复问一个聪明模型，把它的答案攒下来，再用这堆答案去喂自己那个笨一点的模型，让它慢慢学到聪明模型的回答路数。问题在于，一旦一个模型的输出被公开发布出去，后面所有人训练的时候都会自然把它吸进去。这就让「故意蒸馏」和「普通训练里数据正常重叠」变得极难分辨。没有内部审计、没有 logit 比对、没有 tokenizer 取证，几乎没法把账精确算到谁头上。这也是为什么这类指控吵来吵去，最后往往谁也拿不出一锤定音的硬证据。

而且，这种指控在行业里是反复上演的剧目，方向还不止一个。今年 2 月，Anthropic 同样指控过 DeepSeek、Moonshot、MiniMax 三家，说它们合计用两万多个假账户蒸馏 Claude。再往前，OpenAI 也向美国议员递信，指控 DeepSeek 用混淆手段蒸馏自己的模型。你看，连美国的实验室之间、对中国的实验室，指控都是来回飞的。

这里我得交代一段我自己的一手观察。前阵子我真装真跑过小米的 MiMo Code，扒它的设计扒得挺细。它的独立裁判机制很像 Claude Code 的 /goal，它的官方文档干脆明说「兼容 Anthropic Dynamic Workflow 的核心语义」。我当时的体感很清楚：国产工具借鉴、对齐、追赶 Claude 趟出来的那条路，是普遍现象，普遍到几乎是这个阶段的默认动作。但「借鉴成熟设计」和「非法蒸馏盗取能力」之间，隔着一条又粗又模糊的线。这条线落在哪，恰恰是没有证据就掰扯不清的地方。

所以我对「国产模型到底是真本事还是有水分」这个问题，本来就有自己的一手观察。写字节 Seed2.1 的时候我核到它自报对标 Opus、胜率 59.1%，写阿里 Qwen 的时候我也核过它那些自报跑分。我对里头的水分一直是留着一只眼睛的。但留一只眼睛，跟在没证据时就给人定罪，是两回事。

站在我这种用户的位置，我到底该慌不该慌

聊回我自己。这件事如果属实，对我这种天天靠 Claude 干活的人，实际影响是什么？

老实讲，影响没那么直接。我用 Claude，看的是它当下解决我问题的能力，这个能力今天不会因为一封指控信就缩水。它该好用还是好用。我也不会因为这条消息，就对我刚测过的那些国产模型整个改观，毕竟我对它们的判断是亲手跑出来的，不是靠别人一句指控喂给我的。

真正让我在意的，是这件事掀开的那个更大的盖子。前沿模型的能力，正在变成一种可以被大规模、自动化「薅」走的东西。2880 万次交互，2.5 万个账户，这个规模本身就说明，护城河没大家想的那么宽。强模型烧几亿美金训出来的本事，理论上能被人用零头的成本逼近。这对花了血本做研发的公司是真实的痛，对整个行业怎么定规矩，也是个绕不开的难题。

而且这里头有个让我有点出戏的地方。Anthropic 在信里还补了一句，说用这种方式蒸馏出来的模型，往往缺少安全护栏，所以这不只是知识产权的事，还牵扯到更广的安全风险。这话听着是有道理的，但它递信的对象是国会和白宫，落点又踩在中美 AI 竞争这根最敏感的弦上。它讲的安全风险我不否认。我只是觉得，当一份商业指控同时被包装成国家安全议题、又只摆数字不摆证据的时候，我作为旁观者更得提醒自己多留个心眼，别被叙事的力度替换掉对证据的要求。

但难题归难题，定论归定论。在只有一方说法、证据没公开、当事人没回应之前，我能做的最负责任的事，就是先按住不站队。不站 Anthropic 那边喊「实锤了」，也不站阿里那边喊「莫须有」。把已知和未知分清楚，比急着站个队要难，但也诚实得多。

等阿里回应了，等更多证据摆出来了，这件事会清楚很多。在那之前，我宁可让自己的判断温吞一点，也不想为了一句爽快话，把一条单方指控说成板上钉钉的事实。

你呢？当你依赖的工具被卷进这种说不清的指控里，你是会先信指控，还是会跟我一样先按住等等看？评论区聊聊，我挺想知道你这只眼睛是怎么留的。
