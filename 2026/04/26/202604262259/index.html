Claude 桌面客户端有个开发者模式，把国产大模型直接接进 Cowork 界面

发表于 2026-04-26 更新于 2026-06-25 分类于 教程 阅读次数： 本文字数： 4.9k 阅读时长 ≈ 4 分钟
教程介绍 Claude Desktop 藏在 Help 菜单的开发者模式，经 Enable Developer Mode 配置 Anthropic 兼容端点后可不登录账号直接用 Cowork 界面跑国产模型，附 DeepSeek、MiMo、GLM、Kimi、阿里百炼、MiniMax 七家兼容 Base URL 速查表，全程约 4 分钟无需命令行。

大家好，我是飞飞。

昨天那篇 MiMo-V2.5-Pro 的文章发出去之后，群里跳出来一个我没料到的问题：能不能不开 Claude Pro 订阅，直接把国产大模型接到 Claude 桌面客户端的 Cowork 界面里用？

我第一反应是不行。Claude Desktop 我装了大半年，从来没看到任何关于自定义端点的入口。那个登录页面就是登录页面，没账号你连主界面都进不去。

今天翻到蓝点网那篇文章才发现，Anthropic 自己在 Help 菜单里藏了一个开发者模式。打开之后能配第三方推理端点，连 Claude 账号都不用登。蓝点网那篇文章的源头是 Linux.do 上一个叫 @Donnad 的网友，他翻 Anthropic 的官方文档时挖出来的。Anthropic 自己的 support.claude.com 上其实有这个文档，标题叫”Install and configure Claude Cowork with third-party platforms”，原本是给企业 IT 管理员做 MDM 部署用的（让 Claude Desktop 连企业自建的 AWS Bedrock、Google Cloud Vertex AI、Azure Foundry 这种私有推理服务）。但只要端点协议是 Anthropic 兼容的，理论上接什么都行。

国产大模型这边正好这两年纷纷推 Anthropic 兼容接口。这条路就通了。

下面是我今天下午把 DeepSeek V4 接进去之后整理出来的完整流程。

这件事到底解决了什么问题

先说为什么我觉得这个开关值得专门写一篇。

这周我连着写了 Copilot 停 Pro 注册、Claude Code 从 $20 套餐里拿走、GPT-5.5、DeepSeek V4、MiMo-V2.5-Pro 五件事。中间一条贯穿的主线是：海外模型订阅在变贵、变难拿，国产模型在变便宜、变好用。

但有一个心理门槛一直没破：Claude Desktop 那个 Cowork 界面是我用过最舒服的本地 AI 客户端之一。文件拖进去就读、Code Tab 自带工具调用、本地会话历史……这些东西在 Cherry Studio、Chatbox 这类替代品里都能凑齐，但凑不出 Claude 官方那个手感。

所以一直以来的取舍是：要么花 $20 开 Pro 订阅享受这个 UI，要么放弃这个 UI 用国产模型省钱。

开发者模式这一开，第三种路径出来了：保留 Claude Desktop 的 UI，把推理走 DeepSeek 或者 MiMo。订阅省了，UI 留了，按 token 实付。这个组合在两周前我会觉得不可能，今天可以了。

唯一的代价是你不能用 Claude Sonnet 4.6 或 Opus 4.7 这两个模型本身（除非你的中转站包了它们）。换来的是国产模型的 token 价格通常是这两个模型的 1/8 到 1/10。

把开发者模式打开的实际操作

我把 Claude Desktop 关掉重新打开，在登录页面上方的菜单栏依次走 Help → Troubleshooting → Enable Developer Mode。点完之后界面没有任何动静，但菜单栏里多出来一个 Developer 项。

这一步如果你在 Windows 上做，菜单按钮可能点不动（蓝点网那篇里专门提了一句）。绕开方法是先在登录页面里点输入框获取焦点，然后用 Tab 键切到菜单按钮，按回车键打开菜单，再用鼠标选下一级。我在 macOS 上没碰到这个问题，鼠标直接点下来就行。

接下来要做的事是完全退出 Claude Desktop。光最小化没用，得 Cmd+Q 退出。然后再开一次。这次菜单栏里 Developer 项是亮的，点开它，里面有一项 Configure Third-Party Inference。

点进去会弹一个配置面板。最重要的几个字段：

  • Inference Provider：默认是 Anthropic Compatible，意味着你的端点必须实现 Anthropic 的 /v1/messages 协议。如果你的中转站走的是 AWS Bedrock、Google Cloud Vertex AI、Azure AI Foundry 这三家任何一个，可以切到对应选项；接国产模型就保持默认。
  • API Base URL：填你那个国产模型的 Anthropic 兼容端点（下一节会列）。
  • API Key：从模型供应商控制台拿到的密钥。

填完之后点底部的 Apply locally。Claude Desktop 会再次让你完全退出。

第三次启动的时候，登录界面下方会多出一行字”继续使用本地网关”（英文版叫 Continue with local gateway）。点这行，不用任何 Anthropic 账号，直接进 Cowork 主界面。

这就完了。整个流程我在 macOS 上做完一共大概 4 分钟，没有任何命令行操作。

国产大模型的 Anthropic 兼容端点速查

这部分是这篇文章我打算放最久的一节。我把今天能确认的 Anthropic 兼容端点列出来，每条都来自模型官方文档或 SDK 源码，不掺中转站。

模型 Anthropic 兼容 Base URL 控制台
DeepSeek V4 / V4-Pro / V4-Flash https://api.deepseek.com/anthropic platform.deepseek.com
小米 MiMo-V2.5-Pro / V2.5-Flash https://api.xiaomimimo.com/anthropic platform.xiaomimimo.com
智谱 GLM-5 / GLM-5.1（国内） https://open.bigmodel.cn/api/anthropic open.bigmodel.cn
智谱 GLM-5 / GLM-5.1（国际） https://api.z.ai/api/anthropic z.ai
月之暗面 Kimi K2 / K2.5 / K2.6 https://api.moonshot.cn/anthropic platform.moonshot.cn
阿里百炼（Qwen3.6 / Qwen3-Coder） https://dashscope.aliyuncs.com/apps/anthropic bailian.console.aliyun.com
MiniMax M2.7 https://api.minimax.io/anthropic minimaxi.com

几个细节坑要单独提一下：

智谱这家比较特别，它不能用 ANTHROPIC_API_KEY，必须用 ANTHROPIC_AUTH_TOKEN。Claude Desktop 的配置面板里”API Key”那一栏其实对应的就是 token，填进去就好，不用纠结叫法。但如果你之后想在 Claude Code CLI 里同时配多个模型，这个区别会让你折腾两小时。

阿里百炼如果你用国际版（新加坡区），URL 后缀同样是 /apps/anthropic，但前缀换成 https://dashscope-intl.aliyuncs.com。API Key 和 Base URL 必须地域匹配，错了就 403。

DeepSeek V4 接进 Claude Desktop 的时候，思考模式默认是开着的，reasoning_effort 默认是 max。你如果只是聊天用，会感觉响应有点慢，可以考虑切换到 V4-Flash。

MiMo 这边公测期 Token Plan 的 API Key 和按量计费的 API Key 是分开的，要注意你拿的是哪一种。

我接 DeepSeek V4 的实际感受

我今天下午接的是 DeepSeek V4。挑它的原因有两个：昨天 V4 刚发，价格 ¥12/¥24 per M token 在国产里算中等偏低；昨天我写的那篇 V4 文章里说要把内容创作主力切过去，正好今天接进 Claude Desktop 试试 Cowork 的真实手感。

接上之后第一感觉是 Cowork 的 UI 没有任何变化。文件拖进去能读、复制图片能粘贴（虽然 V4 不是多模态，图片会被忽略并提示一下）、Code Tab 切过去能直接跑工具调用、左边栏的会话历史依然存在本地。Anthropic 文档里说”conversation history stays on the user’s local device”这句是真的，我把网络断了之后老会话还能翻出来读。

跑了三个我平时常用的任务：

  • 读一篇昨天写的 GPT-5.5 文章，让它做风格仿写。
  • 让它生成一份 Hexo 博客的 SEO meta tag 优化建议。
  • 让它修改我的一个 content-distributor skill 里的 prompt 模板。

中文写作那一块跟我昨天直接在 DeepSeek 官网开会话的体验差不多，文字感觉一致。但 Code Tab 里跑工具调用的时候，明显能看出 V4 比 Sonnet 4.6 在”先看再做”这个动作上慢一拍。Sonnet 习惯先 ls 一下、读两个文件再动手；V4 倾向于直接按它的理解动手，错了再回来改。这个习惯差异在 Cowork 短任务里影响不大，但在 Code Tab 长任务里会拖时间。

整体感受是：作为一个不订阅 Claude Pro 的临时方案，这条路完全跑得通。但作为长期主力方案，我个人还是会回到 Claude Max 5x + Sonnet 4.6 的组合。

这个开关让我重新想了一遍 $20 这个档位

写到这里我想留一个真实判断。

Claude Pro 那 $20 一档过去的核心价值有三块：Claude Desktop 的 UI、Sonnet/Opus 这两个模型本身、以及 Claude Code 的额度。本周 Anthropic 把第三块从 Pro 里拿掉了；今天这个开发者模式让第一块也不再是 Pro 独占的。

剩下的就只有”Sonnet/Opus 模型本身”这一块还属于 Pro 独占。

如果你过去开 Pro 主要看中的是 Claude Desktop 的工作流而不是模型本身，那今天这条路确实让 Pro 那 $20 显得没那么必要。换到一个国产模型 + Anthropic 兼容端点，按量付费，每月烧 ¥20 到 ¥40 人民币就能覆盖中等使用强度。这是 Pro 月费的 1/4 不到。

但这个判断有两个前提需要明说。

一个是你的工作流里多模态权重多大。Cowork 把图片塞进去这件事，能用的模型必须支持多模态。DeepSeek V4 没有，MiMo-V2.5-Pro 没有，GLM-5.1 有但不够强，Kimi K2.6 有但比较新。如果多模态是你的核心使用场景，留 Sonnet 4.6 还是当前最好的选择。

另一个是 Anthropic 这条路他们能开多久。Configure Third-Party Inference 这个开关原本是给企业 MDM 部署用的。一旦个人用户大规模拿它薅免费 UI 的羊毛，Anthropic 完全有可能在某个版本里加一道账号校验。所以现在用着的人最好做好备份方案的心理准备。

我接下来一周想跑的几个对照测试

写这篇之前我想了一下，要把这件事真正用清楚，光配置成功还不够。我接下来一周打算做这三件事。

把同一个 prompt 在 Cowork 里分别用 DeepSeek V4、MiMo-V2.5-Pro、GLM-5.1 各跑五次，记录响应延迟、token 消耗、错误率。这种小样本不能盖棺定论，但至少能看出在 Claude Desktop 这种短任务场景里哪家是最稳的。

把 Code Tab 的工具调用交给 V4 和 MiMo 各跑一次同样的修改任务（改我那个 RSS feed 生成脚本的描述字段映射）。看哪家能在没有 Claude Code 的工具调用框架辅助下，用 Cowork 自带的工具调用稳定完成。

跑完之后回来更新这篇文章。如果你在跑相同的对照，欢迎评论区交流。

最后留两个我想知道答案的问题。

你今天会去打开 Claude Desktop 的开发者模式吗？如果会的话，你会先接哪一家？我猜大部分人会先接 DeepSeek V4，但如果有人先接了 GLM-5.1 或 MiMo-V2.5-Pro，我想听你和官方端点对接的过程里有没有什么坑。

还有一个我自己想问的：你过去开 Claude Pro 的核心理由是什么？是模型本身，是 Cowork 的 UI，还是 Claude Code 的额度？这三块今天剩下两块的护城河了，我想看看大家的取舍点到底在哪里。
