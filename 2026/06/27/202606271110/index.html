打造你的第二大脑：从知识乱堆到能复用、还能喂给 AI 的个人知识库

发表于 2026-06-27 更新于 2026-07-18 分类于 工具 阅读次数： 本文字数： 2.9k 阅读时长 ≈ 3 分钟
第二大脑不是再装一个笔记软件，是把知识攒成能复用、能喂给 AI 的系统。本文讲清 Obsidian、Notion、飞书知识库三类工具怎么选，给出 PARA 分层和回沉动作，并用我那个攒了大半年、分成五层、被写作 agent 直接当上下文读的真实 Obsidian 库做示范。

哈喽，我是飞飞。

我有一套写文章的 agent 流水线。它每次动笔前，不急着上网搜资料，先翻我自己的一个库。这个库攒了大半年，分了五层，里面是我写每一篇文章时沉下来的研究素材、洞察、一手踩坑记录。AI 不用我临时喂，自己就读进去了。今天想聊的，就是这个库怎么搭。

说白了，就是怎么给自己建一个「第二大脑」。这个词你大概听过，它的核心判断我先甩在这：别把它理解成又装一个笔记软件，它的要害是把你脑子外面的知识，攒成一个能长期沉淀、能被复用、还能喂给 AI 的系统。工具是末节，沉得下来才是要害。

这里要先和我之前一篇文章撇清。基础系列里我写过「零基础搭 AI 知识库」，那篇讲的是 RAG，怎么把资料喂给 AI 让它读，是站在 AI 这一侧。今天这篇反过来，讲的是人这一侧：怎么先把资料攒成一个值得你自己读、能复用的第二大脑。AI 只是这个库的一个消费端，不是主角。一句话，上次讲怎么让 AI 读你的资料，这次讲怎么先把资料攒成一个值得读的库。

第二大脑到底解决什么问题

少数派上有个比喻我很认同。第一大脑就是你脑袋里这台 CPU，活着它就一直在转，又要管知识、又要管任务、又要管目标。任务一多，它就开始抢占算力，处理当前这件事的时候还得保持别的事的上下文，结果哪件都做不专注。

第二大脑就是给这台 CPU 外挂一块硬盘。把不用随时调用的东西卸下来，存进一个外部系统，脑子腾出来只管思考和创造。

这套说法最系统的版本来自 Tiago Forte 的《打造第二大脑》。他提出一个 CODE 四步法：Capture 捕获、Organize 组织、Distill 提炼、Express 表达。他反复强调，这套东西的重点不在记笔记，而在把数字混乱收拾成一个能用的系统。书里还有个更出名的分类法叫 PARA，把所有东西归进四个抽屉：Projects 项目、Areas 领域、Resources 资源、Archives 归档。这套抽屉能套在任何工具上。

我想提醒你的是，别一上来就盯着方法论钻牛角尖。Forte 官方说这套体系是工具无关的，他的学员有用 Evernote 的、用 Notion 的、用 Obsidian 的，两万五千多人各用各的。方法是骨架，先动起来比先想清楚重要。

知识库为什么大多数人搭着搭着就废了

这部分我得坦白，我自己也踩过坑，这套打法我到现在还在调。我观察下来，第二大脑搭废，基本逃不开三个坑。

头一个坑，过度设计。还没存几条东西，先花一晚上设计文件夹层级、调标签体系，把系统搭得无比精致，然后就没有然后了。Forte 把这叫 over-engineering，在真正用它之前就把它设计死了。

另一个坑，工具反复横跳。今天看到有人吹 Obsidian 就搬过去，明天看到 Notion 出新功能又搬回来，后天 Roam 火了再迁一次。每迁一次，进度归零，库永远是空的。有人总结得很狠：最好的系统就是你正在用的那个。建议很简单，一个工具至少用满半年再说换。

最后这个坑最隐蔽，只存不取。看到好文章就剪藏，看到好观点就收藏，存了几千条，要用的时候一条都找不到。有句话点醒我：如果你找不到它，它就等于不存在。知识库的价值不在你存了多少，在你能不能在需要的那一刻把它捞出来。

这三个坑串起来其实是一句话：长期积累能力，等于一个工具用够久，加上每次产出都回沉进去，加上下次能被检索复用。缺一个，库就是死的。

Obsidian、Notion、飞书知识库，到底怎么选

工具我不想给你列长表对照，那种你看完还是不知道选哪个。我按「你是谁、要干嘛」给你分。

我自己重度在用的是 Obsidian，先说它。它最大的特点是本地纯 Markdown 文件，你的笔记就是一堆文本文件躺在你硬盘上，数据完全是你的。它的灵魂是双链和关系图谱，A 笔记里提到一个概念，链到 B 笔记，时间长了这些链接会织成一张网。社区有八万多人，插件生态很猛。代价是它没有开箱即用的模板，PARA 得自己搭，手机同步要么买它官方的 Sync，要么自己用 iCloud、Git 折腾。一句话，它适合愿意折腾、想要数据完全自主、做个人深度积累的人。

Notion 我也想替它说句公道话。你要是想多人协作、想开箱即用、又懒得折腾，那 Notion 确实更对。它数据库、模板、PARA 文件夹全是现成的，Forte 官方就出了 Notion 模板，一小时就能搭起来。它的短板是搜索慢，Forte 自己都吐槽过它检索又慢又笨重；还有迁移难，有个用户在 Notion 里沉了一千八百多页，想搬出来得花几个月。

飞书知识库是另一个物种。它是给组织和团队用的，页面树结构，权限能精细管控，最大的好处是业务经验「不随人走」，人离职了知识还在库里。但拿它做个人笔记，知乎上不少实测说它太重、加载慢。所以飞书知识库的位置很清楚：团队沉淀选它，个人单用偏重。

让 AI 真能读你的库，这一步 Obsidian 占便宜

到这是这篇和那篇 RAG 文章真正分岔的地方。我前面说第二大脑要「能喂给 AI」，这一步不同工具差别很大。

关键在格式。Obsidian 的库是一堆纯 Markdown 文本，Claude Code、MCP 这类 AI agent 能直接读、直接写，你甚至能跑个本地模型读它，数据一步都不出你的机器。Notion 不一样，想让 AI 读你的 Notion，得配 API key、写数据库查询、搭一层中间件翻译格式，每次请求都要绕到 Notion 服务器转一圈再回来。

这就是为什么我那套写作 agent 选了 Obsidian。我的库分了五层：00 一手经验、01 信源、02 洞察、03 成稿、05 封面 prompt 存档。我写每一篇文章，研究素材沉进 01，提炼出的判断沉进 02，写完的成稿进 03，连封面用的 prompt 都存进 05。下次写相关主题，agent 直接读这几层当上下文，不用我重新搜一遍。

跟你说个具体的。光是把这套库的目录结构和命名规则调顺、让 agent 每次都能稳定读对层，我前前后后折腾了好几个晚上。搭起来只是开头，真正的功夫在用着用着发现哪层乱了再回去改。这种细节，换个没真跑过这套流水线的人写不出来。但也正是这个库，让我那篇 RAG 文章里说的「喂资料给 AI」有了真东西可喂。先有一个值得读的第二大脑，喂给 AI 才有意义。

给你一个今天就能动手的最小起步

别等想清楚再开始，照下面这个最小动作走就行。

第一步，挑一个工具，挑完别再换。个人深度积累、想喂 AI、要数据自主，选 Obsidian；要协作、要省事，选 Notion；团队沉淀，选飞书知识库。

第二步，先建四个文件夹就够了，对应 PARA：在做的项目、长期负责的领域、收集的资源、归档。别一上来搭十几层。

第三步，立个回沉的规矩。每做完一件事、读完一篇值得记的东西，逼自己写两三句沉进对应文件夹。写得漂不漂亮无所谓，关键是养成「产出必回沉」的肌肉记忆。

第四步，半年后再回头看。那时你的库里会自然长出一些你都忘了记过的关联，捞出来正好能解释你眼下遇到的问题。这一刻，第二大脑才算真正活了。

绕回开头那个我攒了大半年的库。它一开始也就是几个空文件夹，全靠我一篇篇文章往里沉，才长成现在能直接喂给 agent 的样子。第二大脑从来都不是一次搭好的，得靠你一天天用出来。

最后留个问题给你：你现在是更卡在「不知道选哪个工具」，还是更卡在「工具有了但东西沉不进去」？评论区聊聊，我看看大家是哪个坑里的人多。
