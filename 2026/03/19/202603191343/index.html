微信发条消息就能指挥电脑干活：腾讯QClaw把"养龙虾"变成了全民运动

发表于 2026-03-19 更新于 2026-04-21 分类于 阅读 阅读次数： 本文字数： 3.4k 阅读时长 ≈ 3 分钟

你在地铁上，掏出手机，打开微信，给”QClaw管家”发了一条消息：”把桌面上那个 Excel 文件整理一下，按销售额排序，生成一份图表，存到 Documents 目录。”

手机揣回口袋。到公司打开电脑，文件已经做好了。

这不是未来畅想。这是腾讯刚刚推出的 QClaw 正在做的事。

3 月 9 日，腾讯开始内测 QClaw——一个基于 OpenClaw 的一键部署工具，直连微信和 QQ。3 月 18 日，公测版上线，微信小程序入口全面升级。南华早报（SCMP）的标题很直接：**”龙虾热潮席卷中国，腾讯把 OpenClaw 搬进了微信。”**

Business Insider 的报道更猛：中国科技巨头正在疯抢 OpenClaw 的金矿。

今天我们来拆解：QClaw 到底是什么？腾讯凭什么切入这场”养龙虾”大战？这件事为什么比你想的重要得多？

QClaw 是什么：三层架构，一句话说清

QClaw 不是腾讯从零造的新框架。它是 围绕 OpenClaw 做的一次产品化封装。

你可以把 QClaw 理解成三层东西叠在一起：

最底层：OpenClaw 的能力底座。 文件操作、网页浏览、代码执行、工具调用——所有 OpenClaw 能做的事，QClaw 都能做。5000+ Skills 生态直接继承。

中间层：腾讯做的部署和路由。 一键安装、模型预配置、安全沙箱。不用命令行，不用配 API Key，不用折腾环境。三分钟，三步操作，零代码部署。

最上层：微信和 QQ 入口。 这才是最关键的。扫码绑定后，你在微信里给”QClaw管家”发消息，它就在你的电脑上执行任务。

36氪的评价一针见血：很多人把 QClaw 的亮点理解成”终于不用命令行装 OpenClaw 了”。不对。这不是工具升级，这是入口升级。

为什么是微信：14 亿用户的 AI Agent 入口

OpenClaw 火了之后，飞书、钉钉都在接入。但它们本质上还是”办公里的 AI”。

微信和 QQ 不一样。

微信月活 14 亿。它覆盖的不是单一办公场景，而是用户最日常、最频繁、最生活化的沟通入口。你跟老板聊天用微信，跟朋友约饭用微信，给家人发红包用微信。

一旦 AI Agent 进了微信，产品性质就变了。 它不再是”你打开一个特定的应用去用 AI”，而是”你在最常用的聊天窗口里顺手就能让 AI 干活”。

这就是为什么 QClaw 选择做微信小程序。用户不需要安装新应用，不需要学新界面。微信搜索”QClaw管家”，扫码绑定，就完事了。

飞书能把 OpenClaw 变成数字同事。但飞书做不到把 OpenClaw 送进中国用户最核心的私人聊天场景里。这是腾讯独有的牌。

怎么用：三步上手

第 1 步：下载安装 QClaw

访问 claw.guanjia.qq.com，下载 QClaw 客户端（支持 Mac 和 Windows）。安装后一键启动。

如果你已经装过 OpenClaw，QClaw 可以直接关联已有环境，不用重新配置。

第 2 步：选择模型

QClaw 内置了多个国产大模型：

模型 特点
Kimi-2.5 默认模型，长上下文能力强
MiniMax-M2.5 编码能力突出，性价比高
GLM-5 中文优化，智谱出品
DeepSeek-V3.2 极致便宜

默认模型限时免费。也支持自定义接入其他模型。

第 3 步：绑定微信

在 QClaw 客户端扫码，绑定微信账号。之后在微信小程序”QClaw管家”里就能直接给 AI 发指令。

就这样。从下载到能用，三分钟。

能做什么：四个真实场景

场景 1：远程文件处理

你在外面跑客户，突然需要一份报告。掏出手机，在微信里说：”把 ~/Documents/Q1 目录下的所有 CSV 合并成一个 Excel，加一个汇总 sheet，发给我。”

QClaw 在电脑上执行，做完把文件通过微信小程序发回给你。

场景 2：社媒运营自动化

你是自媒体从业者。让 QClaw 自动生成小红书笔记、一键发布、自动回复评论互动。不用时刻盯着手机。

场景 3：学术研究

你是研究生，面对文献综述任务。QClaw 自动检索近五年指定主题的论文，筛选高引文献，整理成格式规范的综述文档，导出 PDF。

场景 4：日常效率

“帮我整理桌面上的文件，按类型分类到子文件夹里。”
“把最近一周的浏览器书签整理成 Markdown 列表。”
“下载这三个网址的文章，总结成一份摘要。”

这些你一直想做但懒得做的事，现在发条微信就搞定了。

安全：腾讯电脑管家的老本行

把 AI Agent 放到电脑上跑，安全是最大的担忧。今年早些时候的 ClawHavoc 供应链投毒事件，341 个恶意 Skills 被植入 OpenClaw 生态，API Key 被盗、信用卡被刷爆——这些都是真实发生的事。

QClaw 的安全方案来自腾讯电脑管家团队。他们在 18.0 版本里推出了 **”AI 安全沙箱”**——也叫”龙虾管家”：

隔离运行： AI Agent 在沙箱内执行，不能直接触碰系统核心文件。

全程防护： 对 Skills 安装、网络请求、文件操作进行实时监控和拦截。覆盖系统安全、Skills 安全、支付安全、Prompt 安全四个维度。

行为可溯： 所有 Agent 操作有完整日志，出了问题可以回溯。

约束性安全规则： QClaw 内置了一套安全规则，限制 AI Agent 的高权限操作。比如禁止未经确认的文件删除、禁止访问敏感目录、禁止未授权的网络请求。

这是大公司介入开源生态后带来的核心价值——开源提供能力，大厂提供安全。

养龙虾大战：腾讯不是一个人在战斗

QClaw 不是孤例。整个中国科技圈都在抢 OpenClaw 的生态位：

公司 产品 策略
腾讯 QClaw 微信/QQ 入口，本地部署
字节跳动 ArkClaw 火山引擎云端部署，浏览器即用
百度 DuClaw 零部署 OpenClaw 云服务
阿里 CoPaw 通义千问生态绑定
智谱 AutoClaude GLM-5 深度整合
网易 LobsterAI 100% 开源（MIT），主打透明

Business Insider 报道说：”中国人排队到腾讯深圳总部和百度北京办公室门口，找工程师帮忙装 OpenClaw。”

“养龙虾”成了全民运动。而 QClaw 做的事，就是把这个原本需要排队找工程师帮忙的东西，变成了下载一个应用、扫一个码就能用的东西。

QClaw 的真正对手不是 ArkClaw

很多分析把 QClaw、ArkClaw、DuClaw 放在一起比。但 36氪 那篇文章看得更深：QClaw 的真正对手不是其他龙虾产品，而是 Anthropic 的 Claude Cowork + Dispatch。

两个产品的核心逻辑非常像：

  • 都不是单纯卖模型，而是”OpenClaw/Cowork + 高频入口“
  • 都主打”手机发消息，电脑干活”
  • 都强调本地执行、数据不出设备

区别在于：Claude Dispatch 走的是全球市场，入口是 Claude App。QClaw 走的是中国市场，入口是微信。

一个靠工作上下文建立壁垒，一个靠 14 亿微信用户建立黏性。 它们不是简单的替代关系，而是在抢 Agent 时代两种不同但都极其值钱的位置。

值得关注的风险

公平起见，QClaw 也有风险。

第一，生态碎片化。 QClaw、ArkClaw、DuClaw 都是 OpenClaw 的封装版本，但它们互不兼容。Skills 生态可能会分裂。对开发者来说，”一次开发，到处运行”的 OpenClaw 理念正在被大厂的定制化侵蚀。

第二，安全仍是硬伤。 虽然腾讯加了安全沙箱，但 AI Agent 操控电脑的本质风险没有消失。错误的指令可能导致文件丢失、隐私泄露。特别是通过微信远程控制——如果微信账号被盗，攻击者就等于拿到了你电脑的遥控器。

第三，模型能力瓶颈。 QClaw 默认的国产模型在复杂推理任务上和 Claude Opus 还有差距。对于简单的文件整理和信息汇总够用，但如果你需要做复杂的代码开发或长文本分析，可能还需要切换到更强的模型。

写在最后

QClaw 最有意思的地方，不是技术，是入口。

OpenClaw 证明了 AI Agent 能做什么。IronClaw 和 ZeroClaw 在重写它的底层。NVIDIA NemoClaw 在给它加安全壳。而腾讯 QClaw 做了一件更”接地气”的事——把它塞进了 14 亿人每天都在用的微信里。

这就像智能手机时代，真正改变世界的不是处理器更快了，而是微信支付让路边摊老板也能扫码收钱。

QClaw 做的是同样的事：把一个原本属于极客的工具，变成了每个人都能用的日常功能。

当 AI Agent 的入口变成微信聊天框的时候，”养龙虾”就不再是程序员的事了。它是每一个微信用户的事。
