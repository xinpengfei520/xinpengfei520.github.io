K3 压过 Fable 5 只算个信号，还谈不上让我从 Claude Code 搬家

发表于 2026-07-17 更新于 2026-07-18 分类于 观点 阅读次数： 本文字数： 3.6k 阅读时长 ≈ 3 分钟
Kimi K3 在 Arena.ai 前端编码榜以 1679 分登顶，压过 Claude Fable 5 和 GPT-5.6 Sol。Moonshot 自己承认用户体验有明显差距。K3 定价对标 Sonnet 5，是 K2.6 的 3 到 4 倍。作为 Claude Code Max 用户，我判断这只是能力信号，7/27 开权重前不搬家。

哈喽，我是飞飞。

这两天推上朋友圈都在传一张图。Kimi K3 在 Arena.ai 的 Frontend Code Arena 榜单拿到 1679 分登顶，压过 Claude Fable 5 的 1631 和 GPT-5.6 Sol 的 1618。

标题一个比一个吓人。「开源终于翻天」「Claude 这次真要凉」「国产模型压制两大闭源」满屏都是。

你要是也天天用 Claude Code 干前端活，可能跟我一样第一反应是要不要开权重就搬家。我那天下午刚跑完一个博客 skill 的重构，顺手点开榜单看了半个钟头，发现有几件事跟标题不太对得上。

说实话，我扒完那半小时的判断是：K3 前端拿到头名是一个真实的能力信号，但它离「我该搬家」还差很远。

7/27 开权重之后我可能会去跑一轮真活对照。眼下这一张单图，跟 Claude Code Max 20x 一个月给我兜的那点补贴额度一比，动不了我。

那半小时我扒的东西很简单：这个榜单是什么榜单、Moonshot 官方自己怎么说、K3 到底怎么定价。三样看下来，「K3 碾压 Fable 5」的分量就会变。挨个说。

那榜单到底测什么

Frontend Code Arena 是 Arena.ai 从 LMArena 分出来的实时前端编码榜单。

机制说人话就是：用户提一个需求（比如「做一个国际象棋游戏」），两个匿名 AI 模型各造一个网页，用户交互完投谁做得好。最后分数用一套跟国际象棋 Elo 类似的算法（Bradley-Terry 模型）算出来。

截至 7/13 累计 469677 票、96 个模型。

问题在于用户提的都是什么 prompt。Arena.ai 自己把 Top 提问列出来了：VS Code 克隆 4189 次、WhatsApp 克隆 3385 次、国际象棋 3154 次、Hacker News 克隆 2740 次、Twitter profile 布局 2544 次。清一色 UI demo 型任务。

Arena.ai 官方在博客里说得很直白：这个榜单只看用户投票的观感，不管间距、可访问性、响应式布局、交互完整度这些真实前端要抠的东西。

Arena 官方还补了一句更狠的话：「Agents are an entirely different contest」。说人话就是，Agent 场景的能力跟这个榜单没关系。

所以 K3 榜单头名等于什么？说白了就是「人类投票觉得它生成的 Web App demo 更漂亮」。要不要把它翻译成「K3 能写好一个真实业务前端项目」，中间还差好几个台阶。

这个榜单本身有它的价值。能翻上头名说明 K3 在 UI demo 类任务里确实有能力。从 K2.6 的第 18 位一步跳到榜首，爬坡速度是真实的。可这跟真实工程活是两个世界。

官方都没这么说

翻 Kimi K3 的官方发布博客，你会看到一个跟朋友圈标题很不一样的口径。

官方自评的六项编码基准里，K3 赢了两项，其余多数落在二三档。跟 Fable 5 和 GPT-5.6 Sol 直接对比的部分，Moonshot 用了一句非常委婉的原话：「相对 Fable 5 和 GPT-5.6 Sol，K3 在用户体验上仍存在明显差距」。

第三方独立评测 Artificial Analysis 那边，K3 拿到 Intelligence Index 57 分，排整个榜单第 4 位。前面是 Fable 5 的 60、GPT-5.6 Sol 的 59、Claude Opus 4.8 的 56。K3 跟 Opus 4.8 只差 1 分。顺带一句，K3 的幻觉率比上代 K2.6 还高了 12 个百分点，准了但也更敢编了。

现在的局面挺有意思：媒体把 K3 抬得比 Moonshot 自己都高。你以为 K3 是全线压 Fable 5？连 K3 自己团队都没这么说。

定价才是真戏

K3 这次真正值得看的动作在定价。

$3 每百万输入 token，$15 每百万输出 token，缓存命中 $0.30。这个数字跟 Claude Sonnet 5 完全一样。上一代 K2.6 的定价是 $0.95/$4，K3 一步涨到了 3 到 4 倍。

横向对比更直观。K3 单任务成本 $0.94，跟 GPT-5.6 Sol 的 $1.04 基本一档，比 Opus 4.8 的 $1.80 便宜一半。可下面这一排开源模型：GLM-5.2 单任务 $0.32、DeepSeek V4 Pro $0.04。

K3 主动脱离了便宜阵营，向 Sonnet 档看齐。

The Decoder 有一句话我印象很深：这个动作在告诉市场，「中国便宜 AI 时代结束了」。翻译成人话：中国前沿模型开始按前沿定价卖了。K3 不打便宜牌了。

再叠一个 Simon Willison 那天做的实测。他拿 K3 画一只骑自行车的鹈鹕 SVG。用了 13241 个思考 token 生成 3417 tokens 响应，一发 25 美分。

原因是 K3 现在只有 max 一档 reasoning effort（说人话就是模型思考的力度只有「拉满」一档，想少一点都不行）。你就算让它画个简笔画，它也照 max 力度想。我看到这数字直接冒了一句：这玩意进我流水线就是个烧钱大户。

上周我让 agent 批量改 20 个博客文件的 frontmatter 字段，下意识就把 Opus 那档从心里划掉，切到 Haiku 走。这种「用之前先在心里过一遍秤」的绷着感，就是「贵」最真实的体感。

K3 的 max 单档设计等于把这道秤从我手里拿走了。让它去改一个变量名跟让它做一次架构级重构，烧的思考 token 密度一样。账单没法压。

我为什么不会迁过去

我这一年多的模型迁移判断，套的是同一个框架。之前写 GLM-5.2、小米 MiMoCode、xAI Grok Build 那三篇，用的都是这套。这次挪到 K3 上面，账是这样算的。

订阅补贴那本账我心里有数。我天天在 Claude Code Max 20x 里跑活，一个月 200 美元包量，跑到什么份儿上心里都有笔账。

任何按 API 单价做的迁移对比，对订阅制重度用户来说都要先打个 5 到 10 倍的折。K3 单任务 $0.94 看起来不贵。可我在 Claude Code 里跑同类活，Max 额度早就把边际成本摊掉一大半，K3 走 API 每次都是真金白银出账。

跟这层贴着的是 skill 生态迁移成本。我光写文章这一条流水线，就挂着十几个 skill：拉资讯的、查重的、配图的、发公众号的。每一个都是照着 Claude Code 读 skill、认 frontmatter、调 hook 的脾气一点点磨出来的。

真把底座换成别家，这些 skill 调工具时吐出来的格式十有八九会飘，得挨个重新校。光是那套专门防 AI 味的 grep 规则重新调顺就够搭进去好几个晚上。这笔时间账，省模型钱的那本从来不记。

上个月我给博客的自动发布流水线换 GitHub Actions v3 到 v5，就升了三条 workflow 的 checkout 和 setup-node，仅此而已。中间还翻了个车。

有一个 PERSONAL_TOKEN 静默过期，构建每次都成功，只有最后一步 git push 认证失败悄悄红了。我隔了整整一天才发现镜像站挂了。三条 workflow 就能翻这种车，十几个 skill 底座迁移我不敢想。

K3 自己的产品限制也是一块。它现在只有 max 一档思考力度。

我在 Claude Code 里可以按活分层：改变量名走低档、模块重构切 high、真难的架构级问题才拉 max。K3 一档意味着我让它干最琐碎的活它也照 max 想，账单不好压。前面说 Simon Willison 那只鹈鹕 25 美分，我流水线一天要跑几百次这种小任务。扔进 K3 一天光小活就得烧掉一顿饭钱。

权重要 7/27 才开。眼下所有分析都是 Moonshot 官方口径加一个第三方榜单，独立复现还没跑过。让我拿一张 Frontend Code Arena 榜单头名去决定搬家，风险跟性价比我都算不过来。

我之前写 GLM-5.2 那篇的时候提过一句：Claude Code 是三层叠加，底层 Opus 加十几个 skill 再加一摞 memory。国产模型只能替中间那层，真问法是「哪几类活值得换中间那层」。

这次 K3 也是同一个问法。前端 demo 类任务它有能力信号，可我日常的前端活 90% 长这样：在既有代码库里改一个组件、加一个 hook、抠一段响应式布局。画一个新 Web App 那种活我一年碰不到几次。这类活榜单没测过。

看下来，K3 有能力信号。可从我这套 Claude Code 长在骨头上的流水线看过去，这个信号还没到能让我搬家的重量。

等 7/27 再看一眼

7/27 开权重是我会认真去看的时间点。那天几件东西才会到位：本地跑测的独立数据、社区在真实业务前端项目上的实测（不是 VS Code 克隆那种）、榜单之外多一两份独立 benchmark。

还有一条我最关心的路：K3 能不能塞进 Claude Code 当第二底座跑（改一个环境变量 ANTHROPIC_BASE_URL 换后端）。GLM-5.2 那次我在这条路上是走通了的，K3 我不确定。

在这之前，我对 K3 的态度保持在「值得关注，不值得动」。这个榜单登顶挺好，可它离动我 skill 生态还差好几步。

我这一年多养出一个习惯：发布会 PPT 和一张榜单头名的截图，一律先放一放。等社区在真实业务活里跑一批我认识的人再决定。榜单是给厂商看的，我的流水线不是。

你这次会不会等 7/27，还是打算发布当天就开权重上机跑？
