不会写 Prompt？我天天靠精确指令驱动 Claude Code 跑流水线，把这套万能公式掏给你

发表于 2026-06-26 更新于 2026-07-21 分类于 教程 阅读次数： 本文字数： 2.9k 阅读时长 ≈ 3 分钟
普通人写 Prompt 答非所问，根上是把 AI 当百度搜索框敲关键词，没当成给同事派活。我每天靠一份上百条规则手册和一份越喂越厚的 CLAUDE.md 驱动 Claude Code 跑整条内容流水线，攒出一套小白也能抄的万能公式：角色加目标加背景加要求。文末三个改写前后对比，照着填就能用。

哈喽，我是飞飞。

先说一个我观察了很久的现象。很多人用 ChatGPT、豆包、DeepSeek，用着用着就放弃了，理由出奇地一致：「这玩意儿答非所问，还不如我自己写。」我去翻他们的对话记录，发现问题压根不在 AI 笨，在那句话本身。他们打给 AI 的字，跟在百度搜索框里敲关键词一模一样，是「帮我写个朋友圈」「写份周报」「起个标题」。

国外有个做提示词的社区里，有人把这事说得特别准：九成的烂回答，都来自同一个毛病，我们把 prompt 写成了搜索关键词。可你要的明明是一份派活的工作简报。这句话我记到现在。你想想，你要是这么跟一个刚入职的实习生派活，「帮我写个东西」，他大概率也是一脸懵，要么追着问你一堆，要么硬着头皮交一版你完全不想要的。AI 不会追问，它只会硬交。

我自己是把「怎么把需求说清楚」这件事干到了极致的人。我整条博客内容流水线，从拉资讯、选题、写稿、配图到发公众号，全是我用指令驱动 Claude Code 一步步跑出来的。为了让它每次都听懂，我维护着一份上百条的写作规则手册，还有一份越喂越厚的项目说明文件，里面写满了「你该用什么口吻、不许出现哪些词、字数卡在多少」。说白了，我每天上班干的核心活，就是「把需求讲清楚」。这套功夫攒下来，我提炼出一个连我妈都能上手的万能公式，今天毫无保留地给你。

先搞明白：AI 为什么老是答非所问

这事得先从 AI 的工作方式说起。你可以把大模型理解成一个见多识广、但完全不认识你的临时工。它读过的东西比谁都多，可它不知道你是谁、你要干嘛、你给谁看、你想要什么样的成品。它脑子里关于「朋友圈」的素材有几百万种可能，文艺的、搞笑的、卖货的、晒娃的，你只甩一句「帮我写个朋友圈」，它就只能从这几百万种里随便挑一个最大众的塞给你。

塞给你的那个，当然不对你胃口。它没那么笨，只是你压根没告诉它往哪个方向走。

我做内容流水线时对这点的体感特别深。同样让 AI 写一段话，我把背景和要求铺满，它一遍就过；我图省事少写两句，它就开始飘，写出来一股浓浓的「AI 腔」，全是「在当今时代」「赋能」「闭环」这种空话。我那份规则手册里，光是「禁止出现的词和句式」就攒了几十条，全是被它坑出来的。所以问题的解法很朴素：先别急着换更聪明的 AI，回过头把话说全。

万能公式：角色 + 目标 + 背景 + 要求

我把派活这件事拆成四块，缺哪块 AI 就在哪块上自由发挥。记住这四个词就够了。

角色，是让 AI 先站到对的位置上。你想让它像谁一样思考？「你是一个干了十年的资深 HR」和「你是一个会写段子的脱口秀编剧」，写出来的东西天差地别。给它一个身份，它就调出那个身份对应的那套话术和经验。

目标，是你到底要它产出什么。一句话说清楚成品是什么，「写一条朋友圈文案」「写一份给领导看的周报」。别让它猜你要的是一段话还是一篇长文。

背景，是你脑子里有、但它不知道的那些信息。你是谁、为什么写、给谁看、有什么前情。这一块是最多人漏掉的，也是拉开差距最大的一块。AI 没法读你的心，你不喂背景，它只能编。

要求，是成品的硬约束。多少字、什么口吻、要不要 emoji、不能出现什么。这一块决定了交付物能不能直接用。我那份项目说明文件里，最厚的部分就是「要求」，因为我发现，告诉 AI「不要做什么」往往比「要做什么」更管用，能直接堵掉它最爱犯的那些毛病。

四块凑齐，你这句话就从一个关键词，变成了一份合格的工作简报。下面上真东西。

改写一：发个朋友圈

很多人写朋友圈憋半天憋不出，丢给 AI 又嫌它写得假。我们看看差在哪。

改写前，大多数人是这么打的：

帮我写一条朋友圈，关于我今天跑了五公里。

AI 会回你一段很正确、很空、你绝对不会发出去的东西，大概是「今天挥洒汗水，五公里轻松拿下，自律的人生开挂啦」之类，配三个火苗 emoji。看着就尴尬。

套上公式改一下：

角色：你是一个文字接地气、不爱喊口号的普通上班族。
目标：写一条朋友圈。
背景：我平时几乎不运动，今天是被朋友硬拉去跑的，跑完第三公里就想放弃，最后咬牙跑完五公里，腿现在还在抖。
要求：50 字以内，口语化，带点自嘲，不要任何励志金句，最多一个 emoji。

这一版出来的东西就有人味了，大概是「被朋友骗去跑了五公里，第三公里就想原地去世，活着回来了，腿不是我的了」。差别在哪？背景里那句「腿现在还在抖」「咬牙跑完」是只有你知道的真实细节，AI 抓住它就有了具体的画面，而不是空喊自律。

改写二：写一份周报

这是职场人最高频、也最容易写废的场景。

改写前：

帮我写一份这周的工作周报。

AI 完全不知道你这周干了啥，只能给你一个填空模板，「本周完成了 XX 工作，推进了 YY 项目」，等于没写。

套公式：

角色：你是一个汇报逻辑清晰、懂得突出重点的职场老手。
目标：把我这周的流水账整理成一份给直属领导看的周报。
背景：我是做新媒体运营的，这周主要干了三件事，发了 5 篇公众号推文其中一篇阅读破万，跟两个达人谈了合作待签约，还把后台粉丝从 8000 涨到了 8600。下周要开始筹备一场直播。
要求：分「本周成果」和「下周计划」两块，成果部分把破万那篇和涨粉数据放前面，语气平实不浮夸，控制在 200 字内。

你看，背景里那几个具体数字，破万、8000 涨到 8600，是周报的命根子。很多人写周报写得心虚，就是因为只会写「做了什么」，不会写「做到了什么程度」。把数字喂给 AI，它帮你把这些成果摆到领导一眼能看见的位置，这份周报就立住了。

改写三：起一个公众号标题

这是我自己天天在琢磨的活，分享一个真实的对比。

改写前那种问法，「帮我给这篇文章起个标题」，AI 会给你一堆正确的废话标题，平淡得像新闻联播。

我自己起标题时，喂给 AI 的是这样一份简报：角色定成「一个深谙公众号传播、会起钩子但不靠烂俗标题党的运营」；目标是「给这篇讲 Prompt 公式的文章起 8 个备选标题」；背景里我会塞进文章的核心观点、目标读者是谁、我这个号一贯的调性是「真实、不端着、爱说人话」；要求里写死「每个标题 20 字内，要有具体的钩子或数字，不要用『终极』『必看』这种烂大街的词」。

最后那条「不要用什么词」尤其关键。我那份规则手册里专门有一类，就是收集这些被读者免疫的烂词。把这些坑提前堵上，AI 给的备选就能直接用三四个，剩下的我挑一个改改就发。这套打法，就是我把日常驯服 AI 的经验，压缩成了几行可复用的要求。

这套公式，你今天就能用起来

说回最开头那个现象，那些用两下就放弃 AI 的人，缺的从来不是一个更强的模型，是一份把话说全的简报意识。我每天靠一份手册和一份说明文件驱动 Claude Code 跑活，本质上跟你给 AI 写一条朋友圈，是同一件事，把需求讲清楚，把背景喂足，把要求钉死。

刚开始你可能觉得，写四行简报比直接打一句话费劲多了。但你磨一句好 prompt 花的那两分钟，省下的是来回返工五六次的半小时。这是我这一年多攒下来最划算的一笔时间账。

你先别贪多，挑一个你最常让 AI 干的活，比如写周报，把「角色目标背景要求」四行套上去试一次，对比一下跟以前那句话出来的东西差多少。你最近一次被 AI 气到答非所问，是让它干的哪件事？评论区告诉我，我帮你改成一份能用的简报。
