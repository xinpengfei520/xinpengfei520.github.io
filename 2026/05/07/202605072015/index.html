我上周赌 Anthropic 会涨价加限制，今天它租下 Musk 的 22 万块 GPU 把限速翻倍了

发表于 2026-05-07 更新于 2026-06-10 分类于 AI 阅读次数： 本文字数： 4.4k 阅读时长 ≈ 4 分钟

大家好，我是飞飞。

我先把这篇文章最重要的一件事说出来：上周我赌错了。

4 月 30 日那篇《Anthropic 把 Claude Code 日均成本估算从 $6 改成 $13》的文章里，我在结尾给了一个赌注：”Avasare 那句话出来之后，Anthropic 接下来三个月要么砍掉 Pro 一档的 Claude Code、要么涨价、要么在 Max 5x 上加更严的’使用强度’限制。三选一。我个人倾向认为是后两者的组合。”

5 月 6 日，Anthropic 在 Code with Claude 开发者大会上选了第四条路：租下 SpaceX 旗下 Colossus 1 数据中心的全部算力，22 万块 NVIDIA GPU、超 300 兆瓦电力，月内全部到账。同一天宣布 Claude Code 五小时限速翻倍、Pro 和 Max 取消高峰限流、Opus API 速率大幅上调。

不是涨价。不是加限制。是反过来给了所有现有付费用户一倍的容量。

下面写一下我对这件事的看法，以及我赌错的那一段判断逻辑究竟漏掉了什么。

5 月 6 日具体改了什么

把硬动作先列出来，免得后面跑题。

Dario Amodei 5 月 6 日在 Code with Claude 大会上当场宣布的三件事，当天即生效：

  • Claude Code 五小时窗口限速翻倍：覆盖 Pro、Max、Team、按座位计费的 Enterprise 全部档位。
  • 取消 Pro 和 Max 的高峰时段限流：以前白天峰值期间会被收紧的额度，现在拉平。
  • Claude Opus API 速率大幅提升：9to5Google 拿到的数字是 Tier 1 用户输入每分钟 token 数提升 1500%、输出每分钟 token 数提升 900%。其他 Tier 也有类似量级的放宽。

这套组合对应的算力来源是这样一套基础设施叠加：

  • SpaceX Colossus 1：Memphis 数据中心的全部算力。300+ 兆瓦电力、22 万+ NVIDIA GPU（包括 H100、H200、以及下一代 GB200 加速器）。月内交付。
  • Amazon $25B 5GW 协议（4 月签的）：算力的主基座，并将延伸到亚洲和欧洲，专门给金融、医疗、政府这种受监管行业做 in-region inference。
  • Google、Microsoft、Broadcom 等的算力协议：跨多家公布。
  • 轨道算力的兴趣：Anthropic 公开表态愿意和 SpaceX 一起搞”multiple gigawatts of orbital AI compute capacity”。SpaceX 自己最近向 FCC 申请了百万颗卫星建轨道数据中心。这一项目前没有签约。

把 SpaceX 和 Amazon 这两笔加起来看，Anthropic 是把 5 GW + 300 MW 的新增算力同时塞进自己的供给曲线。这是过去半年所有大模型公司里最激进的一次扩张。

我赌错在了哪里

回头看 4 月 30 日那篇文章的逻辑，我赌”涨价或加限制”建立在两条假设上。

一条是订阅制本质上是固定价格、变动成本。当使用强度暴涨时，价格不动只剩两条路：动价格、或者动可用量。

另一条是 Anthropic 的 Head of Growth 那句”current plans weren’t built for this”已经把”现有方案撑不住”的判断直接抛出来了。一家公司不会无缘无故抛这句话。

这两条逻辑本身没错。错在我漏掉了第三个变量：算力供给本身可以爆发式扩张。

我在写那篇文章的时候，潜意识把”算力”当成了一个慢变量。Amazon 的 $25B 协议在 2026 年 2 月签，但 5GW 上线”需要时间”。Google 和 Microsoft 的协议落地节奏也不是月度的。基于这种慢变量假设，订阅制只能用价格或限额来匹配需求。

但 SpaceX 这一笔不一样。Colossus 1 是已建成、已通电、且 Musk 自己用不充分的算力。Anthropic 把它整租下来等于买了一个现成的 300 MW，而且月内就能交付。这种”几乎现货”的算力供给我没算进我的赌注里。

PCMag 的报道里有一句让我看明白这件事的话：”Anthropic is taking over ‘all of the compute capacity’ at xAI’s Colossus 1 supercomputer, amid reports that Elon Musk’s startup is struggling to efficiently harness its GPUs.” Musk 的 xAI 自己用不动这堆 GPU 是已经被 reported 出来的事。Anthropic 接手相当于把一个本来产能闲置的算力池一次性吃满。

我承认这件事没有出现在我赌注的”假设空间”里。这是我赌错的关键。

这件事最有戏剧性的一笔是 Musk 自己

Musk 公开骂过 Anthropic 不止一次。Ars Technica 那篇报道的原话是 “Musk has publicly attacked Anthropic on multiple occasions”。

但 5 月 6 日那天 Musk 在 X 上的发言彻底翻转：

I spent a lot of time last week with senior members of the Anthropic team to understand what they do to ensure Claude is good for humanity and was impressed.

紧接着他补了一条带刺的：

We reserve the right to reclaim the compute if their AI engages in actions that harm humanity.

这句”我们保留撤回算力的权利”是这次合作里最 Musk 的一笔。它技术上不影响 Anthropic 拿到的合同义务（一旦合同签完算力交付方就要按约履约），但它把”商业合作”包了一层”我可以随时翻脸”的姿态。

为什么 Musk 在公开撕了 Anthropic 这么久之后选择翻脸合作？我能猜到的几个原因：

SpaceX 准备 6 月 IPO。投资者要看到的是”我们的 AI 算力业务有真正的大客户”。Anthropic 这个名字在投资者眼里比”自家用 xAI 跑 Grok”分量重得多。

Colossus 1 的算力闲置是公开的痛点。22 万块 GPU 维护成本高昂，跑不满就是亏。

Anthropic 也准备 IPO，6 月节点附近。这一手”算力翻倍 + 用户体验改善”对它的投资者故事同样有用。

两家 IPO-bound 的公司，在 IPO 前两个月做了一笔互相需要对方的交易。这件事看起来是 AI 行业的反常剧情，但放回到资本运作的逻辑里其实很合理。

这件事对昨天那篇”AI 产品分裂论”的修正

我 5 月 5 日写过一篇文章，论点是 OpenAI 押广告路线、Anthropic 押订阅路线，AI 行业从这一周开始被劈成两条路。

5 月 6 日的 Anthropic 动作让那个论点更清晰了，但分裂的层面比我当时想的更深。

OpenAI 那条路：让用户继续免费/低价，用注意力换收入。基础设施需求是”覆盖更多用户、更多场景”。

Anthropic 这条路：让付费用户继续付费、付得起的人用得更爽，用资本支出（算力）换续航。基础设施需求是”够用的算力 + 用户使用强度上限”。

5 月 6 日的硬动作把 Anthropic 这条路的真实底牌摊开了：靠多笔算力协议堆出来的供给曲线，让”扩张算力”赶得上”用户使用强度提升”。如果这条赛跑能跑得过，订阅制不需要涨价、不需要加限制。

Anthropic 现在赌的就是这条赛跑能跑过去。它在去年签的 Amazon $25B 5GW、今年签的 SpaceX Colossus 1，加上 Google / Microsoft / Broadcom 几条线，是它给这条赛跑下的全部资本注码。

如果 Opus 5、Opus 6 在能力提升的同时不大幅增加单位 token 算力消耗，那这套基建还能再撑两年。如果模型尺寸继续翻倍但能力涨得更慢，订阅制还是会被打穿。

对我自己工作流的实际意义

我自己是 Claude Max 5x $100/月，主力工作流是 Claude Code 跑 content pipeline。

5/6 之后我能感受到的具体变化：

我连着两天跑完整博客 pipeline 的时候经常撞 5 小时窗口的限流（4/30 那篇文章里我提过”一周撞限流 6 次降到 2 次”）。窗口限速翻倍之后，正常使用强度下应该完全不再撞墙。

高峰时段限流取消这一条对我意义比较大。我有时候在美西高峰时段（亚洲深夜）跑长任务，那时候原本会被收紧。这个限制取消之后我可以放心通宵跑 ultraview / 长 refactor 任务。

API 速率提升对我直接影响比较小，因为我主要用订阅而不是按 token 走 API。但我那几个小工具（content-publisher 走 wx 接口、Claude Desktop 开发者模式调 API 端点）会受益。

最关键的判断是：短期内（6 月 IPO 节点之前）订阅价大概率不会涨。Anthropic 不会在 IPO 前两个月做一个让用户怒的动作。如果你之前因为 4/30 那条新闻在考虑切走，可以先按兵不动。

中期看（6 个月内）我还是把 Claude Desktop 开发者模式接 DeepSeek V4 的备份方案留着。算力红利能持续多久取决于两件事：Musk 心情 + IPO 之后市场对 Anthropic 业绩压力。前者我没法预测，后者只会向上走。

那个”轨道算力”到底是认真的还是营销

最后留一段说轨道算力。

Anthropic 在公告里说”expressed interest” 和 SpaceX 合作 multiple gigawatts 的轨道算力。SpaceX 最近向 FCC 申请了百万颗卫星建轨道数据中心。

我的判断是这件事 70% 是认真的、30% 是营销。

认真的部分：地面数据中心的电力、土地、冷却已经在大模型这一代开始撞物理上限。Anthropic 自己在公告里说 “compute required to train and operate the next generation of these systems is outpacing what terrestrial power, land, and cooling can deliver on the timelines that matter”。这话不是公关辞令。地面算力的物理瓶颈是真问题。

但轨道算力面临三个硬约束没解：上下行带宽、辐射环境对芯片寿命的影响、卫星之间互联的延迟。这三个约束目前没有公开的工程方案能在 5 年内解掉。

营销的部分：在 IPO 前两个月放一个”我们也在搞太空数据中心”的故事，对投资者画饼非常管用。这件事我会持续盯但不会很快当真。

评论区想问两件事。

如果你也是 Claude Max 用户，5/6 之后这一两天用下来感觉限速真的变松了吗？我自己 5/7 早上跑了一个长任务没撞墙，但样本太小。

另外，你怎么看 Musk 那句”保留撤回算力的权利”？是单纯的姿态，还是合同里真有触发条款？我自己赌是 PR 姿态，没有真触发条款。但这件事如果未来出现摩擦，会很有意思。
