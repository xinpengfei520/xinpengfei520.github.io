agent 循环分四档：我十几个 skill 对号入座，最放手那档坑了我一整天

发表于 2026-07-08 更新于 2026-07-14 分类于 观点 阅读次数： 本文字数： 2.5k 阅读时长 ≈ 2 分钟
Claude Code 官方把 agent 循环分成四档：回合、目标、时间、主动，本质是一道交出判断权的梯子，越往上越省心也越担风险。我拿天天跑的十几个 skill 对号入座：回合占九成，时间循环靠 GitHub Action 每周一自动拉爬虫周报，主动循环那次让部署静默失败一整天才发现。先从回合起步，主动最后上，且必须配一个人能看见的告警。

哈喽，我是飞飞。

这两天 Claude Code 团队发了篇文章，把 agent 的「循环」正经分了类。就是你让 AI 自己转起来干活的那套东西，官方拆成四档：回合、目标、时间、主动。我看完没急着记概念，先对着自己天天跑的那十几个 skill，一档档往上套：我到底跑到了哪一格，哪一格我压根没敢碰。

我前阵写过一篇「怎么把单个 skill 写得不飘」，讲的是把一个 skill 内部抠稳。这篇不一样，说的是这些 skill 转起来的那个圈怎么设计，是上一层的事。四档看着是四个功能，其实是一道梯子。越往上，你交出去的判断越多，能省的手也越多，可一旦出事，你越晚知道。选爬到哪一格，真正的分界不在它多自动，在你敢不敢让它没人盯着自己跑。

四类循环其实是个梯子

先把这道梯子摆开，就四格。

最底下是回合循环。你打一句话，AI 干一轮，自己判断这轮完没完，就停下等你。你看一眼结果，觉得行就往下走，不行就再来一句。每一轮你都在场，都在验。

往上一格是目标循环。你给它一个能验收的目标，它自己一轮轮试，直到达成、或者撞到你设的轮数上限才停。官方那套里，每跑完一轮还有个更小的模型专门来判「到没到」，写代码那个 agent 不给自己打分。

再往上是时间循环。不靠你敲，靠时钟或者事件来敲。每天九点、每来一个 PR，它自己启动跑一遍。你人可以不在，甚至能把它挪到云端，关了电脑它照跑。

最上面是主动循环。连什么时候跑都不用你定了，它自己找活、自己排优先级、自己收尾，跑完一件退一件，整条链没有人在实时盯着。

你发现没有，从下往上每爬一格，你就少盯一点，它就多担一点判断。省心和风险，是朝同一个方向长起来的。这就是我说它是梯子、而不是四个平级选项的原因。

回合循环我天天在跑

说我自己。我这十几个 skill，九成时间跑的都是最底下这格。

我写文章一整条流水线：拉资讯、查重、写稿、润色、配图、分发，全是我在 Claude Code 里手打一句话触发一轮。「帮我配张封面」，content-artist 转一圈，图就传到我图床了；「这篇润一下」，content-polisher 跑一轮，把 AI 味压下去。每一轮跑完它停下，我扫一眼，接着给下一句。

这格最大的好处是踏实。我全程在场，哪一轮飘了当场能拽回来。官方也说，回合循环跑得稳不稳，主要看你把验收那步写得清不清楚，跟模型多聪明关系不大。我这套 skill 里那本防 AI 味的规则手册，喂到一百多次，就是在给「这轮到底合不合格」定一把越来越细的尺子。

所以别小看最底下这格。绝大多数活，回合循环就够用了，稳、可控、出事早知道。往上爬是为了省手，不是为了显得高级。

目标循环到底适合谁

目标循环这格，我盯着看了半天，最后没往上套。老实说一句，我到现在没拿它正经跑过一个活。

它本身没毛病，问题在我的活不对路。目标循环的命根子，是得有一个能机器验收的完成标准。像「所有测试跑通」「接口返回全对上 OpenAPI」这种，机器一测就知道成没成，它才敢自己一轮轮磨到达标。

可我写文章这套，完成标准是软的。AI 味压到多少算干净？字数卡在两千到两千五算合适？这些没有一条能让机器斩钉截铁判过。硬要给它设个目标让它自己跑，它多半会在一个我并不认同的标准上反复打转。

这算不上它的短板。是我还没找到适合上目标循环的活。哪天我拿它去跑一件有硬验收的差事，比如把一批链接的失效状态查到全绿，可能就对上了。这一格适合谁，取决于你的活有没有一条机器能判死的完成线。

时间循环替我盯周报

时间循环这格，我真用上了，还用得挺舒服。

我给博客做 GEO，写了个 GitHub Action，每周一早上九点自己启动，生成一份「AI 爬虫周报」的工单。我人不用在，到点它自己跑。这就是标准的时间循环，靠时钟敲，不靠我敲。

但我留了个心眼：它只管生成，不管拿主意。周报拉出来那些数，哪家爬虫在涨、哪条回流是噪声、该不该动 robots，还是我自己看着办。我把「按点去拉一遍」这段枯燥的搬运交给了时钟，把「看完怎么改」这段判断留在自己手里。

你可能会说，那你干嘛不让它连分析带改一起干了。因为往上再走一格，性质就变了。生成一份报告放那儿，错了也不伤筋动骨；可要让它照着报告自己去改线上配置，那就是下一格的事，风险不是一个量级。

主动循环最强也最险

最上面这格，我既眼馋，又发怵。

眼馋是真见过它多能干。我写过一篇，卡兹克让模型去优化一个网站的运维，它自己起了二十多个 agent 调研、自己提工单、把对接工程师漏答的问题追回来，一整条运维链一晚上跑完。那种自己找活自己收尾的劲儿，正是主动循环。这些是他记录的，我没亲手跑过，先说清楚。

发怵是真被咬过。我那条博客发布流水线，push 一下自动构建、自动部署，还自动把新链接推给搜索引擎收录，挺主动。结果有回负责部署的一个令牌过期了，构建每次都成功，只有最后一步认证悄悄红了。没人盯着 Action 状态，它就这么静默失败了整整一天，我隔天主动去翻才发现，镜像站已经挂了一天。

这件事我记到现在。主动循环最贵的成本，压根不在算力。它烧的是另一样东西：没人盯着的时候，失败会悄没声地一直堆。你越往这格放手，越得给失败本身配一个人能看见的告警，不然它跑得越欢，你摔得越闷。

别急着爬到最后一格

绕回开头那道梯子。四格排下来，很容易让人想一步蹦到最顶上，觉得越自动越牛。官方那篇反倒劝你从最简单的起步，能用回合循环解决的，别去搭一套复杂编排。

我自己就是这么走的，大部分活至今停在最底下那格。给你一个今天就能上手的最小动作：挑你手上最琐碎、又纯只读的那段活，比如每天拉一遍日志、算一算哪个数在涨，交给时间循环替你按点跑，但只让它给结论，别让它动手改任何线上的东西。你要做的，从自己趴着查，变成审它给的那份结论。等你审了几回、回回都对得上，再考虑要不要往上放一格。

这道梯子最难的地方，不在你能爬多高，在你每松一次手之前，有没有先想明白一件事：这一格真出了岔子，我多久才会知道。

说回来问你一句：你手上那些让 AI 自己转的活，跑到哪一格了？又是哪一步，你打死都要自己在场按那个确定键？评论区聊聊，我挺想知道大家那只手，都松到了哪一格。
